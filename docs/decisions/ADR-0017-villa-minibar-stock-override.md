# ADR-0017 — 빌라별 미니바 비치수량 오버라이드 (#2c)

- 상태: 채택 (2026-06-25)
- 관련: [[ADR-0016-minibar-company-standard]](전 빌라 공통 1세트), 원칙2(마진 비공개), [[db-schema-drift-villa-source]]

## 배경

ADR-0016은 미니바를 **회사표준 1세트(`MinibarItem`)**로 통일하고 "빌라별 오버라이드 없음"으로 결정했다. 운영 결과(테오 2026-06-25): **빌라마다 냉장고 크기·현재 재고가 달라** 기본 비치 수량이 동일할 수 없다. 품목·단가는 공통이지만 **비치 수량은 빌라별로 달라야** 체크인 시트의 "기본 비치 수량"이 실제와 맞는다.

## 결정

ADR-0016의 "빌라별 오버라이드 없음"을 **수량(quantity)에 한해 완화**한다.

- 새 모델 `VillaMinibarStock(villaId, minibarItemId, qty)` — (빌라 × 품목) 당 1행, 비치 수량만 저장.
- **가격(`unitPriceVnd`)은 절대 빌라별로 두지 않는다.** 여전히 `MinibarItem` 한 곳에만 존재 → 원칙2(마진 비공개) 불변. 수량은 마진 정보가 아니므로 빌라별 저장이 누수와 무관하다.
- **기본값 = 회사표준 `stockQty`.** 오버라이드 행이 없는 (빌라×품목)은 표준 수량을 그대로 사용.
- **표준과 같은 수량으로 저장하면 오버라이드 행을 삭제**한다 → 이후 표준 수량 변경을 자동 추종(불필요한 동결 방지).

## 변경 범위

- **DB**: `VillaMinibarStock` 테이블 raw SQL CREATE(`prisma db push` 금지 — Villa.source 드리프트 드롭 회피). `prisma/migrations-manual/2026-06-25-villa-minibar-stock.sql`.
- **API**: `PATCH /api/villas/[id]/minibar-stock` — 운영자(`isOperator`) 전용, SUPPLIER 차단. 가격 미조회·미수정. AuditLog 필수.
- **운영자 빌라 상세**: 회사표준 미니바 카드를 **수량 편집기**로 전환(`minibar-stock-editor.tsx`). 단가는 `canViewFinance`일 때만 표시(원칙2). 품목·단가 편집은 여전히 `설정→미니바` 단일 경로.
- **체크인 시트**: 미니바 표의 "기본 비치 수량"을 그 빌라의 오버라이드(없으면 표준)로 인쇄.

## 대안 (기각)

- 빌라별 가격 오버라이드 — 원칙2 위반 위험으로 기각. 수량만.
- 빌라별 품목 on/off(이 빌라엔 소주 없음) — 이번 범위 밖(수량 0으로 표현 가능). 후속 검토.
- `MinibarItem`에 빌라별 JSON 컬럼 — 조회·정합성 불리, 정규화 테이블 채택.

## 결과

- 품목·단가는 회사표준 1세트로 통일 유지, 비치 수량만 빌라 현실(냉장고·재고)에 맞춤.
- 가격은 `villaId` 없는 테이블에만 존재 → 공급자·공개 경로에서 구조적으로 도달 불가(원칙2 강제) 유지.
