# ADR-0044: 요금 기간 겹침 허용 — 레이어 모델 + 밤별 승자 규칙

- 상태: Accepted (2026-07-14, 테오 목업 승인)
- 관련: ADR-0014(기간별 요금), ADR-0031(소비자 직판가), ADR-0042(프리미엄일), 계약 rate-calendar-ux.md
- ⚠ 병합 직전 main에서 ADR 번호 충돌 재확인 (0040 사례)

## 배경

ADR-0014는 기간 겹침을 입력 단계에서 409로 거부했다. 그 결과 "겨울 성수기(12/20~3/1) 안에 설 연휴(2/5~2/11) 웃돈"을 넣으려면 성수기를 3조각으로 쪼개야 했고, 이벤트 해제 시 다시 이어붙여야 했다. 기간이 늘수록(요일·공휴일·이벤트·연도 반복) 폼 리스트 UI로는 "이 날짜에 얼마가 적용되는지"를 파악할 수 없다 — 테오가 캘린더 기반 전면 개편 + 겹침 허용을 요청.

## 결정

### D1. 겹침 허용 + 밤별 승자 규칙 (스키마 무변경 핵심)
한 밤(@db.Date)을 커버하는 기간이 여럿이면 **정확히 하나의 승자 행**이 그 밤의 요금을 전담한다:

> ① 짧은 기간(박 수) 우선 → ② 시즌 등급(PEAK>HIGH>SHOULDER>LOW) → ③ 늦은 startDate → ④ id 사전순 큰 쪽(cuid ≈ 최신 행)

- "금액 큰 쪽 승리"는 ADR-0042에서 기각된 사유 그대로 배제 — 가격이 다축(원가/Net/소비자가 × VND/KRW)이라 승자 정의 불가.
- ①구체성이 1규칙인 이유: "큰 시즌 위에 작은 이벤트를 얹는다"가 사용자 멘탈 모델이고, 캘린더 UI가 밤별 승자를 항상 표시하므로 규칙 암기가 불필요.
- 승자 행 결정 **후**의 가격 해석은 기존 계층 그대로: 프리미엄 컬럼 폴백(premiumX ?? X, ADR-0042) → 채널 계층(CONSUMER=consumer??Net, ADR-0031). 축 우선순위를 새로 만들지 않는다.
- base(isBase=true)는 어떤 기간도 커버하지 않는 밤에만 — 기존과 동일.
- 라이브 데이터는 겹침 0(과거 409 거부)이므로 백필·마이그레이션 불요, 기존 견적 결과 불변.

### D2. 레이어 = 보존되는 원본 (삭제=복원)
겹침 행을 "레이어"로 취급한다. 이벤트 레이어를 지우면 아래 시즌 요금이 자동 복원된다 — 기간 쪼개기/이어붙이기 소멸. 일괄 조정·연도 복사도 원본을 수정하지 않고 **새 레이어 생성**으로만 동작한다.

### D3. batchId (additive 컬럼 1개)
일괄 작업(일괄 조정 ADJUST·선택 적용 SET·연도 복사 COPY_YEAR)이 만든 레이어들은 동일 `batchId`(cuid)로 묶는다 → 그룹 단위 취소("이 작업 전체 취소")·목록 묶음 표시. 수동 생성 행은 null. `2026-07-14_rateperiod_batchid.sql` 라이브 적용 완료.
- label 규약(라벨 접두어로 그룹 식별)은 사용자 편집으로 깨지므로 기각.

### D4. API — 전체 교체에서 레이어 단위 CRUD로
기존 PATCH(deleteMany→createMany 전체 교체)는 동시 편집 시 나중 저장이 먼저 저장을 통째로 덮는다. 캘린더 도구는 증분 작업이므로 레이어 단위 POST/PATCH/DELETE + batch POST/DELETE를 추가한다. 전체 교체 PATCH는 하위호환으로 유지(마법사·기존 편집기). 권한은 기존과 동일(canSetPrice), 전 변경 writeAuditLog.

### D5. 일괄 작업의 서버 구간화
ADJUST는 대상 범위의 밤들을 "현재 승자 행" 기준으로 구간화한 뒤 구간별 레이어를 생성한다(승자가 다른 구간은 각각 조정). 반올림: VND 1,000₫, KRW 100원. 구간화 로직은 lib 순수 함수로 분리해 FE·테스트와 공유.

### D6. 연도 복사는 "복사 + 검토", 자동 반복 아님
"매년 자동 반복" 플래그는 기각 — 설날 등 음력 연휴는 해마다 날짜가 달라 자동 반복이 항상 틀리고, 해석 로직 복잡도만 늘린다. 같은 월·일 시프트(2/29→2/28 보정) 복사 + UI 경고(음력 레이어는 복사 후 날짜 이동) + 가격 인상률(%) 옵션이 정본.

## 기각한 대안
- 겹침 금지 유지 + 캘린더가 자동 분할/병합(painting 모델): 원본 의도(시즌 정의)가 파괴돼 이벤트 해제 시 복원 불가. 테오 요구("중복 중첩")와도 상충.
- 명시적 우선순위 컬럼(수동 정렬): UX 부담. 파생 규칙 ①~④가 결정적이고, 필요 시 additive로 추후 도입 가능(IDEAS).
- 컬럼 단위 레이어 폴백(승자 행의 null 컬럼을 하위 레이어에서 상속): 표현력은 크지만 추론 불가 수준의 복잡도 — 기각, 승자 행 전담 + 행 내 기존 폴백만.
