# 런북: rate-limit 잠금·해제 (보안 P1-S4)

## 정책
- 로그인·가입·공개 토큰 엔드포인트는 슬라이딩 윈도우 rate-limit로 보호된다(`lib/rate-limit.ts`).
- **영구 잠금은 없다.** 한도 초과 시 윈도우(대개 10분)가 지나면 자동 해제된다.
  - 로그인: 전화번호 5회/10분 + IP 20회/10분 (`auth.ts`). 성공 시 전화번호 카운터 즉시 리셋.
  - 가입(signup·partner·vendor): IP 10회/시간. 공개 토큰(/p·/g): 라우트별 토큰·IP 한도.

## 잠금 즉시 해제 (운영자 대응)
정상 사용자가 잠긴 경우(예: 같은 사무실 공유 와이파이로 IP 한도 초과):
1. **10분 대기** — 가장 안전. 윈도우 만료로 자동 해제.
2. **인스턴스 재시작** — 메모리 스토어가 초기화되어 전 카운터가 리셋된다(현재 단일 인스턴스 가정).
   - ⚠ 전체 카운터가 비워지므로, 활성 공격 중에는 권장하지 않는다.
3. (후속) 특정 키만 해제하는 운영자 엔드포인트는 미구현 — 필요 시 `resetRateLimit(key)` 노출.

## 스케일아웃 시
메모리 스토어는 인스턴스별로 독립이라 다중 인스턴스에서 한도가 분산된다.
`setRateLimitStore()`에 Redis 등 공유 `RateLimitStore` 구현을 주입하면 전 인스턴스가 동일 카운터를 공유한다(호출부 무변경, P1-S4 추상화). Redis 구현은 후속(P2-S5).

## IP 한도 false-positive (NAT·공유 와이파이)
베트남 공급자 다수가 한 IP를 공유하면 IP 한도(20/10분)에 묶일 수 있다. 1차 방어는 스푸핑 불가능한
**전화번호 한도**이므로 IP 한도는 보조다. 출시 후 Railway XFF 토폴로지 실측으로 한도·신뢰홉을 조정한다(P2-S4).
