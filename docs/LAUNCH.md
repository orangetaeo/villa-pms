# LAUNCH.md — 오픈 스펙·런칭 플랜

> **원칙: 오픈 기준을 최소로 고정하고, 나머지는 전부 오픈 후 개선한다.**
> 범위를 늘리고 싶은 유혹은 IDEAS.md로 보낸다. 오픈이 늦어지는 것이 최대 리스크.

## 1. 오픈 기준 체크리스트 (이것만 되면 연다)

- [ ] F0 자가 가입 + Zalo 연결 동작
- [ ] F1 빌라 등록 → ADMIN 승인 → 요율 입력 한 사이클 완주
- [ ] F2 캘린더 차단/해제 + iCal 수신 동기화
- [ ] F3 제안 링크 → 가예약(HOLD) → 입금 확정 → 홀드 만료 자동 해제
- [ ] F4 체크인(여권 OCR·동의서·보증금) → 체크아웃(사진 비교) → 청소 검수 게이트
- [ ] F5 Zalo 알림 7종 발송 (표 F5 참조)
- [ ] F6 최소 정산: ADMIN 월 집계 + SUPPLIER 원가 조회
- [ ] 권한 누수 4종 테스트 통과 (QA — 마진·재고 비공개 원칙 검증)
- [ ] 파일럿 실데이터 입력 (쏘나씨 V11/V12/V25 + 썬셋 사나토)
- [ ] 베트남 사용자 1명 실사용 테스트 통과 (T4.4)
- [ ] 배포 스모크 테스트 3종 (OPS)

**오픈 기준이 아닌 것** (오픈 후 개선): 시즌 요율 자동계산 고도화, 정산 PDF·다중 통화, 품질점수, 부가서비스 판매, SMS 폴백, 지도 뷰, 예약 변경 기능, Zalo 로그인

## 2. 파일럿 → 확대 로드맵

| 단계 | 내용 | 종료(승급) 조건 |
|---|---|---|
| P0 파일럿 | 공급자 1명(쏘나씨 보유 중계인) 온보딩, 테오가 옆에서 동행 | 실예약 1건 완주: 제안→홀드→확정→체크인→체크아웃→청소 승인→정산 |
| P1 소규모 | 공급자 3~5명, 빌라 10~15채 | 공급자가 **혼자** 빌라 등록·캘린더 관리 성공 (설명 없이) |
| P2 확장 | 자가 가입 링크 공개 배포, 판매 채널 확대 | 월 예약 안정 운영 + 더블부킹 0건 유지 |

## 3. 첫 달 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|---|---|---|
| 제안 → 가예약 전환율 | ≥ 30% | Proposal USED / 전체 발송 |
| 가예약 → 확정 전환율 | ≥ 50% | CONFIRMED / HOLD 생성 |
| 공급자 자력 등록 | 설명 없이 완료 | T4.4 + 파일럿 관찰 기록 |
| 청소 승인 소요 | 평균 < 24h | CleaningTask 타임스탬프 |
| 더블부킹 | **0건** | iCal 충돌 경보 로그 |
| Zalo 발송 성공률 | ≥ 95% | Notification SENT/전체 |

## 4. 오픈 후 개선 루프 (사업 운영 방식)

1. **수집**: 공급자 피드백은 Zalo(개별·단톡), 한국 여행사 피드백은 카톡 → 테오가 수집
2. **기록**: 모든 요청·아이디어는 IDEAS.md에 먼저 기록 — 즉시 구현 금지
3. **선별**: 격주 우선순위 회의(PM 주관)에서 IDEAS → TASKS 승격. 승격 기준 3문:
   - ① 예약이 더 늘어나는가? ② 공급자 이탈을 막는가? ③ 테오의 운영 시간을 줄이는가?
4. **축적**: 버그·교훈은 코드 수정과 함께 .claude/skills/ 해당 파일 "교훈 축적"에 패턴화

## 5. 데이터 정책

| 데이터 | 정책 |
|---|---|
| 여권 사진 | 체크아웃 90일 후 삭제 (Phase 1 분기별 수동, Phase 2 자동 cron) |
| 증빙 사진 (빌라·체크아웃·청소·파손) | R2 영구 보존, 수정 불가, 타임스탬프+업로더 기록 |
| 감사 로그 (AuditLog) | 전체 변경 기록 영구 보존 — 분쟁 시 1차 증빙 |
| 가격 스냅샷 | HOLD 시점 고정, 소급 변경 금지 |
