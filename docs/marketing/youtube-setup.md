# 유튜브 채널·API 셋업 가이드 — 테오용 (v1, 2026-07-16)

> **누가 읽나요?** 개발자가 아닌 운영자(테오)가 혼자 따라 할 수 있게 만든 안내서입니다.
> **무엇을 하나요?** 빌라고(villa-go) 전용 유튜브 채널을 만들고, 우리 프로그램이 **자동으로 쇼츠(Shorts) 영상을 올릴 수 있도록** 연결하는 것까지가 목표입니다.
> **얼마나 걸리나요?** 채널·연결 작업은 40~60분. 단, **"감사(audit) 신청"은 통과까지 수 주가 걸릴 수 있어** 가장 먼저 신청만 해두고 나머지를 진행합니다.

---

## ⭐ 시작 전에 꼭 알아둘 3가지 (인스타와 다른 점)

유튜브는 인스타보다 관문이 하나 더 있습니다. 헷갈리는 용어부터 정리합니다.

### 1) "심사"가 **두 종류**인데, 우리에게 필요한 건 **하나뿐**입니다.

| 심사 | 무엇을 위한 것 | 우리는? | 서류? |
|---|---|---|---|
| **① 구글 OAuth 앱 검증**(App Verification) | "미확인 앱" 경고를 없애고 **아무나 100명 넘게** 우리 앱에 로그인하게 하려는 것 | **안 합니다.** 우리는 **테오 채널 딱 하나**만 연결하므로, 경고 화면은 "고급 → 계속"으로 넘기면 되고 100명 제한도 무의미 | 불필요 |
| **② 유튜브 API 감사**(Compliance Audit) | API로 올린 영상을 **"공개"로 나오게** 하려는 것 | **합니다(무료 폼).** 이걸 안 하면 우리가 올린 영상이 **강제로 "비공개"** 가 됩니다 | 서류 불요, 사용목적 설명만 |

> **핵심**: ①(구글 앱 검증)은 **안 해도 됩니다.** 자기 채널만 쓰면 경고만 한 번 넘기면 끝. 하지만 ②(유튜브 감사)는 **공개 노출을 원하면 필요**합니다. 이 둘은 완전히 다른 절차라 헷갈리지 마세요.

### 2) refresh token "7일 만료 함정" — 반드시 **"프로덕션(게시됨)"** 상태로 둡니다.

우리 프로그램이 매일 자동 업로드하려면 **"리프레시 토큰"**(자동 로그인 열쇠)이 필요합니다. 그런데:

- OAuth 동의 화면을 **"테스트(Testing)"** 상태로 두면 → 이 열쇠가 **7일마다 죽습니다.** (그러면 일주일마다 다시 연결해야 하는 지옥)
- **"프로덕션(In production / 게시됨)"** 으로 바꾸면 → 열쇠가 **사실상 영구**(6개월 이상 안 쓰면만 만료). ← 우리는 반드시 이 상태.
- 프로덕션으로 바꿔도 (위 ①처럼) **구글 검증은 안 받아도 됩니다.** 그냥 "게시" 버튼만 누르면 됩니다. 우리 계정으로 연결할 때 "미확인 앱" 경고가 한 번 뜨는데, "고급 → 계속 진행"으로 넘어가면 정상 작동합니다.

### 3) 쿼터(하루 업로드 한도)는 넉넉합니다.

- 2025년 12월 정책 변경으로 **영상 1건 업로드 비용이 크게 낮아졌습니다**(약 1,600 → 약 100). 이제 **하루 수십 건**도 여유입니다. 우리 목표(하루 1~3건)는 전혀 문제없습니다. (감사 통과 전에도 업로드 자체는 됩니다 — 단지 "비공개"로 올라갈 뿐.)

## 1단계 — 구글 계정 + 유튜브 채널 만들기 (브랜드 채널 권장)

1. **구글 계정**: 인스타와 마찬가지로 **업무용 지메일**을 씁니다(개인 계정과 분리). 이미 빌라고 전용 지메일이 있으면 그걸 씁니다.
2. **유튜브 채널 생성** — youtube.com 접속 → 로그인 → 오른쪽 위 프로필 → **설정** → **채널 만들기**.
   - 가능하면 **"브랜드 채널"(브랜드 계정)** 로 만드세요. 개인 이름이 아닌 **"빌라고 푸꾸옥"** 같은 채널명을 쓸 수 있고, 나중에 여러 사람이 함께 관리하기 좋습니다.
   - 채널명 예: `빌라고 푸꾸옥 풀빌라` 또는 `Villa-Go 푸꾸옥`.
3. **채널 꾸미기**:
   - **프로필 사진**: `villa-go-logo-640.png` (인스타와 동일 로고)
   - **배너**: `villa-go-banner-800x400-teal.png` 등 빌라고 배너 에셋 재사용
   - **채널 설명**: 인스타 소개글과 같은 톤 + **카카오톡 채널 링크**(`http://pf.kakao.com/_mVAfX`)
   - **채널 링크**: 채널 맞춤설정 → 링크에 카카오톡 채널 주소 추가 (쇼츠 설명 링크는 클릭이 안 되므로, **채널 프로필 링크가 유일하게 클릭되는 통로**입니다 — 꼭 넣으세요)

## 2단계 — Google Cloud 프로젝트 + YouTube Data API 켜기 (무료)

> 여기부터는 **PC 웹브라우저**로 하는 게 편합니다. 위 1단계에서 만든 **같은 구글 계정**으로 로그인합니다.

1. **console.cloud.google.com** 접속 → 로그인.
2. 상단의 프로젝트 선택 → **새 프로젝트(New Project)** → 이름 `VillaGo YouTube` 등 → 만들기.
3. 왼쪽 메뉴 **API 및 서비스(APIs & Services) → 라이브러리(Library)**.
4. 검색창에 **"YouTube Data API v3"** → 클릭 → **사용(Enable)** 버튼.
   > 이걸 켜야 우리 프로그램이 유튜브에 영상을 올릴 수 있습니다. 무료입니다.

## 3단계 — OAuth 동의 화면 만들기 + **프로덕션 게시** (가장 중요)

> "동의 화면"이란: 우리 프로그램이 유튜브 채널에 접근할 때 뜨는 **"이 앱에 권한을 허용하시겠어요?"** 화면입니다.

1. **API 및 서비스 → OAuth 동의 화면(OAuth consent screen)**.
2. **User Type(사용자 유형)**: **외부(External)** 선택 → 만들기.
3. **앱 정보** 입력:
   - **앱 이름(App name)**: `빌라고 유튜브 업로더` (사용자에게 보이는 이름 — 우리 계정으로 로그인할 때 이 이름이 보입니다)
   - **사용자 지원 이메일 / 개발자 연락처**: 업무용 지메일
   - 로고·홈페이지 등은 지금 안 넣어도 됩니다(검증을 안 받으므로).
4. **범위(Scopes)**: 지금 단계에서 굳이 추가 안 해도 됩니다(코드가 연결 시 자동으로 요청합니다). 화면에 나오면 저장하고 넘어갑니다.
5. **저장하고 계속**을 눌러 마무리합니다.
6. **★★ 프로덕션으로 게시(가장 중요) ★★**
   - OAuth 동의 화면 요약 페이지에서 **게시 상태(Publishing status)** 를 봅니다.
   - **"테스트(Testing)"** 로 되어 있으면 → **"앱 게시(Publish app)"** 버튼을 눌러 **"프로덕션(In production)"** 으로 바꿉니다.
   - "확인" 류의 팝업이 뜨면 승인합니다. **검증 신청은 하지 않아도 됩니다**(우리는 자기 채널만 쓰므로).
   > **왜 이걸 꼭 하나요?** 위 "⭐ 알아둘 것 2)" 참고 — 테스트 상태로 두면 자동 로그인 열쇠가 **7일마다 죽습니다.** 반드시 "프로덕션(게시됨)" 상태여야 합니다.

## 4단계 — OAuth 클라이언트(웹) 만들기 + 주소(redirect URI) 등록

> "OAuth 클라이언트"란: 우리 프로그램의 **신분증(ID + 비밀번호)** 입니다. 두 개의 값(**클라이언트 ID**, **클라이언트 시크릿**)이 나오는데, 이걸 5단계에서 관리자 화면에 입력합니다.

1. **API 및 서비스 → 사용자 인증 정보(Credentials)**.
2. 상단 **사용자 인증 정보 만들기(Create Credentials) → OAuth 클라이언트 ID**.
3. **애플리케이션 유형(Application type)**: **웹 애플리케이션(Web application)** 선택.
4. **이름**: `VillaGo YouTube Web` 등 (내부용).
5. **승인된 리디렉션 URI(Authorized redirect URIs)** — **정확히 아래 주소를 추가**합니다 (오타·슬래시 하나까지 정확해야 함):
   ```
   https://villa-go.net/api/youtube/oauth/callback
   ```
   > (참고) 개발용으로 로컬에서도 테스트한다면 `http://localhost:3000/api/youtube/oauth/callback` 도 함께 추가할 수 있습니다. 실서비스는 위 villa-go.net 주소가 필수입니다.
6. **만들기**를 누르면 팝업에 **① 클라이언트 ID**(긴 문자열, `...apps.googleusercontent.com` 로 끝남)와 **② 클라이언트 시크릿**(짧은 비밀 문자열)이 나옵니다.
   > ⚠️ **클라이언트 시크릿은 비밀번호입니다.** 화면을 닫기 전에 두 값 모두 복사해 안전한 곳에 임시 보관하세요. (나중에 다시 볼 수도 있지만, 지금 복사해두는 게 편합니다.)

## 5단계 — villa-go 관리자에서 입력하고 **"유튜브 연결" 버튼** 누르기

> 여기서 실제 "연결(로그인 열쇠 발급)"이 이뤄집니다. 프로그램이 알아서 열쇠를 발급받아 **암호화 저장**하므로, 테오가 긴 토큰을 직접 복사할 필요가 없습니다(인스타보다 편합니다).

1. villa-go 관리자에 로그인 → **마케팅 → 유튜브 → 설정** 패널로 갑니다.
2. **클라이언트 ID** 칸에 4단계의 **①**, **클라이언트 시크릿** 칸에 4단계의 **②** 를 붙여넣고 **저장**합니다.
   - 시크릿은 저장 즉시 **암호화(AES-256, Zalo 열쇠와 같은 방식)** 되어 DB에 보관됩니다. 화면에는 다시 안 보입니다(설정됨 표시만).
3. **"유튜브 연결(Connect YouTube)"** 버튼을 누릅니다.
4. 구글 로그인 화면이 뜨면 → **1단계에서 만든 채널의 구글 계정**으로 로그인합니다.
5. **"미확인 앱" 경고 화면**이 뜰 수 있습니다 → **"고급(Advanced)" → "빌라고 유튜브 업로더(안전하지 않음)로 이동"** 을 눌러 계속합니다.
   > 이 경고는 정상입니다(우리는 구글 검증을 안 받으므로). 우리 소유 채널·우리 소유 앱이라 안전합니다.
6. 권한 허용(Allow)을 누르면 → 자동으로 관리자 화면으로 돌아오고 **"연결됨(connected)"** 표시가 뜹니다. **끝입니다.**
   > 이때 발급된 "로그인 열쇠(refresh token)"는 프로그램이 암호화해 저장하고, **화면·이메일 어디에도 노출되지 않습니다.**

**문제가 생기면:**
- 돌아온 화면에 `error=...` 가 보이면, 대개 **4단계의 redirect URI 오타** 또는 **3단계 프로덕션 게시 누락**입니다. 두 가지를 먼저 확인하세요.
- `error=redirect_uri_mismatch` → 4단계 주소를 정확히 다시 등록.
- 며칠 뒤 갑자기 연결이 풀리면 → 3단계 "프로덕션 게시"가 안 된 것(테스트 상태 7일 만료). 게시 상태를 확인하고 5단계 재연결.

## 6단계 — 유튜브 API 감사(audit) 신청 (공개 업로드 잠금 해제 · 가장 먼저 신청)

> **왜 필요한가요?** 감사(audit)를 통과하지 않은 프로젝트가 API로 올린 영상은 **강제로 "비공개(private)"** 가 됩니다(구글 정책). 감사를 통과해야 우리 프로그램이 **바로 "공개"로** 올릴 수 있습니다. **무료이고 서류는 필요 없습니다**(사용 목적을 글로 설명하는 폼).
>
> **⏰ 통과까지 수 주가 걸릴 수 있으니, 위 단계와 별개로 가능한 한 빨리 신청만 해두세요.**

1. 구글에 **"YouTube API Services - Audit and Quota Extension Form"** 을 검색하거나, YouTube API 문서의 *Quota and Compliance Audits* 페이지에 있는 신청 링크로 들어갑니다.
2. 폼에 프로젝트 정보와 **사용 목적**을 적습니다. 우리 경우 예시:
   - 용도: *"자사(빌라고) 소유의 단일 유튜브 채널에, 자사가 관리하는 푸꾸옥 빌라 홍보 쇼츠를 자동 업로드하는 내부 마케팅 도구. 제3자 계정 관리 없음."*
   - 하루 업로드 예상: 1~3건.
3. 제출하면 유튜브 API 팀에서 이메일로 연락·추가 질문을 할 수 있습니다. 답변하면 됩니다.

**감사 통과 전까지의 운영 방식 (자동화 90%는 그대로 유지):**
- 우리 프로그램은 기본값으로 영상을 **"일부 공개(unlisted)"** 로 업로드하도록 설정돼 있습니다(`YT_PRIVACY_STATUS=unlisted`).
- 즉, 감사 전에도 **자동 업로드·제목·설명·태그는 전부 자동**으로 되고, 마지막에 테오가 **유튜브 스튜디오에서 "공개"로 클릭 한 번**만 바꾸면 됩니다.
- 감사 통과 후엔 설정을 **"public"** 으로 바꾸면 클릭 없이 바로 공개됩니다.

## 7단계 — 채널 워밍업 (자동화 켜기 전, 인스타와 동일)

> 갓 만든 채널이 첫날부터 자동으로 하루 여러 건을 올리면 스팸으로 오해받을 수 있습니다. **1~2주간 하루 1건 수동 업로드**로 채널 신뢰를 쌓은 뒤 자동화를 켭니다.

- [ ] 채널 프로필 100% 완성(로고·배너·설명·카톡 링크)
- [ ] 1~2주간 **하루 1건 수동 업로드**(인스타 릴스로 만든 9:16 영상을 그대로 쇼츠로 올려도 됨 — "1회 제작 2플랫폼")
- [ ] 첫 구독·조회 반응 확인
- [ ] 2주 경과 후 개발팀에 알림 → 관리자 화면에서 **킬스위치 해제**(`YT_AUTOPOST_PAUSED` OFF) + 일 업로드 상한 설정

## 8단계 — 보안 주의사항 (꼭 지키기)

1. **클라이언트 시크릿·리프레시 토큰은 비밀번호입니다.** 카톡·이메일 평문 공유 금지. 원칙적으로 **관리자 화면 입력 → 암호화 저장**만 사용하고, 사람이 들고 다니는 사본을 남기지 마세요.
2. **구글 계정 2단계 인증(2FA)** 을 켜세요. 채널·GCP·API 열쇠를 모두 쥔 계정이라 특히 중요합니다.
3. **개인 계정과 분리** — 채널·지메일 모두 업무 전용.
4. **연결 해제/재발급**: 유출이 의심되면 GCP → 사용자 인증 정보에서 **클라이언트 시크릿을 재발급**하거나, 구글 계정 → 보안 → "내 계정에 액세스한 앱"에서 연결을 끊었다가 5단계로 다시 연결하면 됩니다.
