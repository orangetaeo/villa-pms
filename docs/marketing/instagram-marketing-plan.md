# 인스타그램 마케팅 기획 — 한국인 대상 푸꾸옥 빌라 노출 (v1, 2026-07-16)

> 목표: 등록된 빌라 사진·정보를 활용해 **매일 3건 자동 포스팅** → 한국인 여행객 노출 → **카카오톡 채널 1:1 문의** 유입.
> 원칙: 사업 핵심 원칙(재고 비공개·마진 비공개)과 충돌하지 않는 "쇼케이스형" 노출. 전체 공실 현황·원가·마진은 절대 노출 금지.

---

## 0. 전체 그림 (퍼널)

```
인스타 피드/릴스 (한국 감성 사진 + 오버레이 텍스트)
  → 프로필 링크(link-in-bio) 또는 DM
    → DM 자동응답으로 카카오톡 채널 유도 (pf.kakao.com/_mVAfX)
      → 카카오 1:1 상담 → 제안 링크(/p/[token]) 발송 → 가예약
```

- 인스타 캡션에는 **클릭 가능한 링크를 넣을 수 없다** (인스타 정책). 유일한 클릭 링크는 프로필 1곳 + DM 버튼.
- 따라서 모든 포스팅 CTA는 "프로필 링크 클릭" 또는 "DM 주세요" 2종으로 통일하고, DM은 자동응답으로 카카오 채널을 안내한다.

## 1. Phase 0 — 계정 준비 (테오 수동 작업, API로 불가)

인스타그램 **계정 생성은 API로 불가능** — 반드시 수동으로 만들어야 한다.

### 1-1. 계정 생성 절차
1. 인스타그램 앱에서 신규 계정 생성 (추천 핸들: `@villago.phuquoc`, `@villago_phuquoc` 등 — villa-go.net 브랜드 정합)
   - 가입 이메일: 비즈니스용 이메일 사용 (개인 계정과 분리)
2. **프로페셔널 계정(비즈니스)으로 전환**: 설정 → 계정 유형 → 프로페셔널 → 비즈니스 선택
   - API 사용의 필수 전제. 개인 계정은 API 발행 불가.
3. **Facebook 페이지 생성 + 연결** (권장)
   - 2024년 이후 "Instagram API with Instagram Login"은 FB 페이지 없이도 가능하지만, 광고 집행·Meta Business Suite 통합 관리를 위해 페이지 연결 권장.
4. 프로필 세팅:
   - 프로필 사진: villa-go 로고 (villa-go-logo-640.png 재사용)
   - 소개: "푸꾸옥 프라이빗 풀빌라 | 한국어 상담 | 예약·견적은 카톡 👇"
   - **링크: 카카오톡 채널** (pf.kakao.com/_mVAfX) — 또는 링크트리 스타일로 villa-go.net에 /link 랜딩 페이지를 만들어 [카톡 상담 | 빌라 보기 | Zalo] 3버튼 제공 (권장)
5. **Meta 개발자 앱 생성** (developers.facebook.com)
   - 앱 유형: Business (개인 Facebook 계정으로 생성 가능 — 서류 불필요)
   - 제품 추가: Instagram → Instagram API 설정
   - 필요 권한: `instagram_business_basic`, `instagram_business_content_publish`(발행), `instagram_business_manage_messages`(DM), `instagram_business_manage_comments`(댓글)
   - **운영 방침(확정 2026-07-16): 사업자 서류가 없으므로 App Review·비즈니스 인증(Business Verification)은 배제한다.** 자기 소유 계정만 운영하는 구조에서는 **Standard Access(앱에 연결된 자기 계정)만으로 발행·DM 웹훅 모두 동작** — 최신 콘솔(Instagram API with Instagram Login)에서는 테스터 초대 없이 **앱 대시보드 "Add an Instagram account"로 계정 연결 → "Generate token" 버튼으로 60일 장기 토큰까지 즉시 발급**된다 (INTEG 실조사 2026-07-16, 상세=instagram-account-setup.md).
   - Standard Access 한계(알아둘 것): 앱에 역할이 없는 **타인 계정**은 다룰 수 없음(우리는 해당 없음). 향후 사업자등록 후 인증을 추가하면 제한 해제·광고 집행 확장 가능 — 그때 가서 하면 되고 지금 블로커 아님.
   - ※ 용어 구분: 2번의 "프로페셔널(비즈니스) 계정 전환"은 인스타 앱 내 무료 전환으로 **서류 불필요** — 사업자 인증과 별개이며 API 사용의 필수 전제라 유지한다.
6. 장기 액세스 토큰 발급 → AppSetting에 암호화 저장 (Zalo creds와 동일한 AES-256-GCM 패턴 재사용)

### 1-2. 계정 워밍업 (중요 — 자동화 전 2주)
- **새 계정에서 즉시 API 자동 포스팅을 돌리면 스팸 계정으로 플래그될 위험**이 있다.
- 첫 1~2주는 수동 포스팅(하루 1~2개) + 팔로우/좋아요 등 정상 활동으로 계정 신뢰도를 쌓은 뒤 자동화 전환.
- 이 기간에 콘텐츠 템플릿 A/B를 수동으로 시험해 반응 좋은 스타일을 확정하는 용도로 활용.

## 2. 콘텐츠 전략 (한국인 감성)

### 2-1. 포스팅 유형 믹스 (일 3건)
| 슬롯 | 시간(KST) | 유형 | 내용 |
|---|---|---|---|
| ① 아침 | 07:30 | 빌라 쇼케이스 (캐러셀) | 빌라 1개 — 대표컷+수영장+침실+거실 4~7장, 오버레이 텍스트 |
| ② 점심 | 12:30 | 감성/정보 컷 | 푸꾸옥 여행 정보·뷰 컷·부가서비스(BBQ/마사지/티켓) 어필 |
| ③ 저녁 | 20:00 | 빌라 쇼케이스 or 릴스 | 저녁·야경·수영장 라이트업 컷 — 저녁 8~10시가 한국인 인스타 골든타임 |

- 캐러셀(여러 장)이 단일 이미지보다 체류시간·도달이 좋음 → 기본 포맷은 **캐러셀**.
- 릴스(사진 슬라이드쇼 영상)는 도달이 가장 좋지만, **API 발행 시 인스타 내장 음악 사용 불가**(라이선스) — 무음 or 자체 BGM 필요. Phase 2에서 도입.

### 2-2. 오버레이 텍스트 스타일 (경쟁사 벤치마킹 기반)
한국 여행 인스타 계정(풀빌라·동남아 여행사 계정)의 공통 패턴:
1. **1장째(커버)**: 상단 or 중앙에 감성 헤드라인 — 큰 세리프/명조 계열 + 흰색/크림 텍스트 + 은은한 그림자
   - 예: "푸꾸옥에서 눈 뜨자마자 수영장", "우리 가족만 쓰는 프라이빗 풀빌라"
2. **하단 정보 바**: 침실 수 · 최대 인원 · 해변 거리 — 아이콘+짧은 텍스트
   - 예: "🛏 침실 3 · 👨‍👩‍👧 최대 8인 · 🏖 해변 도보 5분"
3. **가격 뱃지**(선택): "1박 XX만원~" 시작가 표기 — 오른쪽 상단 작은 뱃지
4. **마지막 장(CTA 카드)**: 브랜드 컬러 배경 + "예약·견적 문의는 프로필 링크 → 카카오톡" + 로고

> 벤치마킹 태스크: 실행 전 다낭/나트랑/푸꾸옥 풀빌라 한국 계정 5~10곳 캡처·분석해 폰트·색·레이아웃 레퍼런스 보드 확정 (별도 태스크, Playwright or 수동).
> 폰트: Pretendard(고딕) + 마루부리/나눔명조(감성 헤드라인) 조합 권장 — 상업적 무료.

### 2-3-1. 카피라이터(COPY) 체계 — 카피 품질·해시태그 SEO 전담 (확정 2026-07-16)

캡션·오버레이 문구·해시태그가 성패를 가르므로 "Gemini에 대충 시키기"가 아니라 **전담 카피라이터 역할**을 둔다. 2층 구조:

**① COPY 서브에이전트 (.claude/agents/copy.md — 신설, model: opus)**
- 역할: 마케팅 카피라이터 — 오버레이 헤드라인·캡션 템플릿·해시태그 전략 수립과 갱신. 코드 작성 안 함(문구·사전·프롬프트 산출).
- 산출물 관리: `docs/marketing/copy-guide.md`(톤앤매너·금칙어·CTA 문구 사전)와 `docs/marketing/hashtags.md`(해시태그 사전) — 살아있는 문서.
- 정기 태스크(월 1회 권장): 웹 검색으로 한국 여행 인스타 트렌드·해시태그 볼륨 변화·시즌 키워드(방학/명절/얼리버드) 조사 → 두 사전 갱신 → 갱신본이 런타임 프롬프트에 자동 반영.

**② 런타임 캡션 생성 (Gemini + 카피 가이드 주입)**
- 콘텐츠 생성 cron이 Gemini 호출 시 시스템 프롬프트에 카피라이터 페르소나 + copy-guide 요약 + hashtags 사전을 주입 — 매 포스트가 사람 카피라이터 톤을 유지.
- 해시태그 조합 규칙(코드 레벨): 대형 3~4 + 중형 5~7 + 틈새 4~6 + 시즌 태그(사전의 활성 시즌 섹션) 자동 배합, 매 포스트 완전 동일 세트 반복 금지(스팸 시그널) — 사전 내 무작위 로테이션.
- 금칙어 가드(코드 레벨): 원가·마진 추정 표현, 확정 가격 박제, 과장 광고 표현(최저가 보장 등) 필터 — 승인 큐 전에 자동 검사해 위반 시 플래그 표시.

### 2-4. 부가서비스 어필 + 가격 노출 정책 ⚠
- 대상: BBQ·마사지·티켓(빈원더스/사파리/케이블카)·가이드·차량·조식.
- **가격 정책 (마진 비공개 원칙과의 조화):**
  - 티켓처럼 **공시가가 대중에 노출된 품목**: "공식가 그대로" 또는 "공식가보다 저렴" 문구까지만. **구체 판매가를 이미지에 박제하지 않는다** — 가격은 변동·협상 여지가 있고, 박제된 숫자는 나중에 발목 잡는다.
  - 마사지·BBQ 등: "빌라로 찾아오는 출장 마사지", "장보기부터 세팅까지" 등 **편의성 중심** 어필. 가격은 "카톡 문의 시 견적" 통일.
  - 절대 금지: 원가·마진 추정 가능한 표현("업체가 X에 주는데 우리는…"), 공급자 원가 언급.
- 포스팅 소재: ServiceCatalog의 티켓 7종(빈원더스·사파리·케이블카 등) + 부가서비스 사진(업체 제공분 저작권 확인 필요).

### 2-5. 사진 사용 동의 — 해결됨 ✅
- 공급자 사진의 마케팅 사용 동의는 **이미 공급자로부터 확보 완료** (테오 확인, 2026-07-16) — 선행 과제 아님.
- 유일한 잔여 주의: 게스트가 찍힌 사진은 절대 사용 금지 (초상권) — 콘텐츠 생성기에서 사람 얼굴 포함 컷은 후보에서 제외(승인 단계에서 육안 확인).

## 3. 기술 아키텍처

### 3-1. 사용 API — Instagram Graph API (Content Publishing + Messaging)
| 기능 | API | 비고 |
|---|---|---|
| 피드/캐러셀 발행 | `POST /{ig-user-id}/media` (컨테이너 생성) → `/media_publish` | 이미지는 **공개 URL 필수** (R2 public 또는 presigned), **JPEG만**, 8MB 이하 |
| 캐러셀 | 자식 컨테이너 N개(최대 10장) → 부모 CAROUSEL 컨테이너 → publish | |
| 릴스 | media_type=REELS, 동영상 URL | Phase 2 |
| 발행 한도 | **24시간당 API 발행 100건** (계정당) | 일 3건이므로 여유 충분 |
| DM 수신 | Webhooks (`messages` 필드 구독) | HTTPS 콜백 → 우리 서버 `/api/webhooks/instagram` |
| DM 발신(답장) | `POST /{ig-user-id}/messages` | **24시간 응답 창** — 유저 마지막 메시지 후 24h 내만 발신 가능 (human_agent 태그로 7일 확장) |
| 댓글 관리 | `/comments` | 댓글에도 자동 안내 답글 가능 (Phase 2) |
| 인사이트 | `/insights` (도달·저장·프로필 방문) | Phase 2 성과 대시보드 |

**⚠ API 최대 제약 — 반드시 알아야 할 것:**
1. **발행된 포스트의 수정·삭제는 API로 불가능.** 캡션 수정도 안 된다. 수정·삭제는 인스타그램 앱에서 수동으로만 가능. → 관리자 페이지의 "수정" 기능은 **발행 전 큐 단계에서만** 가능하고, 발행 후에는 조회 + 인스타 앱 딥링크 제공까지가 한계. **이것이 발행 전 승인 큐가 필수인 이유.**
2. 캡션에 링크 넣어도 클릭 불가 — CTA는 프로필 링크/DM으로만.
3. 웹훅은 앱이 Live 모드여야 수신 — 비즈니스 인증 서류(사업자등록) 준비 필요.

### 3-2. 이미지 합성 파이프라인 — 디자인은 Stitch, 런타임 렌더는 satori (확정 2026-07-16)

**역할 분리: Stitch = 디자인 타임(템플릿 시안), satori = 런타임(매일 자동 렌더).**
Stitch는 사람이 쓰는 디자인 도구라 매일 3건 자동 생성 루프에 넣을 수 없다. 대신 프로젝트 표준(Stitch-first 규칙)대로 **템플릿 디자인을 Stitch에서 생성**하고, 그 HTML/Tailwind export를 satori용 템플릿 코드로 1회 변환해 두면 이후 발행은 전자동이 된다.

```
[디자인 타임 — 1회 + 시즌별 갱신]
  DESIGN: 경쟁사 벤치마킹 → Stitch 프롬프트로 오버레이 템플릿 시안 생성
    (1080×1350 아트보드, 커버/정보바/가격뱃지/CTA카드 4종 × 변형)
  → design/stitch/instagram-templates/ 에 export 저장 (기존 규칙 동일)
  → BE: export HTML/Tailwind → satori 템플릿(JSX/satori-html)으로 변환
    ※ satori는 CSS 부분집합만 지원(flexbox 중심, grid·backdrop-filter 등 불가)
      — Stitch 프롬프트에 "flexbox만, 단순 레이어" 제약을 명시해 변환 손실 최소화

[런타임 — 매일 cron 자동]
  VillaPhoto(R2 원본)
  → sharp 리사이즈 (1080×1350 4:5 세로 — 피드 점유 최대)
  → satori 템플릿에 데이터(빌라명·침실·해변거리·카피) 주입 → SVG → 텍스트 레이어
  → sharp composite 합성 → JPEG 변환 (인스타는 JPEG만)
  → R2 `instagram-renders/` 버킷 업로드 (공개 URL) → 발행 시 이 URL 사용
```
- 폰트: Pretendard(정보) + 마루부리/나눔명조(감성 헤드라인) — 상업적 무료, satori에 폰트 파일 번들.
- 템플릿은 JSON 파라미터(문구·색·배치)만 갈아끼우는 구조 — 새 시안이 나오면 Stitch→변환만 반복.
- admin 콘텐츠 큐의 미리보기 = 실제 발행될 렌더 결과물(동일 렌더러) — 보이는 그대로 발행.
- 기존 sharp 이미지 압축 모듈(글로벌 템플릿 utility-modules) 재사용.

### 3-3. DB 모델 (additive raw SQL — 스키마 규약 준수)
```prisma
model InstagramPost {
  id           String   @id @default(cuid())
  villaId      String?  // 빌라 쇼케이스면 연결, 정보/서비스 포스트면 null
  kind         IgPostKind   // VILLA_SHOWCASE | SERVICE | INFO | REELS
  status       IgPostStatus // DRAFT | PENDING_APPROVAL | QUEUED | PUBLISHING | PUBLISHED | FAILED | CANCELLED
  scheduledAt  DateTime     // 발행 예정 슬롯
  caption      String       // 승인된 최종 캡션
  mediaJson    Json         // [{srcPhotoId, renderedUrl, templateId, overlayText}]
  igMediaId    String?      // 발행 후 인스타 media id
  igPermalink  String?      // 발행 후 포스트 URL
  publishedAt  DateTime?
  failReason   String?
  createdBy    String?
  createdAt    DateTime @default(now())
}

model InstagramMessage {   // DM 인박스
  id           String   @id @default(cuid())
  igThreadId   String   // 대화 스레드
  igSenderId   String   // 상대 IGSID
  senderName   String?
  direction    MsgDirection // IN | OUT
  text         String?
  attachments  Json?
  igMessageId  String   @unique
  receivedAt   DateTime
  readByAdmin  Boolean  @default(false)
}
```
- 토큰·설정: AppSetting에 `IG_ACCESS_TOKEN`(암호화), `IG_USER_ID`, `IG_AUTOPOST_PAUSED`(킬스위치 — Zalo 킬스위치 패턴 재사용).
- 모든 발행/실패는 AuditLog 기록 (글로벌 절대 규칙).

### 3-4. 자동화 흐름 (cron)
```
[매일 새벽 cron] 콘텐츠 생성기
  → 포스팅 이력 없는(또는 가장 오래된) 빌라 로테이션 선정
  → 사진 선별(공간 다양성: 외관→수영장→거실→침실) + 템플릿 렌더 + Gemini 캡션
  → InstagramPost(PENDING_APPROVAL, 슬롯 3개 배정) 생성
  → 운영자에게 Zalo 알림 "오늘 인스타 초안 3건 승인 대기"

[운영자] admin 페이지에서 미리보기 → 문구 수정 → 승인 (또는 반려)

[슬롯별 cron 07:30/12:30/20:00 KST] 발행기
  → QUEUED 중 시간 도래분 발행 (컨테이너 생성→상태 폴링→publish)
  → 성공: PUBLISHED + permalink 저장 / 실패: FAILED + Zalo 경보
```
- Railway cron 등록은 기존 GraphQL API 방식 재사용 (db-backup에서 검증됨).
- **발행 고아 자동 회수(2026-07-23)**: 발행 중 프로세스가 끊겨 PUBLISHING에 갇힌 행은 45분 초과 시 자동으로 FAILED + 운영자 경보가 된다(`lib/marketing/reap-stale-publishing.ts`, 발행 cron 2종 + `youtube-edit-jobs`(*/5분)에서 실행). **자동 재발행은 하지 않는다** — 이미 올라갔을 수 있으므로 계정에서 실제 발행 여부를 확인한 뒤 사람이 재발행한다.
- 승인 편의: "3건 일괄 승인" 버튼 + 승인 없이 24h 지나면 스킵(무단 발행 금지). 운영이 익숙해지면 "자동 승인 모드" 옵션 추가 가능.

### 3-5. 관리자 페이지 (admin, ko/다크)
```
/marketing/instagram
  ├─ 콘텐츠 큐    : 오늘/예정 포스트 카드 (렌더 이미지 미리보기 + 캡션 편집 + 승인/반려/시간 변경)
  ├─ 발행 이력    : PUBLISHED 목록 + permalink + (Phase 2) 좋아요·도달 인사이트
  └─ DM 인박스   : 스레드 목록 + 대화 뷰 + 답장(24h 창 표시) + "카카오 안내 보내기" 원클릭
```
- DM 인박스는 기존 /messages(Zalo 인박스) UI 패턴 재사용 — 80개 limit + 비정규화 교훈 적용.
- 새 admin NS → ADMIN_CLIENT_NAMESPACES 추가 + ko/vi 동시 (기존 함정 메모 준수).

### 3-6. DM → 카카오톡 채널 유도
1. **DM 자동응답 (수신 웹훅 트리거)**: 첫 수신 시 즉시 자동 응답
   - "안녕하세요, 빌라고 푸꾸옥입니다 🌴 빠른 상담은 카카오톡 채널에서 도와드려요 → pf.kakao.com/_mVAfX (프로필 링크에서도 바로 연결됩니다)"
   - DM에서는 링크가 클릭 가능하므로 카카오 링크 직접 발송 OK.
2. **Ice Breakers** (DM 시작 시 선택지 버튼): "예약 문의" / "가격 문의" / "부가서비스 문의" 3버튼 → 각각 맞춤 자동응답 + 카카오 유도.
3. 자동응답 후에도 대화는 admin DM 인박스에 쌓임 — 운영자가 직접 이어받기 가능 (24h 창 내).
