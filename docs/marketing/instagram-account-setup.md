# 인스타그램 계정·API 셋업 가이드 — 테오용 (v1, 2026-07-16)

> **누가 읽나요?** 개발자가 아닌 운영자(테오)가 스크린샷 없이도 혼자 따라 할 수 있게 만든 안내서입니다.
> **무엇을 하나요?** 빌라고(villa-go) 전용 인스타그램 계정을 만들고, 우리 프로그램이 **매일 자동으로 사진을 올릴 수 있도록** 연결하는 "열쇠(액세스 토큰)"를 발급받는 것까지가 목표입니다.
> **얼마나 걸리나요?** 계정 만들기 + 연결까지 실제 작업은 30~40분. 그 뒤 **2주 워밍업**(6단계)을 거친 후에 자동 포스팅을 켭니다.

---

## ⭐ 시작 전에 꼭 알아둘 3가지

1. **사업자등록증·서류가 전혀 필요 없습니다.**
   Meta(=페이스북/인스타그램 회사)의 "비즈니스 인증(Business Verification)"이나 "앱 심사(App Review)"라는 복잡한 승인 절차는 **우리는 하지 않습니다.** 이 절차들은 "**남의 인스타 계정**을 대신 관리하는 서비스"에만 필요합니다. 우리는 **우리가 직접 만든 우리 소유 계정 하나만** 운영하므로, 서류·심사 없이 "개발 모드(Development Mode) + 내 계정을 앱에 등록"만으로 자동 포스팅과 DM(쪽지) 자동응답이 **전부 정상 작동**합니다.

2. **인스타 계정은 사람이 손으로 만들어야 합니다.** 프로그램(자동)으로는 계정을 못 만듭니다. 그래서 이 문서의 1~2단계는 테오가 휴대폰으로 직접 하는 작업입니다.

3. **용어 두 개만 미리 구분하세요.** 비슷해서 헷갈립니다.
   | 용어 | 뜻 | 서류 필요? |
   |---|---|---|
   | **프로페셔널(비즈니스) 계정 전환** | 인스타 앱 안에서 무료로 계정 종류만 바꾸는 것. API를 쓰려면 반드시 필요 | ❌ 불필요 (2분, 무료) |
   | **비즈니스 인증 / 앱 심사** | Meta가 사업체 서류를 검증하는 별개 절차 | 우리는 **안 함** |

## 1단계 — 인스타그램 계정 만들기 + 프로필 세팅

### 1-1. 계정 생성
1. 휴대폰에 **인스타그램(Instagram) 앱**을 설치하고 엽니다. (이미 개인 계정으로 로그인돼 있다면 로그아웃하거나, 앱 하단 프로필 → 계정 추가로 **새 계정**을 만듭니다. 개인 계정과 반드시 분리하세요 → 7단계 참고)
2. **가입 이메일**은 개인 이메일이 아니라 **업무용 이메일**을 쓰세요. (예: 빌라고 전용 지메일. 나중에 여러 사람이 관리할 수 있고, 개인 프라이버시와 섞이지 않습니다.)
3. **핸들(사용자 이름, @ 뒤에 오는 아이디)** 후보 — 위에서부터 되는 것으로:
   - `villago.phuquoc`
   - `villago_phuquoc`
   - `villago.pq`
   - `villagophuquoc`
   > 핸들은 나중에 바꿀 수 있지만, 팔로워가 기억하는 주소이므로 **처음에 브랜드(villa-go)와 맞는 걸로** 정하는 게 좋습니다.
4. **표시 이름(Name)**은 검색이 잘 되도록 한글·지역을 넣으세요. 예: `빌라고 푸꾸옥 풀빌라` 또는 `Villa-Go 푸꾸옥`.

### 1-2. 프로필 채우기
1. **프로필 사진**: 빌라고 로고를 올립니다. (파일: `villa-go-logo-640.png` — 테오에게 전달된 로고 이미지. 정사각형으로 잘 보이게 가운데 정렬)
2. **소개글(Bio)** 예시 — 그대로 쓰거나 골라서:
   ```
   🌴 푸꾸옥 프라이빗 풀빌라 전문
   🇰🇷 한국어 상담 · 실시간 예약·견적
   👨‍👩‍👧 가족·커플·단체 맞춤 빌라
   👇 예약·문의는 카카오톡 채널로
   ```
   > 인스타 소개글은 150자 제한이 있으니 이모지 포함 짧게. 핵심은 "**한국어 상담**"과 "**아래 링크(카톡)**"입니다.
3. **프로필 링크(가장 중요 — 인스타에서 클릭되는 유일한 링크입니다)**:
   - 소개글 아래 **웹사이트/링크** 칸에 카카오톡 채널 주소를 넣습니다:
     ```
     https://pf.kakao.com/_mVAfX
     ```
   - (권장, 나중에) 링크를 여러 개 걸고 싶으면 villa-go.net에 `/link` 랜딩 페이지를 만들어 [카톡 상담 · 빌라 보기 · Zalo] 3버튼을 제공하고, 그 주소를 프로필 링크로 넣습니다. → 지금은 카톡 링크 하나로 시작해도 충분합니다.

## 2단계 — 프로페셔널(비즈니스) 계정으로 전환 (무료·서류 불필요)

> API로 자동 포스팅을 하려면 계정이 **프로페셔널(비즈니스)** 종류여야 합니다. 개인 계정으로는 안 됩니다. **2분이면 끝나고, 돈·서류·승인 대기 전혀 없습니다.**

**휴대폰 앱 기준 (2026년 7월 현재 메뉴):**
1. 인스타 앱에서 내 **프로필**로 갑니다 (오른쪽 아래 프로필 사진 탭).
2. 오른쪽 위 **≡ (메뉴)** → **설정 및 활동(Settings and activity)**.
3. 아래로 내려 **비즈니스용(For professionals)** 항목의 **계정 유형 및 도구(Account type and tools)** 를 누릅니다.
4. **프로페셔널 계정으로 전환(Switch to professional account)** 을 선택.
5. **카테고리**를 고릅니다. (예: `여행사`, `호텔 및 숙박`, `부동산` 중 가까운 것 — 나중에 변경 가능)
6. 계정 종류에서 **비즈니스(Business)** 를 선택합니다.
   > "크리에이터(Creator)"도 API가 되지만, 우리처럼 **업체**는 **비즈니스**가 맞습니다.
7. 페이스북 페이지 연결 화면이 나오면 **"건너뛰기 / 나중에(Skip)"** 를 눌러도 됩니다.
   > **⭐ 페이스북 페이지는 필수가 아닙니다.** 2024년부터 우리가 쓸 최신 연결 방식("Instagram API with Instagram Login")은 **페이스북 페이지 없이도** 자동 포스팅·DM이 됩니다. 페이지를 만들려면 나중에 광고를 돌릴 때 붙여도 되고, 지금은 넘어가세요.
8. 완료(Done)를 누릅니다.

**확인 포인트:** 전환하면 계정이 **공개(Public)** 로 바뀝니다 (마케팅 계정이므로 정상입니다). 프로필에 "프로페셔널 대시보드(인사이트)" 메뉴가 생겼으면 성공입니다.

## 3단계 — Meta 개발자 등록 + 앱 만들기 (개인 페이스북 계정으로 가능)

> 여기부터는 **컴퓨터(PC) 웹브라우저**로 하는 게 편합니다. 개발자 등록은 무료이고, **테오의 개인 페이스북 계정**으로 하면 됩니다. (사업체 계정·서류 불필요)

### 3-1. 개발자 등록
1. 브라우저에서 **developers.facebook.com** 접속.
2. 오른쪽 위 **로그인(Log In)** → 테오의 개인 페이스북 계정으로 로그인.
   > 페이스북 계정이 없으면 하나 만들어야 합니다(무료). 이 계정은 "앱 관리자" 역할만 하며, 인스타 계정과는 별개입니다.
3. 처음이면 **개발자로 등록(Register / Get Started)** 안내가 뜹니다 → 동의하고 진행. 휴대폰 번호 인증(문자)과 역할 선택("개발자")을 요구할 수 있으니 따라갑니다.

### 3-2. 앱(App) 만들기
1. 상단 메뉴 **내 앱(My Apps)** → **앱 만들기(Create App)**.
2. **앱 이름(App name)**: 아무거나 내부용. 예: `VillaGo Instagram`. (사용자에게 안 보이는 내부 이름입니다. 이름에 "Instagram/IG/insta" 같은 상표 단어를 넣으면 거부될 수 있으니 `VillaGo Marketing` 처럼 쓰는 게 안전합니다.)
3. **앱 유형(Use case / App type)**: **비즈니스(Business)** 를 선택합니다.
   > 2026년 화면에서는 "무엇을 만들고 싶나요?" 식의 **사용 사례(Use case)** 를 먼저 고르라고 나올 수 있습니다. 그럴 땐 **"Instagram" 관련(예: Manage messaging and content on Instagram)** 을 고르거나, 유형 선택 화면에서 **Business** 를 고르면 됩니다.
4. 연락 이메일 등을 확인하고 **앱 만들기(Create app)** 를 눌러 완성합니다. (비밀번호 재확인 요구될 수 있음)

> 이 시점의 앱은 자동으로 **"개발 모드(Development Mode)"** 이며, 자동으로 **"표준 액세스(Standard Access)"** 권한을 가집니다. **우리에겐 이 상태가 최종 상태입니다** — 여기서 심사를 더 받을 필요가 없습니다(1페이지 상단 설명 참고).

## 4단계 — Instagram 제품 추가 + 우리 인스타 계정 연결 (핵심)

> 이 단계가 "우리 프로그램 ↔ 우리 인스타 계정"을 잇는 열쇠 발급 준비입니다.

1. 방금 만든 앱의 **대시보드(Dashboard)** 왼쪽 메뉴에서 제품을 추가합니다. **제품 추가(Add product)** 목록 또는 상단에서 **Instagram** 을 찾아 **설정(Set up)** 을 누릅니다.
2. Instagram 안에 두 가지 설정 방식이 보입니다. **반드시 아래 것을 고르세요:**
   - ✅ **"API 설정 · Instagram 로그인(API setup with Instagram login)"** ← **이것을 선택**
   - ❌ "API 설정 · Facebook 로그인(API setup with Facebook login)" ← 이건 페이스북 페이지가 필요한 옛 방식. 고르지 마세요.
   > **왜?** "Instagram 로그인" 방식이 **페이스북 페이지 없이** 인스타 계정만으로 바로 연결되고, 대시보드에서 **버튼 한 번으로 60일 토큰**까지 뽑아주는 가장 쉬운 최신 경로이기 때문입니다.
3. "API setup with Instagram login" 화면에서 **1. 인스타그램 계정 추가(Add an Instagram account / Generate access token 근처의 "Add account" 버튼)** 를 누릅니다.
4. 팝업이 뜨면 **우리가 1~2단계에서 만든 비즈니스 인스타 계정**의 아이디·비밀번호로 로그인하고, 권한 허용(Allow)을 눌러 앱과 연결합니다.
   > 이 "계정 추가 + 로그인 + 허용" 과정이 **곧 "내 계정을 이 앱에 등록"** 하는 것입니다. 이렇게 등록된 우리 소유 계정은 개발 모드에서도 **자동 포스팅·DM이 정상 작동**합니다. (심사 불필요)

> **📌 기획서와 달라진 점**: 예전(구 방식)에는 **Roles(역할) → Instagram Testers → 테스터 초대 → 인스타 웹에서 초대 수락** 이라는 여러 단계를 거쳤습니다. **최신 "Instagram 로그인" 방식에서는 이 테스터 초대 절차가 없어지고**, 위 3~4번처럼 화면에서 **"계정 추가(Add account)" 후 로그인** 한 번으로 대체됩니다. 만약 화면에 "테스터 초대" 방식만 보인다면(옛 앱), 아래 참고 절차를 쓰세요.
>
> <details><summary>참고: 옛 "테스터 초대" 방식(화면에 이것만 보일 때)</summary>
>
> 1. 앱 대시보드 → **App roles(앱 역할) → Roles(역할)**.
> 2. 아래 **Instagram testers** 섹션 → **Add Instagram testers** → 우리 인스타 **핸들** 입력 후 초대.
> 3. 그다음 **인스타그램 웹(instagram.com)** 에 그 계정으로 로그인 → **설정 → 앱 및 웹사이트(Apps and websites) → 테스터 초대(Tester invites)** 에서 **수락(Accept)**.
> 4. 앱의 Roles 화면에서 상태가 **Active(활성)** 로 바뀌면 완료.
> </details>

## 5단계 — 장기(60일) 액세스 토큰 발급 + 관리자에게 전달

> **액세스 토큰**이란: 우리 프로그램이 "이 인스타 계정에 글을 올려도 된다"는 것을 증명하는 **긴 비밀번호 같은 문자열**입니다. 이걸 우리 관리자 화면에 넣어두면 매일 자동 포스팅이 됩니다.

### 5-1. 토큰 뽑기
1. 4단계의 **"API setup with Instagram login"** 화면에서, 연결된 우리 인스타 계정 옆의 **토큰 생성(Generate token / Generate access token)** 버튼을 누릅니다.
2. 인스타 로그인/허용을 한 번 더 확인할 수 있습니다 → 허용.
3. **긴 문자열(토큰)이 화면에 나타납니다.** 여기서 나오는 토큰은 **처음부터 "장기(60일) 토큰"** 입니다.
   > ⚠️ **이 화면을 닫으면 토큰을 다시 볼 수 없습니다.** 반드시 지금 **전체를 복사**해서 안전한 곳에 임시 저장하세요.
4. 같은 화면 근처에 표시되는 **인스타그램 사용자 ID(Instagram User ID / IG User ID)** — 숫자로 된 값 — 도 함께 복사해 둡니다. (예: `178414... ` 같은 긴 숫자. 우리 프로그램이 "어느 계정에 올릴지" 식별하는 번호입니다.)

### 5-2. 60일마다 갱신되는 구조 (개발자가 자동화하지만, 원리만 알아두기)
- 이 토큰은 **60일 뒤 만료**됩니다. 대신 **만료 전에 자동으로 60일씩 연장**할 수 있습니다.
- 개발팀이 우리 프로그램 안에 **자동 갱신(하루에 한 번 토큰을 60일 더 연장)** 을 넣을 예정이라, **정상 운영 중에는 테오가 손댈 일이 없습니다.**
- 딱 하나 주의: 만약 **60일 넘게 자동 갱신이 멈춰** 있으면(예: 서버가 오래 꺼져 있었음) 토큰이 완전히 죽어서, **5-1을 다시 해서 새 토큰을 발급**해야 합니다. 이럴 때 이 문서를 다시 보면 됩니다.

### 5-3. 관리자에게 안전하게 전달하기
- 위에서 복사한 **① 액세스 토큰**과 **② 인스타 사용자 ID(IG User ID)** 두 개를 개발 담당에게 전달합니다.
- **전달 방법(중요)**: 토큰은 계정 비밀번호와 같습니다. **카카오톡·이메일 평문으로 그냥 보내지 마세요.** 아래 중 하나로:
  - (권장) 나중에 만들어질 **관리자 설정 화면**(`/marketing/instagram` 설정)에 테오가 **직접 입력** — 그러면 프로그램이 자동으로 **암호화(AES-256, Zalo 열쇠와 같은 방식)** 해서 DB에 저장합니다. 사람 손을 거의 안 거치는 가장 안전한 방법입니다.
  - (임시) 지금 당장 개발자에게 넘겨야 하면, 1회용 비밀 메모(예: 비밀번호 관리자 공유, 또는 만료되는 보안 노트)로 전달하고, 전달 후 원본 메모는 삭제합니다.
- 전달이 끝나면 테오 휴대폰·메모장에 남긴 토큰 사본은 **지웁니다.**

> **한 줄 요약**: 5단계에서 나오는 **긴 문자열(토큰)** + **숫자 ID** 두 개를 복사 → 관리자 화면에 입력(또는 개발자에게 안전하게 전달) → 끝.

## 6단계 — 계정 워밍업 체크리스트 (자동화 켜기 전 2주, 필수)

> **왜 2주를 기다리나요?** 갓 만든 계정이 **첫날부터 프로그램으로 하루 3건씩 자동 포스팅**을 하면, 인스타가 이를 **스팸봇으로 오해해서 계정을 정지(밴)** 시킬 위험이 큽니다. 사람이 직접 쓰는 정상 계정처럼 2주간 "몸을 풀어" 신뢰도를 쌓은 뒤 자동화를 켜야 안전합니다. (이 기간에 어떤 사진·문구가 반응 좋은지도 미리 테스트할 수 있어 일석이조입니다.)

**매일/기간 체크리스트:**

- [ ] **프로필 100% 완성** (1일차): 로고·소개글·카톡 링크·표시 이름까지 빈 곳 없이 채우기 (1~2단계에서 이미 대부분 완료)
- [ ] **수동 포스팅 하루 1~2건** (2주 내내, 직접 손으로 올리기):
  - 등록된 빌라 사진 중 예쁜 것 위주로, 캐러셀(여러 장)로.
  - 캡션은 기획서 `2-3 캡션 템플릿` 스타일 참고. 해시태그 10~20개.
  - **매일 같은 시간대**(예: 아침·저녁)를 지키면 더 좋습니다.
- [ ] **정상적인 "사람 활동"** (2주 내내):
  - 다른 푸꾸옥/여행/풀빌라 계정 몇 개 **팔로우**.
  - 관련 게시물에 **좋아요·진심 댓글** 몇 개(하루 소량, 몰아서 대량 금지 — 그것도 스팸 신호).
  - 스토리 1~2개 올려보기.
- [ ] **첫 팔로워·문의 반응 확인** (1주차 즈음): DM이 오면 손으로 답해보며 카톡 유도 문구를 다듬기.
- [ ] **반응 좋은 스타일 메모** (2주차): 어떤 빌라·문구·해시태그가 저장·도달이 좋았는지 기록 → 자동화 템플릿·캡션에 반영.
- [ ] **2주 경과 후**: 개발팀에 "워밍업 완료" 알림 → 관리자 화면에서 **자동 포스팅 스위치 ON**.

> 워밍업 중에는 위 5단계 토큰을 이미 발급해 둬도 됩니다. 토큰만 있고 **자동 포스팅 스위치는 꺼둔 채**로 손으로만 올리면 됩니다.

## 7단계 — 보안 주의사항 (꼭 지키기)

1. **액세스 토큰(긴 문자열)은 절대 남에게 보여주거나 공개된 곳에 올리지 마세요.**
   - 토큰은 **계정 비밀번호와 동급**입니다. 유출되면 남이 우리 계정에 마음대로 글을 올리거나 DM을 보낼 수 있습니다.
   - 카카오톡·이메일·메신저에 **평문으로 복붙 금지**. 캡처해서 공유 금지. 화면 공유 방송 중 노출 금지.
   - 원칙적으로 토큰은 **관리자 화면에 입력해 암호화 저장**하고, 사람이 들고 다니는 사본은 안 남기는 게 최선입니다.
   - 혹시 유출이 의심되면: 5단계에서 **새 토큰을 재발급**하면 옛 토큰은 무효가 됩니다. (필요 시 인스타 → 설정 → 앱 및 웹사이트에서 앱 연결을 끊었다가 다시 연결)

2. **2단계 인증(2FA)을 반드시 켜세요.**
   - 인스타 앱 → 설정 → **계정 센터(Accounts Center)** → 비밀번호 및 보안 → **2단계 인증** → 켜기.
   - 인증 방법은 **인증 앱(예: Google Authenticator)** 을 권장(문자보다 안전). 백업 코드는 안전한 곳에 보관.
   - Meta 개발자 계정(테오 개인 페이스북)에도 2단계 인증을 켜두세요 — 앱을 조작할 수 있는 계정이라 마찬가지로 중요합니다.

3. **개인 계정과 반드시 분리하세요.**
   - 이 인스타는 **업무 전용**입니다. 테오 개인 인스타(사진·친구)와 섞지 마세요.
   - 가입 이메일도 업무용으로 분리(1-1 참고). 나중에 담당자가 바뀌어도 인수인계가 쉽고, 개인 프라이버시가 보호됩니다.
   - 비밀번호는 개인 계정과 **다르게**, 비밀번호 관리자에 저장.

## 부록 — 한눈에 보는 전체 흐름 & 완료 확인표

```
[테오 수동, 30~40분]
1. 인스타 앱에서 계정 생성(@villago.phuquoc) + 프로필(로고·소개·카톡링크)
2. 프로페셔널(비즈니스) 전환   ← 무료·서류X
3. developers.facebook.com 개발자 등록 + Business 앱 생성   ← 개인 FB로
4. 앱에 Instagram 제품 추가 → "API setup with Instagram login" → 우리 계정 로그인 연결
5. "Generate token" 버튼 → 60일 토큰 + IG User ID 복사 → 관리자 화면에 입력
        ↓
[2주 워밍업]
6. 손으로 하루 1~2건 포스팅 + 정상 활동   ← 자동화 스위치는 OFF
        ↓
[개발팀]
   렌더·발행·DM 자동화 개발 → 워밍업 끝나면 스위치 ON
```

**완료 확인표 (개발팀에 넘기기 전 스스로 점검):**
- [ ] 인스타 계정이 **비즈니스(프로페셔널)** 로 전환됨 (프로필에 인사이트 메뉴 보임)
- [ ] 프로필 링크가 **카카오톡 채널(pf.kakao.com/_mVAfX)** 로 걸림
- [ ] Meta 개발자 앱이 **Business 유형**으로 만들어짐
- [ ] 앱에 **Instagram(API setup with Instagram login)** 이 추가되고 우리 계정이 연결됨
- [ ] **액세스 토큰(긴 문자열)** 과 **IG User ID(숫자)** 두 개를 확보 → 안전하게 전달/입력
- [ ] 인스타 계정 **2단계 인증 켬**, 개인 계정과 분리
