## 재생성 절차

**스크립트가 정본이다: `smoke/m1-30cut.mts`** (위 표와 같은 30컷이 하드코딩돼 있다).

```bash
npx tsx smoke/m1-30cut.mts cut      # 원본에서 30컷 재단
npx tsx smoke/m1-30cut.mts audit    # ★ 렌더 게이트와 같은 검수를 로컬에서 먼저
npx tsx smoke/m1-30cut.mts upload   # R2 업로드(약 250MB, 키는 state.json에 누적)
npx tsx smoke/m1-30cut.mts create   # 대본·메타 → YoutubeShort(PENDING) → cron이 렌더
npx tsx smoke/m1-30cut.mts audit 11,28   # 컷 번호를 주면 그 컷만(cut도 동일)
```

- 대본은 `buildNarrationScript` → `normalizeScript(raw, 30, clipKinds)` ← **clipKinds 필수**(이동 컷 흡수).
- 검수 `error`가 나오면 **렌더가 멈추고** 컷 번호가 `editError`에 담긴다 → 그 컷 시작 지점만 옮겨 재시도.
- ★ 업로드 전에 `audit`를 반드시 돌린다. 렌더에서 걸리면 cron 한 사이클(업로드+렌더 대기)을 통째로 버린다.
