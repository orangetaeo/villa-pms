# 유튜브 쇼츠 마케팅 기획 — 빌라 숏츠 자동 생성·업로드 (v1, 2026-07-16)

> 목표: ① 빌라 정보만으로 숏츠 자동 생성 ② 직접 촬영 영상을 자동 편집(텍스트·로고)해 숏츠 생성 ③ YouTube API로 자동 업로드·노출 → 한국인 여행객 유입.
> 원칙: 인스타 기획과 동일 — 재고·마진 비공개, 발행 전 승인 큐, 카피 사전(copy-guide·hashtags) 공유.

---

## 0. 전체 그림 — 인스타와의 관계

**핵심 전략: "1회 제작 → 2플랫폼 배포".** 인스타 P2에서 만든 릴스 MP4(9:16, 1080×1920)는 유튜브 쇼츠 규격과 동일하다. 같은 영상을 인스타 릴스와 유튜브 쇼츠에 동시 발행해 제작 비용을 반으로 줄인다.

```
[콘텐츠 1] 빌라 사진·정보 → (기존 릴스 파이프라인 재사용) → 9:16 MP4
[콘텐츠 2] 테오 직접 촬영 영상 → 자동 편집(9:16 정규화+로고+텍스트+인트로/아웃트로) → 9:16 MP4
                     ↓ 승인 큐 (인스타와 동일 UX)
        ┌────────────┴────────────┐
   인스타 릴스 발행(기존)      유튜브 쇼츠 업로드(신규, YouTube Data API)
                     ↓
   유튜브 노출 → 채널 프로필 링크·고정 댓글·영상 내 CTA → 카카오톡 채널 유입
```

**⚠ 유튜브 쇼츠의 링크 제약 (인스타와 다른 점):** 쇼츠의 설명·댓글 링크는 **클릭이 안 되게 막혀 있다**(스팸 방지 정책). 전환 경로는 ① 채널 프로필의 공식 링크 ② 영상 마지막 CTA 카드("카카오톡 채널에서 '빌라고' 검색") ③ 고정 댓글 텍스트(복사 가능) 3중으로 설계한다. 일반(가로) 영상의 설명 링크는 클릭 가능하므로, 반응 좋은 쇼츠는 나중에 일반 영상 버전도 고려(Phase 2).

## 1. Phase 0 — 채널·API 준비 (테오 수동, 사업자 서류 불요)

1. **구글 계정 + 유튜브 채널 생성** (브랜드 채널 권장 — 채널명 "빌라고 푸꾸옥" 류, 로고·배너=villa-go 에셋 재사용, 채널 설명에 카카오 링크)
2. **Google Cloud 프로젝트** 생성 → YouTube Data API v3 활성화 (무료)
3. **OAuth 클라이언트** 생성(웹 앱) + 동의 화면 구성 → 테오 채널 계정으로 1회 승인 → refresh token 확보 → AppSetting 암호화 저장(YT_REFRESH_TOKEN, lib/secret-crypto 재사용)
   - ⚠ **동의 화면을 "테스트" 모드로 두면 refresh token이 7일마다 만료**(2026-07 실조사 확인, 테스트 상태 전용 규칙) → **"프로덕션 게시(In production)"로 전환 필수** → 그러면 refresh token은 사실상 영구(6개월 미사용 시만 만료).
   - **★두 심사는 별개다(실조사 정정)**: ⑴ **구글 OAuth 앱 검증**(Trust&Safety) = 미확인 앱 경고 제거·100명 초과 허용용 → **우리는 안 함**(자기 채널 1개라 경고는 "고급→계속"으로 통과, 100명 제한 무의미). youtube.upload은 sensitive scope지만 프로덕션+미검증 상태로도 자기 계정 업로드 정상 동작. ⑵ **YouTube API 감사**(아래 4) = 공개 업로드 잠금 해제용 → **필요**. 이 둘을 혼동 금지.
4. **★최대 관문 — API 업로드 영상의 공개 잠금**: **감사(audit) 미통과 프로젝트**(2020-07-28 이후 생성)가 업로드한 영상은 **강제 비공개(private)**(2026-07 현행 확인). ※위 3-⑴의 OAuth 앱 검증과 별개 — 잠금 해제는 아래 유튜브 감사로만. 해결책:
   - **"YouTube API Services - Audit and Quota Extension Form"** 제출(무료, 사용 목적 설명 — 사업자 서류 불요) → 유튜브 API 팀이 이메일로 후속 연락. **소요 기간은 공식 미고지(커뮤니티 사례 수 주)** → 착수와 별개로 최우선 신청.
   - 감사 통과 전 운영: **unlisted로 자동 업로드(YT_PRIVACY_STATUS 기본값) → 유튜브 스튜디오에서 수동 "공개" 전환**(클릭 1회) — 자동화 90%는 유지됨.
5. **쿼터(2026-07 실조사 — 기획 v1 정정)**: **2025-12-04 정책 변경으로 영상 업로드 1건 비용이 약 1,600 → 약 100 유닛으로 인하**. 기본 일 10,000 유닛 기준 **일 약 100건 업로드 여유**(구 "일 6건 상한"은 폐기). 우리 목표(일 1~3건)는 전혀 문제없음. 조회·상태 확인은 대부분 1유닛/호출. ※코드 기본 상한 `YT_DAILY_UPLOAD_CAP=6`은 안전판일 뿐 실제 API 한도는 아님 — 필요 시 상향 가능.
6. **채널 워밍업**: 인스타와 동일 — 신규 채널 즉시 대량 자동 업로드 금지, 첫 1~2주 일 1건 수동 업로드로 시작.

## 2. 콘텐츠 1 — 빌라 정보 기반 숏츠 (기존 자산 재사용)

- **소스**: 기존 릴스 파이프라인(lib/instagram/reels.ts) 산출물 그대로 — 빌라 사진 4~7장 슬라이드쇼(커버 헤드라인+엔딩 CTA 오버레이).
- **유튜브 최적화 추가분**:
  - 엔딩 CTA 카드를 유튜브 버전으로 분기: "카카오톡 채널 → '빌라고' 검색" (링크 클릭 불가 대응)
  - **제목·설명·태그 자동 생성**: Gemini + copy-guide/hashtags 사전 재사용. 제목 예: "푸꾸옥 풀빌라, 이 가격에 전용 수영장?ㅣ빌라고" (금칙어 가드 동일 적용). 설명에 카카오 링크 텍스트+해시태그(#Shorts #푸꾸옥 #풀빌라).
  - BGM: 쇼츠는 무음이 특히 불리 → **유튜브 오디오 라이브러리**(채널에 무료·안전) 트랙을 수동 다운로드해 assets/audio에 지정하는 방식 권장(라이선스 명확). 자동 파이프라인 기본은 기존 합성음/무음 유지.
- **생성 주기**: 인스타 draft cron에 유튜브 타깃 추가 — 같은 날 같은 빌라로 인스타 캐러셀+유튜브 숏츠를 함께 초안 생성(승인은 각각).

## 3. 콘텐츠 2 — 직접 촬영 영상 자동 편집 숏츠 ★신규 핵심

### 3-1. 촬영 가이드 (테오용 문서 — 편집 자동화의 절반은 촬영 규격화)
- 세로(9:16) 촬영 기본, 가로 촬영 시 중앙 크롭됨을 안내
- 클립 단위 촬영(장면당 5~15초 여러 개 > 3분 한 방) — 자동 컷 편집 품질이 크게 좋아짐
- 권장 샷 리스트: 입장→거실 파노라마→수영장(입수 컷)→침실→뷰/석양 (빌라당 5~8클립)

### 3-2. 자동 편집 파이프라인 (ffmpeg — 릴스 파이프라인 확장)
```
관리자 업로드(클립 N개 + 빌라 선택 + 헤드라인 선택/입력)
  → ① 정규화: 각 클립 9:16 변환(세로=그대로, 가로=중앙 크롭 or 블러 패딩), 해상도·fps 통일
  → ② 컷 편집: 클립별 하이라이트 구간 추출(기본=앞 N초, 옵션=장면 전환 감지로 베스트 컷)
  → ③ 브랜딩: villa-go 로고 워터마크(우상단 반투명) + 인트로 타이틀(빌라명·감성 헤드라인, satori 오버레이 재사용) + 아웃트로 CTA 카드
  → ④ 자막 텍스트: 구간별 정보 자막(침실 수·해변 거리 등 — 관리자가 입력/수정)
  → ⑤ BGM 합성 + 총 길이 15~60초로 조정 → MP4 → R2 업로드
  → 승인 큐(미리보기 재생) → 발행
```
- **Phase 1 자동화 수준(현실적)**: 정규화·로고·인트로/아웃트로·헤드라인은 전자동, 클립 순서·자막 문구·컷 길이는 관리자가 화면에서 조정(드래그 순서+텍스트 입력). "업로드하면 30초 뒤 초안 완성, 사람은 다듬고 승인만."
- **Phase 2 (선택)**: Gemini 영상 이해로 클립별 베스트 구간·자동 자막 제안 (Gemini가 영상 입력 지원 — 비용 대비 효과 검증 후).
- **인프라 주의**: 원본 영상 수백 MB 업로드 → 서버 직행 금지, **R2 presigned 직업로드** 후 서버는 R2에서 처리. ffmpeg 인코딩 수십 초~수 분 → cron/백그라운드 잡으로 비동기 처리(요청 타임아웃 회피), Railway 메모리·임시 디스크 확인.

## 4. 자동 업로드 (YouTube Data API)

- **모델**: `YoutubeShort` (InstagramPost 패턴 복제 — status DRAFT→PENDING_APPROVAL→QUEUED→PUBLISHING→PUBLISHED/FAILED, videoUrl, title, description, tags, ytVideoId, ytPrivacyStatus, sourceType VILLA_AUTO|UPLOADED, villaId?). 인스타 포스트와 1:1 연결 필드(선택)로 "같은 콘텐츠 2플랫폼" 추적.
- **업로드 클라이언트** lib/youtube/upload.ts: videos.insert(resumable, snippet+status) — 감사 통과 전에는 privacyStatus=unlisted/private로 업로드하고 관리자 화면에 "스튜디오에서 공개 전환" 딥링크 표시. 토큰 갱신(refresh→access) 자동.
- **cron**: instagram-publish와 동일 슬롯 구조(쇼츠 골든타임: 점심 12시·저녁 19~21시 KST). 쿼터 가드(일 업로드 상한 카운터). 킬스위치 YT_AUTOPOST_PAUSED.
- **admin**: /marketing 아래 "유튜브" 탭 — 인스타 큐 UI 재사용(카드에 영상 미리보기 플레이어), 촬영 클립 업로드 화면(콘텐츠 2), 설정 패널(OAuth 연결 상태·쿼터 사용량·킬스위치).
- **성과**: YouTube Analytics API로 조회수·구독 전환 일 1회 수집 → 인스타 인사이트와 나란히 표시 (기존 스냅샷 테이블 scope 확장).
