# 계약: T1.7(FE) 운영 설정 화면 — 시즌 달력·홀드 시간·환율

날짜: 2026-06-11 / 담당: FE(화면)·BE(API) / 평가: QA (독립)
선행: T1.7(BE) lib/pricing.ts 완료 (docs/contracts/T1.7-pricing.md), T1.2 사이드바·admin 토큰

## 범위

1. **`/settings`** (Stitch b8-settings 변환, ADMIN 다크):
   - 시즌 달력(SeasonPeriod) 연 단위 관리 — 기간 목록 + 추가/수정/삭제 (LOW/HIGH/PEAK, 시작·종료일, 라벨)
   - 홀드 시간 기본값(AppSetting `HOLD_HOURS_DEFAULT`, 기본 48)
   - 환율(AppSetting `FX_VND_PER_KRW`, 1 KRW = x VND) 수동 입력
2. **API** (BE, ADMIN 전용 + writeAuditLog):
   - `GET/POST /api/seasons`, `PUT/DELETE /api/seasons/[id]` — SeasonPeriod CRUD. 기간 검증(start < end), 겹침은 허용하되 응답에 경고(pricing의 PEAK>HIGH 우선 규칙 문서화됨)
   - `GET/PUT /api/settings` — AppSetting 키 화이트리스트(HOLD_HOURS_DEFAULT·FX_VND_PER_KRW)만 허용, 그 외 키 400. 값 형식 검증(홀드: 1~168 정수, 환율: 양수 소수 — lib/pricing getFxVndPerKrw 파서와 호환)
