# T0.9 — 스키마 드리프트 검증 (읽기 전용, push 없음)

> 담당: TDA · 합의: QA/OPS · 상태: 착수 (2026-06-16)

## 1. 배경·스코프

T0.9 잔여 = "schema v1.2(통화·비품·Zalo 양방향 ADR-0003) + T1.2b REJECTED 추가분이 프로덕션 Neon DB에 push 되었는지 확인".

**제약 (병렬 세션)**: `prisma/schema.prisma`가 현재 zca-js 세션(ADR-0006)에 의해 **작업 트리에서 수정 중**.
따라서 본 검증은:
- **HEAD(커밋된) 스키마 기준**으로만 DB 일치 여부를 본다. 작업 트리의 미커밋 zca-js 추가분은 검증 대상에서 제외(그 세션의 push 책임).
- **읽기 전용** — `prisma db push`·`migrate`·스키마 파일 수정 **절대 금지**. prod DB 메타데이터 조회만.

## 2. 검증 항목 (prod DB에 존재해야 함 — 커밋 스키마 기준)

DB 메타데이터(information_schema·pg_enum) 조회로 확인:
1. **enum Role**: ADMIN·SUPPLIER·CLEANER
2. **enum VillaStatus**: REJECTED 값 존재 (T1.2b)
3. **Villa.rejectionReason** 컬럼 (T1.2b)
4. **VillaAmenity** 테이블 + AmenityCategory enum(KITCHEN·BATHROOM·APPLIANCE·MINIBAR) (v1.2)
5. **Booking** 통화 컬럼: saleCurrency·totalSaleKrw·totalSaleVnd·fxVndPerKrw (v1.2 ADR-0003)
6. **ProposalItem** 듀얼 통화: priceKrwPerNight·totalKrw·priceVndPerNight·totalVnd
7. **ZaloConversation·ZaloMessage** 테이블 (v1.2 양방향 채팅)
8. **NotificationType** enum: VILLA_REJECTED·TAMTRU_PASSPORT 등
9. **AppSetting·AuditLog** 테이블 (v1.1)
10. **VillaRate**: supplierCostVnd·salePriceVnd·salePriceKrw·marginType·marginValue

## 3. 완료 기준

1. 위 10항목 각 존재/부재를 prod DB 조회 결과로 판정(증적 = 쿼리 출력).
2. 전부 존재 → "DB ⊇ 커밋 스키마, T0.9 충족" 판정.
3. 누락 발견 → 어떤 모델/컬럼이 빠졌는지 명시 + **push 필요 보고**(직접 push 금지 — 사용자 확인 + 스키마 전담 세션 조율 필요).
4. zca-js 미커밋 추가분(있다면)은 "그 세션 push 대기"로 별도 표기 — 본 검증 PASS/FAIL과 분리.

## 4. 검증 방법

- prod DB 조회: 임시 node 스크립트(PrismaClient `$queryRawUnsafe` on information_schema/pg_enum) 또는 `prisma db execute`. **커밋하지 않는 일회성 스크립트**(또는 temp 경로).
- 스키마 파일·DB 변경 0. 결과는 본 계약 또는 보고로만 기록.

## 5. 비고
- 이 작업은 LAUNCH.md 오픈 체크리스트의 사전 점검 성격(DB 일관성). push가 필요하면 zca-js 세션 완료 후 일괄 처리 권고.
