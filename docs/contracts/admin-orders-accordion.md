# 계약: ADMIN 부가옵션 주문 목록 아코디언화 (대량 대응 UX)

- 상태: 착수 (2026-07-12)
- 브랜치: wt/admin-orders-accordion
- 배경(테오): 주문이 많아지면 통짜 대형 카드 나열이 스크롤 과다 — 훑기·처리 불가 수준.

## 설계
1. **접힘 요약 행(기본)**: 품목명·구분(variant 라벨)·수량·이용일·상태 배지·발주 상태 배지·판매가(권한
   게이트)·티켓 카운터(N/M) 한 줄(모바일 2줄 wrap 허용, 가로 스크롤 금지 유지).
2. **펼침(아코디언, 행당 토글)**: 티켓 패널(선택 이용자·QR·첨부/삭제)·업체(공급자 변경·발주)·게스트
   메모·시간제안·확정가 입력·확정/제공완료/취소 액션 — 기존 기능 전부 펼침 영역으로 이동(유실 0).
   처리 필요 신호(요청 상태·미해결 제안·발행 미달)는 접힘 행 배지로 노출해 놓치지 않게.
3. **상태 필터 칩 + 건수**: 전체/요청/확정/취소(+제공완료는 확정에 포함 표기 또는 별도 — 구현 판단).
   클라 필터(패널 데이터는 이미 전량 로드). 기본=전체.
4. 다크 테마 유지·PR #262 카드 반응형(가로 스크롤 0) 유지·펼침 상태는 처리 액션 후에도 유지(refresh 후
   해당 행 유지 시도 — 어려우면 접힘 초기화 허용).
