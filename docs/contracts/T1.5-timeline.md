## 범위 (In)

1. **lib/timeline.ts** — 타임라인 데이터 단일 소스
   - 순수층(단위 테스트 대상):
     - `buildDayAxis(from, days=30)` — UTC 자정 Date 배열 (half-open 규약과 동일 축)
     - **오늘의 정의 (QA 합의 조건 1)**: `from` 기본값 = **Asia/Ho_Chi_Minh 기준 오늘 날짜의 UTC 자정 Date** (`Intl.DateTimeFormat` 변환 — lib/ical.ts 선례). 서버가 UTC여도 베트남 00:00~07:00에 첫 열이 어제로 밀리지 않음. M/D 라벨은 `getUTCMonth()+1`/`getUTCDate()`로 생성(서버 로컬 TZ 비의존) — 테스트로 증명
     - 셀 상태 enum 고정(T2.6 재사용 호환 계약): `"EMPTY" | "HOLD" | "CONFIRMED" | "CHECKED_IN" | "BLOCKED" | "NOT_SELLABLE"`
     - `computeVillaRow(villa, bookings, blocks, axis)` — 셀 상태 배열. 우선순위 **CHECKED_IN > CONFIRMED > HOLD > BLOCKED**, 점유 없음이면 `villa.isSellable ? EMPTY : NOT_SELLABLE`(검수 게이트 미통과 공실 = 판매불가 빨강 테두리 — SPEC F7). 날짜 비교는 half-open(체크아웃일 셀은 비점유)
   - DB층: `loadTimeline(db, {from, days})` — status=ACTIVE 빌라 전체 + 구간 겹침 점유 예약(OCCUPYING_BOOKING_STATUSES)·CalendarBlock 로드 → rows. **전체 재고 조망 — ADMIN 전용 소비 전제 주석 명시**(재고 비공개 원칙, 호출측 가드 책임). villa select는 `id, name, isSellable` 최소화(QA 권고 2)
   - 셀에는 **상태만** 전달 — 고객명·금액·예약 id는 비포함(타임라인 단계 불필요, 상세 연결은 T2.6/T2.5)
2. **components/admin/timeline-matrix.tsx** — b1 "Timeline Matrix Container" 그대로 변환(서버 컴포넌트, props 렌더 전용): 카드(bg-admin-card·rounded-xl·border), 헤더 제목+범례 4종(확정 파랑/가예약 빗금/차단 회색/판매불가 빨강 테두리), 가로 스크롤 테이블, 빌라명 sticky 첫 열(min-w 140px), 날짜 열 M/D(min-w 40px), 셀 상태 클래스 매핑(b1: confirmed bg-blue-600, checked-in bg-indigo-500, hold 빗금 45° 줄무늬+amber dashed border, blocked bg-slate-600, not-sellable border-2 red). 빈 상태(ACTIVE 빌라 0) 안내 포함
3. **app/(admin)/dashboard/page.tsx** — placeholder 교체: 기존 인사말 유지 + 타임라인 카드 렌더(`loadTimeline(prisma)`, 오늘부터 30일). 스탯 카드·활동 피드·경보 배너는 T2.6 범위 — 자리 문구만 유지
4. **단위 테스트** lib/timeline.test.ts — 축 생성(UTC 자정·개수·M/D 라벨 getUTC* 기반), half-open 경계(체크아웃일 비점유·back-to-back), 우선순위 중첩(HOLD+BLOCK 겹침 등), isSellable=false 공실=NOT_SELLABLE·점유 시 점유 상태 우선, 비점유 예약 상태(CANCELLED 등) 제외, **윈도우 경계 클리핑**(축 시작 전 시작~축 안으로 걸치는 예약, 축 끝을 넘는 구간 — QA 권고 1)
