# T4.2 — 파일럿 시드 스크립트 (쏘나씨 V11/V12/V25 + 썬셋 사나토 A3)

> 담당: BE · 합의: QA · 상태: 착수 (2026-06-16)

## 1. 목적·근거

- LAUNCH.md 오픈 기준 체크리스트의 "파일럿 실데이터 입력" 충족.

## 2. 범위 (신규 파일만 — hot working tree 충돌 0)

- **신규**: `prisma/seed.ts` — 멱등(upsert) 독립 실행 시드 스크립트.
- **신규**: `prisma/seed.test.ts` — 시드 데이터 정합성 단위 테스트(순수 검증 함수 분리 export).
- **수정 금지 구역**: 다른 세션 활성 파일 전부 — `prisma/schema.prisma`, `messages/*.json`, `package.json`,
  `app/(supplier)/my-villas/**`, `app/(admin)/**`, `lib/cleaning.ts·hold.ts·proposal.ts·gemini.ts`,
  `app/api/villas/**`. 시드는 이들을 **읽기만** 하고 import해서 사용한다.
- **package.json 동결 준수**: `prisma.seed` 설정·스크립트 추가하지 않는다. 실행은 `npx tsx prisma/seed.ts`
  (tsx v4.22.4 설치 확인됨, Node v24).

## 3. 시드 내용

### 사용자 (upsert by 고정 id)
- ADMIN 테오 (`seed-admin-theo`, locale ko) — 비밀번호 환경변수 `SEED_ADMIN_PASSWORD`(미설정 시 더미).
- SUPPLIER 파일럿 중계인 (`seed-supplier-pilot`, locale vi).

### AppSetting (upsert by key — 앱이 기대하는 키 전체)
- `HOLD_HOURS_DEFAULT` = "48"
- `FX_VND_PER_KRW` = "18.87" (1 KRW = x VND)
- `BANK_NAME` / `BANK_ACCOUNT_NUMBER` / `BANK_ACCOUNT_HOLDER` (입금 카드)
- `CONTACT_KAKAO_URL` / `CONTACT_PHONE` (공개 페이지 문의)
- 키 상수는 `@/app/api/settings/validators`·`@/lib/hold`·`@/lib/pricing`에서 import (하드코딩 금지).

### SeasonPeriod (upsert by 고정 id) — 2026 연간 시즌 달력
- LOW(기본)·HIGH(여름 성수기)·PEAK(연말·설) 구간.

### Villa ×4 (upsert by 고정 id, status=ACTIVE, isSellable=true)
- 쏘나씨 V11/V12/V25 (complex "쏘나씨"), 썬셋 사나토 A3 (complex "썬셋 사나토"). 파일럿 공급자 소유.

### VillaRate ×(4×3) (upsert by `villaId_season` 복합 unique)
- 시즌별 supplierCostVnd + marginType PERCENT + marginValue + salePriceVnd + salePriceKrw.
- salePriceKrw = salePriceVnd ÷ FX_VND_PER_KRW 환산 후 천원 단위 라운딩.

### VillaPhoto (upsert by 고정 id) — 빌라당 최소 외관·거실·침실 placeholder URL (등록 필수 충족).

## 4. 완료 기준 (테스트 가능)

1. `npx tsx prisma/seed.ts` 두 번 연속 실행 시 행 수 동일(멱등) — upsert 검증.
2. `npx vitest run prisma/seed.test.ts` 통과: ① AppSetting 키가 SETTING_KEYS 전부 포함, ② 각 Villa에
   3개 시즌 요율 모두 존재, ③ salePriceVnd > supplierCostVnd(마진 양수), ④ salePriceKrw 라운딩 정합,
   ⑤ 금액 타입 규칙(VND BigInt, KRW number).
3. 프로덕션 DB 적재는 실행자가 `DATABASE_URL` 확인 후 수동 — 스크립트는 멱등하므로 재실행 안전.
4. `npm run typecheck` 시 seed 관련 신규 에러 0건(기존 lib/zalo.ts T3.5 이슈는 무관).
