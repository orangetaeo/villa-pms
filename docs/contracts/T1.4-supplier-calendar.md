## 범위

`app/(supplier)/calendar` — 빌라별 월 달력 (모바일 390px, vi):

1. **달력 화면** (a3 변환)
   - 상단 빌라 선택 칩 행 (자기 빌라만 — supplierId 스코프), 월 이동 (이전/다음)
   - 날짜 셀 4상태 — 색+패턴 병행(색 단독 금지, a3): Trống(흰 배경+초록 외곽선) / Đã đặt(파랑 실선 #2563EB 흰 글자 — CONFIRMED·CHECKED_IN) / Giữ chỗ(연파랑 #DBEAFE+파선 테두리 — HOLD) / Đã khóa(회색+대각 빗금 — CalendarBlock)
   - 범례: 스와치 ≥16px, 텍스트 ≥14px (#1F2937), 셀과 동일 패턴
   - **예약 상세(고객명·금액) 비노출** — "Đã đặt" 표시만 (마진·재고 비공개 원칙)
   - 가격 어디에도 미표시
2. **차단 토글** (바텀시트)
   - 빈 날짜 탭 → 바텀시트: 날짜 표시 + 큰 버튼 "Khóa ngày" / 차단 날짜 탭 → "Hủy khóa"
   - MANUAL 블록만 해제 가능. ICAL 블록은 안내만(해제 불가 — T1.6 동기화 소유). 예약 셀은 바텀시트 미오픈(또는 정보만)
   - 과거 날짜는 읽기 전용
   - SPEC의 "기간 선택"은 a3 확정 디자인(단일 날짜 토글)을 우선 — 연속 차단은 다중 탭으로 대체, 기간 UI는 IDEAS 보류
3. **데이터·API**
   - 조회: 서버 컴포넌트에서 Prisma 직접 쿼리 (월 범위 Booking[HOLD|CONFIRMED|CHECKED_IN] + CalendarBlock, half-open 겹침 — `lib/availability.ts`의 OCCUPYING_BOOKING_STATUSES·overlapsHalfOpen 재사용, 판정 중복 구현 금지)
   - `POST /api/calendar-blocks` — SUPPLIER 전용, 빌라 소유권 검증, **날짜 UTC 자정 정규화**(availability-pattern 교훈), 예약·기존 블록 겹침 시 409, CalendarBlock(MANUAL, [d, d+1)) 생성 + `writeAuditLog`
   - `DELETE /api/calendar-blocks/[id]` — 소유 빌라 + MANUAL만, `writeAuditLog`
4. **하단 탭바** (supplier 공통 — layout TODO 해소)
   - `components/supplier/tab-bar.tsx`: Villa(/my-villas) · Lịch(/calendar) · Dọn dẹp(/cleaning) · Thu nhập(/earnings), 활성 표시, a3/a6 디자인 톤
   - /cleaning·/earnings는 최소 placeholder 페이지(후속 태스크 안내 텍스트만, 임의 디자인 금지)
   - app/(supplier)/layout.tsx에 탭바 장착 + 본문 하단 패딩

## 완료 기준 (테스트 가능)

1. SUPPLIER 로그인 → /calendar → 자기 빌라 칩만 노출, 타인 빌라 데이터 접근 불가(쿼리 스코프 + API 403/404)
2. 빈 날짜 탭 → Khóa ngày → 셀이 빗금 차단으로 갱신 + CalendarBlock(MANUAL) 생성 + AuditLog. 차단 탭 → Hủy khóa → 해제 + AuditLog
3. 예약 날짜(HOLD/CONFIRMED) 차단 시도 → 409, UI에서도 토글 불가. 과거 날짜 토글 불가
4. 4상태 셀이 색+패턴으로 구분(빗금·파선 렌더 확인), 범례 일치, 고객명·금액·가격 DOM에 0건
5. 날짜는 UTC 자정 정규화 — 셀 날짜와 저장된 블록 날짜 불일치 없음 (Asia/Ho_Chi_Minh 표시 기준)
6. 하단 탭바 4탭 동작(Lịch 활성), my-villas에서도 탭바 노출
7. "biệt thự" 0건, 비로그인/ADMIN·CLEANER의 POST·DELETE 차단, tsc·lint 통과 (충돌 구역 파일 git diff 0)
