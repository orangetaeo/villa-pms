# 계약: ADMIN 주문 목록 품목 그룹핑 (구분 분리 주문을 한 구매로 표시)

- 상태: 착수 (2026-07-12)
- 브랜치: wt/admin-orders-grouping
- 배경(테오): 키스브릿지 1건 구매(무료 1+일반 2)가 두 행으로 나뉘어 보임 — 실제 한 구매.
  구분별 주문 분리는 데이터 설계(가격·발행·처리 상이)로 유지하되 표시를 묶는다.

## 설계
- 그룹 키 = catalogItemId(없으면 type) + serviceDate (소비자 측 group-orders.ts와 동일 원칙,
  admin 데이터 shape에 맞춘 순수 함수 — lib에 두고 단위 테스트).
- **그룹 헤더 행**: 품목명 · 총수량(Σ) · 이용일 · 합계 금액(판매가 합) · 대표 배지(그룹 내 최고
  우선순위 상태: 요청>확정>취소 — 처리필요 신호 orderAttention도 그룹으로 승격 표시) · 티켓
  카운터 합(발행 N/M 합).
- 그룹 펼치면 **구분 라인들**(기존 아코디언 행 그대로 — 라인별 접힘/펼침·배지·액션 전부 유지).
  1주문 그룹은 헤더 없이 기존 단일 행(불필요한 중첩 방지).
- 상태 필터 칩은 라인 기준 유지(그룹 내 일부만 매칭 시 그룹 표시하되 매칭 라인만).
