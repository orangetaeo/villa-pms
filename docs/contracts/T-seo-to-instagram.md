# T-seo-to-instagram — 장소 글을 인스타 소재로 재사용

## 배경 (테오 제안 2026-07-23)

> 사진이 있는 블로그 글이 승인될 때 인스타그램 포스팅·릴스·유튜브 쇼츠에 바로 쓸 수 있게 하면
> 풍부한 콘텐츠를 공급할 수 있지 않을까?

**지금 콘텐츠 공급은 실제로 멈춰 있다.** IG·YT 자동 초안은 빌라당 상한(기본 1)에 걸려 있고
빌라 2곳 모두 이미 발행분이 있어 **적격 빌라 0곳**이다(2026-07-23 드라이런 확인).
반면 장소 글에는 운영자가 직접 찍은 사진이 10장 넘게 붙어 있다 — 이미 있는 재료를 안 쓰고 있었다.

## 이미 있는 이음새 (그래서 스키마 변경이 거의 없다)

- `InstagramPost.villaId`는 **nullable**이다 — 모델 주석에 "정보/서비스 포스트면 null"로 이미 명시됨
- `buildSlides` → `renderCarousel` → `InstagramPost` 경로는 슬라이드 배열만 받으면 동작한다
- 릴스는 같은 슬라이드로 `renderAndBuildReel`을 탄다

## 범위

| # | 항목 | 내용 |
|---|---|---|
| A | 스키마 | `InstagramPost.seoArticleId` additive(nullable, SetNull) — "이 글로 이미 만들었나" 판정 축 |
| B | 소재 선택 | `lib/instagram/place-draft.ts` — 발행된 장소 글 중 IG 포스트가 없고 사진 4장 이상인 것 |
| C | 슬라이드 | 장소 사진(역할 순: 외관→음식→내부) + 커버 헤드라인 + CTA. 빌라 전용 `info` 슬라이드는 쓰지 않는다 |
| D | 캡션 | 장소 전용 생성기 — 사실은 `oneLiner`·`tips`·사진 설명·동네까지만 |
| E | cron | 빌라 후보가 슬롯을 못 채우면 **장소 글로 채운다**(빌라 우선, 장소가 보충) |
| F | 테스트 | 소재 선택 조건·중복 방지·캡션 금지 사실 |

## 절대 규칙

- **가격·영업시간·전화번호·주소를 캡션에 넣지 않는다** — 장소 글과 동일한 이유(바뀌면 틀린 글)
- **등록된 사실 밖을 지어내지 않는다** — 캡션 프롬프트에 장소 사실만 주입
- **한 글당 IG 포스트 1개** — `seoArticleId`로 판정. 도배 방지(빌라당 상한과 같은 취지)
- 빌라 초안 경로는 **한 줄도 바꾸지 않는다**(장소는 남는 슬롯만 채운다)
- 승인 게이트 유지 — 생성물은 `PENDING_APPROVAL`. 자동 발행 아님

## 이번 범위에서 제외 (다음 단계)

- **릴스·유튜브 쇼츠**: 나레이션(TTS)·쇼츠 파이프라인이 빌라 사실 구조에 묶여 있어 별도 작업이 필요하다.
  캐러셀이 먼저 돌아가는 것을 보고 같은 소재로 확장한다.
- 승인 **시점 트리거**: 무거운 렌더를 서버 액션에서 돌리면 승인이 느려지고 실패 시 되돌리기 어렵다.
  → 발행된 글을 cron이 집어가는 방식(기존 파이프라인과 동일한 리듬)

## 완료 기준

1. 장소 글 1편(사진 4장 이상) → IG 초안 1건 `PENDING_APPROVAL` 생성, `villaId=null`·`seoArticleId=글`
2. 같은 글로 두 번 만들지 않는다
3. 사진 4장 미만·이미 포스트 있음 → 후보에서 제외
4. 캡션에 가격·영업시간·전화 문자열이 없다(테스트)
5. 빌라 후보가 있으면 빌라가 우선 슬롯을 가져간다
6. `lint && typecheck && vitest && build` 통과
