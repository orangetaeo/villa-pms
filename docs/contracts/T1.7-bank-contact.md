# 계약: T1.7-bank-contact — 입금 계좌·연락처 설정 UI (b8 Card 3 변환)

## 배경
3라운드 디자인(커밋 1ebc65c)이 b8-settings에 **Card 3 "입금 계좌·연락처"**를 추가.
키 5종(BANK_NAME·BANK_ACCOUNT_NUMBER·BANK_ACCOUNT_HOLDER·CONTACT_KAKAO_URL·CONTACT_PHONE)은
이미 공개 제안 완료/만료 페이지(`app/p/[token]/done`·`expired-view`)가 소비 중이나
ADMIN 입력 UI가 없어 DB 직접 입력에 의존 — T2.2가 "T1.7 /settings 입력 UI 잔여(FE)"로 명시한 공백.

## 완료 기준 (테스트 가능)
1. ADMIN이 /settings Card 4에서 5필드 입력·저장 → AppSetting 5키 upsert, 빈 값은 삭제(clear)
2. 저장 후 공개 완료 페이지에 은행 정보, 만료 페이지에 연락처 노출 반영
3. 비ADMIN(SUPPLIER)·비로그인 PUT → 403/401
4. 화이트리스트 밖 키 → 400 INVALID_KEY (기존 가드 유지)
5. CONTACT_KAKAO_URL은 http(s) URL만, 전화·계좌번호 형식 검증
6. 모든 변경에 writeAuditLog 기록 (글로벌 절대 규칙)
7. `npm run typecheck` 통과, settings route 단위 테스트 통과
8. 마진·재고 누수 0건 (admin 전용 — 누수 위험 낮음, QA 확인)
