# 계약서: 미체크인 티켓 구분별 수량 선택 (ticket-variant-qty-precheckin)

- 상태: 착수 (2026-07-13)
- 담당: FE (Opus) / QA 독립 검증
- 발단: 테오 실기기 테스트 — 게스트 부가옵션에서 나이트사파리 성인2·어린이1·시니어1 혼합 선택 불가

## 배경 (Triage)

- 체크인 명단 존재 시: TICKET+variants 카드는 인원별 자동판정 모드(이름 칩) — 혼합 구분 구매 이미 가능 (구분별 1주문 분리, PR #248/#251).
- **미체크인(명단 빈 배열)**: `option-card.tsx` 폴백이 구분 1택 라디오 + 수량 스테퍼 1개 → 혼합 선택 UI상 불가.
  또한 규칙 구분(어린이·시니어·무료)은 서버 `TICKET_GUESTS_REQUIRED`(P2-B, 봉인 가드)로 거부됨 —
  현재 UI는 선택 가능해 보이지만 실제로는 성인(무규칙 기본 구분)만 주문 가능한 오해 유발 상태.

## 범위 (서버 가드 불변 — 봉인 유지)

1. **미체크인 TICKET+variants 카드**: 구분 1택 라디오+공용 스테퍼 → **구분별 수량 스테퍼**로 교체.
   - 무규칙 구분(성인 등): 수량 자유 선택.
   - 규칙 구분(bornBeforeYear·heightMaxCm 등): 스테퍼 비활성 + "셀프 체크인 후 인원별 선택 가능" 안내(체크인 링크) — 제출 후 400 대신 사전 유도.
   - 구분별 소계 줄 + 카드 합계(체크인 모드와 동일한 표기 패턴).
2. **제출**: 수량>0 구분마다 1주문(variantKey+quantity) — 기존 서버 스펙(구분별 분리 주문)에 정합. ticketGuests 미첨부(무규칙 구분이라 서버 통과).
3. **라벨**: TICKET 카드 구분 섹션 제목이 `timeLabel`("시간 선택") 오용 중 → 티켓 전용 "구분 선택" 라벨 신설(guest-i18n 5언어: ko/en/ru/zh/vi).
4. 하단 합계(grandTotal)·필수값 검증(canSubmit)·활성 판정(quantity)을 구분별 수량 합 기준으로 정합화.

## 수정 금지 구역

- `lib/ticket-order-validation.ts` (서버 봉인 가드), `app/api/**` — 서버 무변경.
- 체크인 명단 모드(isTicketVariantPerson)의 기존 동작·payload 불변.
- 비-TICKET variants(마사지·BBQ 시간 선택 등)는 기존 1택 유지.

## 완료 기준 (테스트 가능)

- [ ] 미체크인 상태에서 TICKET 카드에 구분별 스테퍼 노출, 무규칙 구분 2개 이상 수량 동시 선택 가능.
- [ ] 규칙 구분은 비활성 + 체크인 유도 안내 노출(스테퍼 조작 불가).
- [ ] 제출 시 수량>0 구분 수만큼 주문 생성(각각 올바른 variantKey·quantity·단가), /orders에서 확인.
- [ ] 체크인 명단 존재 예약에서는 기존 이름 칩 모드 그대로(회귀 0).
- [ ] TICKET 구분 섹션 라벨이 "구분 선택"(5언어), 비-TICKET은 기존 라벨 유지.
