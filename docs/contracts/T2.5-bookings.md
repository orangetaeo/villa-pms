## 범위 (In)

1. **`/bookings` 목록** (b5 변환, RSC + prisma 직접 조회 — 신규 목록 API 없음, 누수 표면 최소화)
   - 상태 탭 6종 + 건수: 전체/가예약(HOLD)/확정(CONFIRMED)/체크인(CHECKED_IN)/체크아웃(CHECKED_OUT)/취소·만료(CANCELLED+EXPIRED+NO_SHOW). 건수는 상태 외 필터(월·빌라·채널·검색) 적용 상태에서 집계
   - 필터: 월(YYYY-MM, 기본=Asia/Ho_Chi_Minh 기준 당월, 해당 월과 half-open 겹침), 빌라 select(전체 빌라 — ADMIN 화면), 채널 select(직접/여행사/랜드사), 검색 q(고객명 contains·예약번호 contains), 초기화. 전부 searchParams — 새로고침·공유 가능
   - 정렬 체크인 임박순(checkIn asc), 페이지네이션(b5 푸터 "N / M" + 전체 건수)
   - **T2.6 프리셋 파람 (QA 합의 조건 3 — 옵션①)**: `?filter=today-checkin`(오늘 checkIn·CONFIRMED)·`?filter=today-checkout`(오늘 checkOut·CHECKED_IN) 지원 — SPEC F7 대시보드 카드 링크 목적지. 오늘=Asia/Ho_Chi_Minh 기준. T2.6은 본 page.tsx를 수정하지 않고 링크만 건다
   - 행: 빌라/고객명/채널(agencyName 병기)/체크인/체크아웃/박/금액/상태 배지. **금액 표기(QA 조건 4)**: b5 모크 충실 — `formatThousands` 재사용 + 접미사(KRW `1,234,000원`, VND `28,400,000₫`). 기존 `formatKrw`(₩ 접두)는 무변경. HOLD 배지는 timer 아이콘+**남은 시간**(`formatRemainingHours` — 1h 이상 "N시간 남음", 미만 "N분 남음", 경과 "만료 처리 대기", **holdExpiresAt null → null 반환·미표기(QA 조건 2)**). EXPIRED 행 opacity-60. 행 전체 클릭 → 상세(Link stretch — ResponsiveTable 재사용, <768px 카드 전환=T6.7). PAGE_SIZE=20 고정
   - 하단 스탯 미니그리드 4종(b5): 금일 체크인/금일 체크아웃(+완료 N)/미확정 가예약 — 3종은 **전역(오늘 VN 기준, 목록 필터 비적용)**, 가동률만 **선택 월** 기준(비대칭 명시 — QA 권고). 가동률 = 순수 함수 `computeOccupancyRate`: **점유 상태 = CONFIRMED·CHECKED_IN·CHECKED_OUT·NO_SHOW(날짜 점유 보상 — SPEC F3), HOLD·CANCELLED·EXPIRED 제외(QA 합의 조건 1)**, half-open·월 경계 클리핑, 분모 = 현재 ACTIVE 빌라 수 × 월 일수(월 중 승인 시점 무시 — 근사 허용 선언), 빌라 0대 → 0%
2. **`/bookings/[id]` 상세** (b11 변환, RSC)
   - 헤더(예약 번호 축약 #끝8자리·상태 배지), 상태 타임라인 스트립(가예약→확정→체크인→체크아웃 4단계 진행 표시; CANCELLED/EXPIRED/NO_SHOW 종결 상태는 스트립 대신 사유 포함 상태 배너 — b11에 없는 상태의 처리, 편차 선언)
   - 예약 정보 카드(빌라(링크)/채널/일정+박/인원/연락처), **가격 스냅샷 카드**(판매 통화 뱃지, 총 판매가(saleCurrency 해당 컬럼), 공급자 원가 VND, 조식 여부, "홀드 시점 가격으로 고정됨" 푸터) — 원가는 ADMIN 전용 화면이므로 표시 허용(레이아웃 가드)
   - 결제 기록 테이블(Payment: 일시/수단/금액(통화별)/비고 — 빈 상태 안내. 결제 등록 UI는 범위 밖)
   - **액션 패널 (상태별, client component)**: HOLD→[입금 확정(POST /api/bookings/[id]/confirm), 취소(사유 입력 필수 → POST cancel)] + 만료 카운트다운 / CONFIRMED→[체크인 진행(**disabled** — T3.1), 취소, 노쇼 처리(**disabled** — T3.4·정산 정책 연계)] / CHECKED_IN→[체크아웃 진행(**disabled** — T3.3)] / 종결 상태→액션 없음. 409(HoldRejectedError) 메시지 표시, 성공 시 refresh
   - 활동 로그(AuditLog entity=Booking·entityId, 최근 20건 — 액션 라벨·처리자(시스템/사용자명)·시각), **내부 메모**(Booking.note — 저장 버튼)
3. **PATCH /api/bookings/[id]** (BE 신규) — body `{ note: string(≤2000) }`만 허용(zod). 첫 줄 ADMIN 검사(401/403 분리 — T1.4 교훈), **상태·금액 등 다른 필드 전이 불가**(note 전용 — 전이 무결성은 confirm/cancel/expire 전용 경로 유지), 미존재 id → 404, 빈 문자열 → null 저장(메모 삭제 — QA 권고), writeAuditLog(UPDATE, changes.note old/new), 응답 select 최소화(id·note)
