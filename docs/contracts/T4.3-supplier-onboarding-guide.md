## 1. 목적·근거

- LAUNCH.md P1 목표 "공급자가 **혼자** 빌라 등록·캘린더 관리 성공(설명 없이)"의 화면 지원.
- 베트남 중계인이 앱 첫 사용 시 4개 핵심 기능(등록→캘린더→청소→알림)을 그림+최소 텍스트로 이해.
- 근거 문서: docs/SPEC.md F0(가입·온보딩)·F1~F5, docs/DESIGN.md(공급자 화면 원칙: 아이콘+사진, 텍스트 최소).

## 2. 디자인 선행 (design-first 규칙)

- Stitch로 가이드 화면 1장 생성 → `design/stitch/a-guide/` 저장 (신규 폴더, 충돌 0).
- 공급자 디자인 시스템: 라이트 테마, teal primary(#0d9488 계열), 큰 터치 영역, material-symbols 아이콘,
  Be Vietnam Pro + Noto Sans KR 폴백. 한 화면에 강조 버튼 ≤3.

## 3. 범위 (신규 위주 — hot working tree 회피)

- **신규**: `app/(supplier)/guide/page.tsx` — 서버 컴포넌트(getTranslations "guide"), 모바일 1장 가이드.
  4개 카드(빌라 등록·캘린더 관리·청소 사진·Zalo 알림) 각 아이콘+제목+1줄 설명, 하단 "내 빌라로 이동" CTA.
- **신규**: `design/stitch/a-guide/` — Stitch export.
- **추가만(공유 파일·빠른 커밋)**: `messages/vi.json`·`messages/ko.json` — `guide` 네임스페이스 신규 +
  `tabs.guide` 키 추가. 기존 키 수정 0.
- **안전 수정**: `components/supplier/tab-bar.tsx` — 진입점. 현재 4탭 유지하되 가이드 진입은
  **홈 외 hot 파일을 건드리지 않도록** TabBar에 5번째 항목 "Hướng dẫn"(가이드, icon `help`) 추가.
  (a6 BottomNav 4→5 확장은 DESIGN 자가검토로 검증 — material bottom nav 5개 허용 범위)

## 4. 완료 기준 (테스트 가능)

1. `/guide`가 SUPPLIER 세션(vi)에서 렌더 — 4개 기능 카드 + CTA, 하드코딩 텍스트 0(전부 i18n 키).
2. TabBar에 "Hướng dẫn" 진입 → `/guide` 이동, active 하이라이트 정상.
3. vi·ko 양쪽에 `guide` 네임스페이스 + `tabs.guide` 키 존재(키 누락 시 next-intl throw) —
   `tests/guide-i18n-keys.test.ts`로 키 동등성 검증.
4. 비로그인 접근 시 `/login` 리다이렉트(supplier layout 기존 가드 재사용 — 신규 API 표면 0).
5. `npm run typecheck` + `npx vitest run` 신규 에러 0. 권한 누수 검토: 가이드는 정적 안내 —
   판매가·마진·재고 데이터 미참조(QA leak-checklist 자명 통과).
