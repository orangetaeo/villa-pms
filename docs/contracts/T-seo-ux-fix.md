# T-seo-ux-fix — SEO 관리 화면 정리 + 노출 제어 (테오 지적 6건)

메오키친 첫 등록 실사용에서 나온 지적. 6건 중 4번은 설계 버그다.

| # | 지적 | 원인 | 조치 |
|---|---|---|---|
| 1 | 사진을 한 장씩만 올릴 수 있다 | 업로더가 단일 파일 | **다중 선택 업로드**(파일 여러 개 → 썸네일 목록 → 설명 각각) |
| 2 | 여러 장 올렸는데 미리보기가 일부만 보여 등록 여부를 알 수 없다 | 목록에서 `take: 4`로 잘랐음 | **전체 표시 + 총 장수** + 사진별 내리기 버튼 |
| 3 | 자료 사진 목록의 카드가 너무 크다 | 카드 1장이 3열 그리드 폭 전체 | **조밀한 썸네일 그리드**(고정 높이, 정보는 한 줄) |
| 4 | **자료 사진에 식당 사진이 섞여 나온다** | ★설계 버그 — 자료 사진 목록이 `placeId` 유무를 구분하지 않음 | 자료 사진 = **범용 사진만**(`placeId: null`). 장소 사진은 장소 화면에서만 관리 |
| 5 | 링크 나열·폼·주제 선택까지 화면이 어지럽다 | 화면마다 링크를 흩뿌리고 체크박스 17개를 한 번에 노출 | **공통 탭 1줄**로 통일, 폼 2단 정리, 주제 선택은 **접힌 그룹**으로 |
| 6 | 발행된 글을 내릴 방법이 없다 | 상태가 PUBLISHED뿐 | `SeoArticle.publicHidden` additive + 공개 조회 2곳 필터 + 승인 큐에서 토글 |

추가: **"이 장소로 지금 글 만들기"** — 자동 생성은 3곳부터지만, 운영자가 명시적으로 누르면 1곳으로도 만든다(테오 요청: 메오키친 샘플).

## 절대 규칙

- `publicHidden` 필터는 **`getPublishedArticles` / `getPublishedArticleBySlug` 두 곳**에만 넣는다 —
  이 둘이 "공개 라우트·sitemap·RSS가 공유하는 단일 진입점"이라 여기만 막으면 전부 막힌다.
- 사진 중복 URL은 **저장 단계에서 무시**한다(메오키친에 같은 URL 2행이 실제로 들어갔다).
- 삭제는 없다 — 사진·장소·글 모두 `active`/`publicHidden`으로 내린다.

## 완료 기준

1. 사진 여러 장을 한 번에 선택해 업로드 → 각 사진에 설명을 달아 한 번에 저장
2. 장소 화면에서 등록된 사진이 **전부** 보이고, 장수가 표시되고, 개별로 내릴 수 있다
3. 자료 사진 목록에 장소 사진이 **나오지 않는다**
4. 자료 사진 목록이 한 화면에 여러 장 들어오는 크기다
5. 세 화면(승인 큐·장소·자료 사진)이 같은 탭 네비게이션을 쓴다
6. 발행된 글을 비노출로 내리면 `/blog` 목록·상세·sitemap에서 사라지고, 되돌릴 수 있다
7. 메오키친으로 샘플 초안 1편 생성(승인 대기)
8. `lint && typecheck && vitest && build` 통과
