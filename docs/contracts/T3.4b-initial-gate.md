## TDA 결정 (ADR-0006으로 기록)

**빌라 최초 APPROVE 시 초기 검수 CleaningTask(PERIODIC) 자동 생성** — 공급자가 현 상태 사진 제출 → ADMIN /inspections에서 승인 → 기존 게이트 메커니즘(미결 CHECKOUT 0건 → isSellable=true)으로 개방.
- **ADMIN 수동 개방 토글은 도입하지 않음** — 사진 검수 없는 게이트 우회는 원칙 3 위반 (대안 기각 사유)
- 스키마 변경 없음 — CleaningType.PERIODIC 재사용(note "초기 검수" 표기), enum 추가(INITIAL)는 규칙 #6(스키마 전담) 비용 대비 이득 없음
- 멱등: **해당 빌라에 CleaningTask가 0건일 때만** 생성 — 재승인(REACTIVATE)·중복 APPROVE 시 미생성, 기존 검수 이력 빌라는 영향 없음
