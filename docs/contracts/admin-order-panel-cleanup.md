# 계약: ADMIN 주문 패널 정리 — 정산 일원화·좌우 스크롤 제거·역할 축소

- 상태: 착수 (2026-07-12)
- 브랜치: wt/admin-panel-cleanup
- 배경(테오): ① 예약 상세 부가옵션 주문 리스트에 좌우 스크롤 발생. ② "정산완료" 버튼(=업체
  건별 지급 표시)이 예약 상세 패널과 /service-orders 정산 허브 두 곳에 존재 — 중복. 정산은
  부가서비스 정산 카테고리(허브)로 일원화. ③ 패널 역할은 완료 전 주문의 공급자 변경 중심으로 축소.

## 설계
1. **정산 일원화**: service-orders-panel의 건별 정산 컨트롤(SettleControl·settleVendor·정산
   상태 표시 중 조작 UI) 제거 — 조회성 정산 상태 뱃지는 유지 가능(간단 표시). 허브(/service-orders)
   정산 탭이 유일한 정산 조작처 — 허브에 건별 정산이 없으면 최소 추가(공급자 묶음 내 개별 체크/버튼).
   서버 API(PATCH markSettled)는 불변(허브가 사용).
2. **좌우 스크롤 제거**: 패널 주문 표(overflow-x-auto table)를 반응형 카드/랩 레이아웃으로 재배치
   — 정산 제거로 컬럼 수도 줄어 자연 해소. 모바일·데스크톱 모두 가로 스크롤 0.
3. **역할 축소**: 남기는 조작 = 공급자 변경(+자동 재발주)·발주 보내기·상태 전이(확정·취소)·티켓
   대리 첨부·시간제안 적용. 금액(원가·판매가) 조정은 유지(운영자 확정 업무). 정산만 허브로.

## 범위
1. app/(admin)/bookings/[id]/service-orders-panel.tsx — 레이아웃 재배치·정산 조작 제거
2. app/(admin)/service-orders/service-orders-view.tsx — 건별 정산 필요 시 보강
3. messages/ko.json·vi.json — 문구 정리(키 추가/삭제 최소)
4. 테스트: 기존 패널 테스트 회귀, 허브 건별 정산(추가 시)
