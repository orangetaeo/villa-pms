- 상태: 착수 (2026-07-12)
- 브랜치: wt/ticket-card-totals
- 배경(테오 실측 2건):
  1. 게스트 신청 실패("요청 처리 중 문제") — 원인=운영자가 에디터에서 variant 가격을 비워 저장(케이블카
     2개 null) → 주문 시 NO_PRICE 400. **데이터는 보정 완료** — 재발 방지 코드 필요.
  2. 카드가 "1장당 요금"만 표시 — **티켓(품목)별 합계 요금** 표시 요구.

## 범위

1. **variant 가격 필수 가드(재발 방지)**
   - 카탈로그 write API(app/api/services/catalog/route.ts·[id]/route.ts): variants[].priceVnd 필수
     (숫자 문자열, "0" 허용) — 누락 시 400 `VARIANT_PRICE_REQUIRED`. addons/modifiers는 현행(가산이라 null=0 허용) 유지.
   - 에디터(catalog-manager.tsx): variant 가격 빈칸 저장 시 인라인 경고 + 저장 차단(ko/vi).
2. **게스트 카드 품목 합계 표시**
   - option-card.tsx: 선택 수량>0이면 카드 하단에 **합계**(단가×수량) 표시. 티켓 인원 선택 모드는
     **구분별 소계 줄**(예: "성인 ×2 = 3,000,000₫ / 어린이 ×1 = 1,100,000₫") + 카드 합계.
     기존 "단가/1장·N개 선택"과 병행(대체 아님). KRW 환산은 하단 총계만(현행 유지).
   - guest-i18n 5언어 키(합계 라벨 등 최소).
3. **오류 문구 구체화(소액)**: 게스트 제출 실패 시 서버 코드 매핑 — VARIANT_REQUIRED/NO_PRICE/
   UNKNOWN_VARIANT는 "품목 설정 오류 — 운영자에게 문의해주세요"(5언어), 그 외 기존 일반 문구.
4. 테스트: write API 가격 누락 400 / 카드 합계·소계 계산 순수 함수 / 기존 회귀.
