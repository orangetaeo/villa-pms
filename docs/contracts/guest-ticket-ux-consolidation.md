# 계약: 게스트 티켓 이용자 정보 1회 입력 UX 통합

- 상태: 착수 (2026-07-12)
- 브랜치: wt/guest-ticket-ux
- 배경(테오): ① 여권 정보(이름/연생)가 티켓 카드마다 반복 노출 — 산만. ② 신장 등 이용자 정보를
  **한 번만 입력**하고 전체 티켓에 적용되게. 단 품목별 이용 인원은 다를 수 있음(키스쇼 2명·심포니 3명).

## 설계 (UX-VN·TDA)

- **"티켓 이용자 정보" 통합 카드 1개**(신규): 카탈로그에 TICKET 품목이 있고 체크인 명단이 있으면
  티켓 품목들 상단에 1회 렌더. 투숙객 행마다: 이름 · 생년월일 + **신장(cm) 입력 1칸**
  (신장 규칙(heightMaxCm) 품목이 하나라도 있을 때만 입력칸 노출). 현장 측정·차액 고지도 여기 1회만.
- **각 티켓 카드 경량화**: 이용자 선택은 **이름 칩 체크만**(생년월일·신장 입력·고지 반복 제거).
  칩에 자동 판정 구분 라벨(성인/어린이 등)+단가를 소형 배지로. 선택 수=수량, 구분별 소계·합계(PR #250) 유지.
- **상태 리프트**: heightCm을 카드별 state → 폼 전역 state(사람 인덱스→heightCm)로. 판정·그룹핑은
  기존 순수 모듈(ticket-variant-logic/rules) 재사용 — 입력 원천만 공유 상태로 교체.
- **제출 payload·서버 불변**: 주문별 ticketGuests(name·birthDate·heightCm)·그룹 분리 그대로. 클라 전용 변경.

## 범위
1. `app/g/_components/guest-options.tsx` — 통합 카드 + 전역 인원 상태(신장 포함)
2. `app/g/_components/option-card.tsx` — 인원 칩 간소화(공유 상태 참조)
3. `app/g/_components/ticket-variant-logic.ts` — 필요 시 시그니처 조정(순수성 유지)
4. `lib/guest-i18n.ts` — 5언어 키(통합 카드 제목·안내 등), 불용 키 정리
5. 테스트: 공유 신장→품목별 판정 일관성, 기존 그룹핑·소계 회귀
