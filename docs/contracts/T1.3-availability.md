## 범위 (In)

1. **lib/availability.ts** — 가용성 판정 단일 소스
   - 순수 함수 층 (DB 무관, 단위 테스트 대상):
     - `overlapsHalfOpen(aStart, aEnd, bStart, bEnd)` — [start, end) half-open 겹침 판정
     - `assertValidStayRange(checkIn, checkOut)` — checkIn < checkOut 검증 (위반 시 throw)
     - `evaluateAvailability(input)` — villa.status, 겹침 예약 수, 겹침 차단 수를 받아 `{ available, sellable, reasons[] }` 반환. sellable = available AND villa.isSellable (검수 게이트)
   - DB 래퍼 층 (Prisma):
     - `checkAvailability(db, villaId, range)` — 단일 빌라 판정. `db`에 PrismaClient 또는 `$transaction` 클라이언트 주입 가능 (T2.3 HOLD 트랜잭션 내 재검증 재사용 목적)
     - `findSellableVillaIds(db, villaIds, range)` — 제안 생성(T2.1)용 일괄 필터
   - 점유 상태 상수: `OCCUPYING_BOOKING_STATUSES = [HOLD, CONFIRMED, CHECKED_IN]`
2. **단위 테스트** — vitest 도입(devDependency + `npm test` 스크립트 + vitest.config.ts)
   - 순수 함수 층 전수 테스트: half-open 경계(체크아웃일 = 다음 체크인일 허용), 역전/동일 날짜 거부, 상태 조합(ACTIVE 아님 / 예약 겹침 / 차단 겹침 / isSellable=false)별 reasons
