# 계약: T1.2 ADMIN 빌라 목록·승인·요율 편집

날짜: 2026-06-11 / 담당: FE(화면)·BE(API) / 평가: QA (독립)

## 범위

1. **공통 사이드바** — `components/admin/sidebar.tsx`: 다크, 9메뉴(대시보드·빌라·예약·제안·메시지·검수·정산·사용자·설정), b9·b10의 사이드바 마크업 기준. <1024px 햄버거 드로어(T6.7). 이후 모든 (admin) 화면이 재사용
2. **`/villas` 목록** (Stitch b9-villas-list 변환) — 상태 필터(전체/승인 대기/운영/중단), 승인 대기 카운트, ResponsiveTable(<768px 카드 — T6.7)
3. **`/villas/[id]` 상세** (Stitch b10-villa-detail 변환) — 기본 정보·사진(공간별)·비품 읽기 전용 카드(T6.4)·시즌 요율 편집(원가는 읽기, 마진·salePriceVnd·salePriceKrw 편집)·승인/반려 액션
4. **API** (BE):
   - `PATCH /api/villas/[id]` — 승인(status: PENDING_REVIEW→ACTIVE)·중단·재개. ADMIN 전용, writeAuditLog
   - ~~반려(REJECT)~~ 보류 — VillaStatus에 반려 상태 없음 + 공급자 수정 화면 미구현. 공급자 빌라 수정 태스크와 함께 설계 (TASKS 등재)
   - `PUT /api/villas/[id]/rates` — 시즌별 VillaRate upsert (marginType/marginValue/salePriceVnd/salePriceKrw). ADMIN 전용, writeAuditLog. 금액은 BigInt(VND)·Int(KRW), 문자열로 직렬화

## 완료 기준 (테스트 가능)

- [ ] ADMIN 로그인 → /villas에서 PENDING_REVIEW 빌라가 "승인 대기" 필터에 보임
