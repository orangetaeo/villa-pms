# 계약: T3.6 여권 Zalo 전달 (임시거주신고)

날짜: 2026-06-16 / 담당: INTEG(+FE 버튼) / 평가: QA (독립)
근거: SPEC F4(129~130행 "공급자에게 전달 → 여권 사진 공급자 Zalo 발송 → tamTruSentAt 기록"), F5 알림표(여권 전달), Stitch b3-checkin(193행 "공급자에게 전달" 버튼·189행 "전달 완료 HH:MM"). 점유: TASKS.md 4423c4f

## 맥락
베트남 법상 외국인 투숙 시 집주인(공급자)이 임시거주신고(tạm trú) 의무. 운영자가 체크인 시 받은 여권 사진을 공급자 Zalo로 전달해 신고를 위임한다. TAMTRU_PASSPORT 빌더(lib/zalo.ts:189)·CheckInRecord.tamTruSentAt·passportPhotoUrls는 이미 존재 — **연결만 하면 됨**.

## 범위

1. **`POST /api/bookings/[id]/tamtru`** (신규, ADMIN 전용 + AuditLog):
   - 전제: CheckInRecord 존재 + passportPhotoUrls 비어있지 않음 (없으면 400)
   - 공급자(villa.supplier) 대상 `enqueueNotification(TAMTRU_PASSPORT, payload: { villaName, guestName, checkIn, passportPhotoUrls })` — 빌더가 읽는 필드 + 첨부 URL
   - CheckInRecord.tamTruSentAt = now 기록 (재전달 허용 — 갱신)
   - 멱등/재전달: 이미 보낸 경우도 재발송 가능(공급자가 못 받았을 때), AuditLog에 기록
2. **Zalo 이미지 첨부**: `dispatchPendingNotifications`/`sendZaloText`가 현재 텍스트 전용. 여권 사진 첨부는 Zalo OA 이미지 발송 API 필요 — **기술 현실성을 INTEG가 판단**:
   - 가능하면: payload.passportPhotoUrls → 이미지 메시지 발송 + ZaloMessage.attachmentUrls 기록
   - Phase 1 토큰 미확보로 복잡하면: 텍스트 알림(빌더 문구) + passportPhotoUrls를 ZaloMessage.attachmentUrls·payload에 기록(전달 증빙) + 실 이미지 발송은 잔여로 명시. **단 tamTruSentAt·알림 큐잉은 반드시 동작**
3. **FE 버튼** (b3-checkin): 체크인 완료된 예약 상세 또는 체크인 화면에 "공급자에게 전달" 버튼 + tamTruSentAt 있으면 "전달 완료 HH:MM" 표시 (디자인 그대로). 클릭 → API → refresh
