## 범위 (In)

1. **lib/ical.ts** — iCal 동기화 단일 소스
   - 순수 함수 층 (DB·네트워크 무관, 단위 테스트 대상):
     - `parseIcs(icsText)` — RFC 5545 파서(의존성 추가 없음 — package.json 동결): 라인 언폴딩(CRLF+공백/탭), VEVENT의 UID·DTSTART·DTEND 추출. `VALUE=DATE`(종일)와 DATE-TIME(`Z`·TZID) 모두 **UTC 자정으로 정규화**(@db.Date·half-open 규약, T1.3 QA 권고). DTEND 누락 시 DTSTART+1일(RFC 5545 종일 규정), DURATION 미지원은 스킵+경고. `STATUS:CANCELLED` 이벤트 제외. UID 누락 이벤트 스킵+경고. endDate ≤ startDate 인 이상 이벤트 스킵+경고. 같은 피드 내 중복 UID는 마지막 이벤트 채택+경고. 반환: `{ events, warnings }`
     - **날짜 결정 기준 (QA 합의 조건 2)**: 날짜는 **이벤트 타임존 기준 로컬 캘린더 날짜**를 취해 UTC 자정 Date로 저장한다. ① `VALUE=DATE`·bare `yyyymmdd` → 그 날짜 그대로 ② TZID·floating DATE-TIME → 값의 날짜부가 곧 로컬 캘린더 날짜 ③ `Z`(UTC 순간) → `Intl.DateTimeFormat`(timeZone=Asia/Ho_Chi_Minh, 빌라 소재 타임존)으로 로컬 날짜 변환 — 의존성 없음. DTEND에 자정이 아닌 시간 성분이 있으면 **올림(ceil, +1일)** — 재고 보수 원칙(언더블록 방지). Intl이 거부하는 미지원 TZID는 이벤트 스킵+경고
     - `diffIcalEvents(existingIcalBlocks, parsedEvents)` — UID 기준 생성/날짜변경(갱신)/소멸(삭제) 분류 — 멱등 동기화의 단일 판정
     - `findEventBookingConflicts(events, occupyingBookings)` — iCal 이벤트와 점유 예약(HOLD/CONFIRMED/CHECKED_IN)의 half-open 겹침 감지 (`overlapsHalfOpen` 재사용) — 더블부킹 경보 소스
   - DB 래퍼 층:
     - `syncVillaIcal(db, villa, fetchFn?)` — villa.icalImportUrls 순회 fetch(타임아웃 15s) → parseIcs → diff → `$transaction`으로 CalendarBlock upsert(source=ICAL, icalUid)·소멸 UID 삭제 → 충돌 감지 결과·통계 반환. fetchFn 주입 가능(테스트용)
     - `runIcalSync(db, fetchFn?)` — status=ACTIVE & icalImportUrls 비어있지 않은 전체 빌라 순회, 빌라 단위 실패 격리(한 빌라 실패가 전체 중단 금지), 요약 반환
     - `findUnresolvedIcalConflicts(db)` — 현재 ICAL 블록과 점유 예약이 겹치는 미해결 충돌 조회 — F7 대시보드 경보 배너(T2.6)가 import할 단일 소스
   - **안전 규칙 (재고 보수 원칙)**:
     - **(QA 합의 조건 1)** CalendarBlock에는 출처 URL 필드가 없으므로 삭제 판정은 URL 단위가 불가능하다. 따라서 **소멸 UID 삭제는 해당 빌라의 모든 icalImportUrls가 fetch+파싱에 성공한 경우에만, 전체 피드 UID 합집합 기준으로 수행**한다. 1개 URL이라도 실패하면 그 빌라는 upsert만 수행하고 삭제는 전부 스킵 (일시 장애로 재고가 열려 더블부킹 되는 사고 차단). 실패는 에러 수집 후 응답·로그로 보고. 서로 다른 피드가 같은 UID를 내면 경고 수집(마지막 채택)
     - 삭제는 해당 빌라의 `source=ICAL` 블록만 — MANUAL 블록 절대 불변
     - 내부 예약과 충돌해도 블록은 생성(점유 우선·보수적) + 충돌 목록 보고 — 해소는 ADMIN 수동(SPEC F2)
     - 변경(생성/갱신/삭제)마다 `writeAuditLog`(userId=null=시스템, entity="CalendarBlock") — 감사 로그 절대 규칙. 트랜잭션 커밋 성공 후 기록(롤백 시 유령 로그 방지)
     - **SSRF·응답 크기 (QA 조건 4)**: http/https 스킴만 허용, 응답 본문 상한 5MB, fetch 타임아웃 15s — 위반 시 해당 URL 실패 처리
     - `findUnresolvedIcalConflicts`는 **ADMIN 전용 소비 전제** — 호출측(T2.6 등)이 role 검사 책임, 함수 주석에 명시
2. **app/api/cron/ical-sync/route.ts** — cron 진입점 (Railway 30분)
