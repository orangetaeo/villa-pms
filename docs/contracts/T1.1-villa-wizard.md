# 계약 — T1.1 SUPPLIER 빌라 등록 마법사 5단계

2026-06-11 / 담당: UX-VN / 평가: QA / 근거: SPEC F1, ADR-0003 결정2, design/stitch a2·a2b·a1·a9·a5

## 범위

`app/(supplier)/my-villas/new` — 5단계 등록 마법사 (모바일 390px 기준, vi 기본):

| 단계 | 디자인 | 내용 | 저장 대상 |
|---|---|---|---|
| 1/5 기본 정보 | a2-basic-info | 빌라명, 단지, 침실/욕실/최대인원 스테퍼, 수영장·조식 토글 | Villa |
| 2/5 위치·참고 | a2b-location-info | 주소, 월 임대 시세(VND) — 전부 선택, 건너뛰기 가능 | Villa.address/monthlyRentVnd |
| 3/5 사진 | a1-photo-upload | 공간별 업로드 그리드(외관·거실·주방·침실N·욕실N·베란다·수영장), 진행 카운터 | VillaPhoto |
| 4/5 비품 | a9-amenities | 카테고리 4탭(주방/욕실/가전/미니바), 체크 타일 + 미니바 수량 스테퍼, 건너뛰기 가능 | VillaAmenity |
| 5/5 원가 | a5-rate-input | 시즌 3종 원가/박(VND), 3개 모두 입력 전 제출 비활성 | VillaRate |

- 제출 API: `POST /api/villas` — SUPPLIER 전용, villa.status=PENDING_REVIEW, `writeAuditLog` 동시 기록
- 사진 업로드 API: `POST /api/uploads` — 로컬/볼륨 디스크 임시 저장(`lib/storage.ts`), T0.4 R2 전환 시 함수 1곳 교체
- i18n: messages/vi.json·ko.json **키 추가만**, 비품 품목 사전 키(`amenities.items.*`) 포함
- globals.css: 배지/버튼/th/tabular-nums `white-space: nowrap` 전역 규칙 추가 (T5.5)
- (supplier) 라우트의 locale을 사용자 locale(vi)로 적용되도록 확인

## 범위 밖 (구현 금지)

- 로그인·인증(T0.3 타 세션): `app/(auth)/`, `auth.ts`, `middleware.ts`, `tailwind.config.ts`, `package.json` 수정 금지
- ADMIN 승인 화면(T1.2), R2 업로드(T0.4), 이미지 압축(sharp 미설치)
- 하단 탭바 완성형(a6 내 빌라 홈은 별도 태스크) — 마법사는 풀스크린 플로우

## 완료 기준 (테스트 가능)

1. SUPPLIER 로그인 후 `/my-villas/new` 진입 → 5단계 순차 진행 → 제출 → DB에 Villa(PENDING_REVIEW)+Photo+Amenity+Rate 생성, AuditLog 기록
2. 비로그인/CLEANER 접근 시 마법사·API 모두 차단(redirect/403). API는 supplierId=세션 사용자 강제(바디의 supplierId 무시)
3. 단계 표기 ICU `Bước {n}/{total}` → "Bước 1/5"~"Bước 5/5", 헤더 전 단계 "Đăng ký villa"
4. VND 입력·표시는 공급자 규칙 **점 구분**(1.500.000) + ₫, KRW·마진·판매가 어디에도 미노출
5. 시즌 3종 미입력 시 제출 버튼 disabled, 입력 시 활성 (a5)
6. 2/5·4/5 "Bỏ qua bước này" 동작, 사진 0장이어도 경고 후 진행 가능(업로드는 T0.4 의존 — 인터림 디스크 저장 동작)
7. "biệt thự" 문자열 0건, 영어 잔존 0건(vi 화면), `npm run lint` + `npx tsc --noEmit` 통과

## 검증 방법

QA가 코드 리뷰 + 권한 누수 체크리스트(leak-checklist) + 빌드·타입 검사. Playwright 프로덕션 검증은 배포 후(로컬 DB 미연결 환경 감안).
