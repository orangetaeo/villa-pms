# T-tutorial-onboarding-9 — 체크인/체크아웃 폼 인라인 안내 (마지막 갭)

- 상태: 착수 (2026-07-10, worktree `worktree-tutorial-onboarding` 계속)
- 배경: 테오 "인터랙티브 폼에도 튜토리얼 넣어야 하는 거 아니야?" — 맞음. 단 형태는 코치마크가 아니라 **인라인 안내**(T-4 마법사 WizardGuide 방식): 오버레이는 여권 촬영·서명 등 작업 중 방해, 폼은 항상 보이는 힌트가 정답. 마법사는 T-4에서 완료 — **체크인/체크아웃 폼 4종이 남은 갭**.

## 범위

### 1. 인라인 가이드 컴포넌트 공용화
- `app/(supplier)/my-villas/new/wizard-guide.tsx` → `components/inline-guide.tsx`로 승격(공용), `variant: "light" | "dark"` 추가(관리자 다크 폼 대응 — 기존 teal 라이트 + 다크 톤). 마법사 3곳 import 경로 갱신(스타일 무변화).
- 원칙 유지: 화면당(또는 섹션당 1개) ≤2문장·아이콘 고정·닫기 없음.

### 2. 폼 4종에 섹션별 인라인 안내 (회의에서 대상 섹션·문구 확정)
| 폼 | 후보 섹션 |
|---|---|
| 관리자 체크인 (/bookings/[id]/checkin) | 여권 업로드+OCR(자동 인식·명단 대조)·보증금·동의서 서명(서명 후 wifi 노출 규약) |
| 관리자 체크아웃 (/bookings/[id]/checkout) | 사진 비교(기준 대비)·파손 차감·미니바 사용량(수량=사용분)·정산 확정(청소 태스크 자동 생성) |
| 공급자 체크인 (/my-bookings/[id]/checkin, vi) | 여권 촬영(자동 읽기)·보증금·서명 |
| 공급자 체크아웃 (/my-bookings/[id]/checkout, vi) | 현재 사진 촬영(기준 비교)·파손·미니바·환불액 |
- 안내 원칙: "지금 뭘 + 왜"(업무 규칙 포함 — 예: "서명이 끝나야 게스트에게 와이파이가 보입니다", "체크아웃하면 청소 대기가 자동 생성됩니다"). 폼당 배너 2~3개 상한(과다 금지 — 회의 판단).
- 금액 수치·판매가·마진 무참조. 공급자 폼은 vi 명령형.

## 완료 기준
1. 4폼 각각 확정 섹션에 인라인 가이드 렌더(관리자=다크 변형·공급자=라이트), 기존 폼 동작·유효성 무영향.
2. 마법사 기존 3곳 가이드 시각 회귀 없음(공용화 후 동일 렌더).
3. i18n ko/vi 동시(관리자 폼도 vi 필수 규칙). 화이트리스트: 문구는 RSC 번역→props(기존 labels 패턴 재사용 — 화이트리스트 무변경).
4. tsc·build·전체 vitest. 투어 31종 회귀 없음.
