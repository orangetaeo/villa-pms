# T-guest-vnd-display — 게스트 화면 금액 표기 VND(₫) 통일

- 상태: 완료 (2026-07-13) — QA PASS(결함 0). tsc0·vitest 3011·build 통과. 초과 수납(환불 예정) 행 동반 추가
- 담당: TDA(방침) + FE(게스트 화면 전수), QA 검증
- 배경 (테오 지시): "소비자 화면의 금액 표기는 베트남 동으로 해줘. 그래야 다국적 사람들을 한번에 커버 가능."
  현재 게스트 화면(/g/*)은 한국 고객 배려로 서비스 가격을 KRW 환산(주문 시점/표시 시점 priceKrwCeil)으로
  주 표기하는 곳이 많음. 다국적(러·중·베 등) 고객에겐 ₩ 표기가 무의미하고, 실제 결제 통화도 현지(₫).

## 방침 (TDA)

1. **게스트 대면(/g/*) 화면의 서비스·미니바·청구 금액 = VND(₫) 원천 단일 표기.** KRW 환산 병기·주 표기 제거.
   - 대상: 부가옵션 신청 페이지(카탈로그 가격·티켓 구분 소계·합계), 신청 내역(orders), 정산 영수증(receipt)의
     서비스 라인·총 이용 금액(₫만 — "환산 합계 ≈" 행도 KRW-원천 청구가 0이면 자연 소멸), 기타 /g 하위 금액 표기 전수.
2. **원천이 진짜 KRW인 데이터는 그 통화 유지**(왜곡 금지): ₩ 수납 라인(결제 내역), ₩ 보증금(수취·상계·환불액),
   KRW-원천 주문(priceVnd null — 현 데이터엔 거의 없음), DIRECT(KRW 채널) 숙박요금 표시.
3. 운영자(admin)·벤더 화면은 변경 없음(이번 범위 밖).
4. 서버 데이터·payload 무변경 — 표시 계층만. priceKrw 스냅샷 저장은 유지(운영자 참고용).

## 완료 기준
1. /g/* 전수 grep: 서비스 금액의 KRW 환산 주/병기 표기 0 (원천 KRW 데이터 제외).
2. 영수증: 서비스 라인·총 이용 = ₫만. 결제 내역·보증금의 ₩(원천)은 유지.
3. 신청 페이지: 가격·소계·합계 ₫. 언어 5종에서 통화 기호 일관(₫).
4. tsc·vitest·build 통과, 게스트 누수 무회귀.
