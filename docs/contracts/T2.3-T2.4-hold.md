## 범위 (In)

1. **lib/hold.ts** — HOLD 수명주기 단일 소스
   - 순수 함수 층 (단위 테스트 대상):
     - `resolveHoldHours(settingValue, override?)` — 우선순위: override(제안별 24/48h) > AppSetting `HOLD_HOURS_DEFAULT` > 기본 48. 1~168 범위 밖·비정수 거부
     - `computeHoldExpiresAt(now, hours)`
     - `evaluateProposalForHold({proposalStatus, proposalExpiresAt, itemBookingId, now})` — ACTIVE·미만료·미사용 item 검증, 사유 코드 반환
   - DB 층:
     - `createHoldFromProposalItem(prisma, input)` — **단일 $transaction**: ① `pg_advisory_xact_lock(hashtext(villaId))` 빌라 단위 잠금 ② 제안 검증 ③ `checkAvailability(tx)` 재검증 — sellable 아니면 거부("마감되었습니다" 사유, SPEC 엣지: 더블부킹 최종 방어선) ④ 판매가 = **ProposalItem 스냅샷 복사**(고객이 본 가격, `assertSaleAmountColumns` 검증) + 원가 = `quoteStayForVilla(tx)` HOLD 시점 합산 + fx = Proposal 복사 ⑤ Booking(HOLD, holdExpiresAt) 생성 ⑥ ProposalItem.bookingId 연결 + Proposal→USED ⑦ 공급자 Notification(BOOKING_HOLD, PENDING) 큐 적재 — **payload에 판매가·마진 미포함**(마진 비공개) ⑧ AuditLog(tx)
     - `expireHolds(prisma, now)` — holdExpiresAt 경과 HOLD → EXPIRED (개별 tx + `updateMany status=HOLD` 가드로 확정 경합 차단), 공급자 Notification(HOLD_EXPIRED) + AuditLog, 요약 반환
     - `confirmHold(prisma, {bookingId, actorUserId, now})` — HOLD→CONFIRMED. 만료 시각 경과 시 거부(cron 선행 원칙). Notification(BOOKING_CONFIRMED) + AuditLog
     - `cancelBooking(prisma, {bookingId, cancelReason, actorUserId})` — CANCELLED 전환, cancelReason 필수(SPEC)
2. **app/api/cron/expire-holds/route.ts** — `Bearer ${CRON_SECRET}` 인증(ical-sync와 동일 패턴, 미설정 시 500), expireHolds 호출, 요약 응답. SPEC 주기 5분(Railway cron 등록은 OPS 잔여)
3. **app/api/bookings/[id]/confirm·cancel/route.ts** — POST, 첫 줄 `auth()` ADMIN 검사(api-pattern), zod 입력 검증(cancel: cancelReason min 1), `serializeBigInt` 응답
4. **단위 테스트** (lib/hold.test.ts) — 순수 함수 층 전수: 홀드 시간 우선순위·범위 검증, 만료 시각 계산, 제안 상태별 거부 사유

## 합의 편차 (QA 판단 요청)

- **HOLD→CANCELLED 허용**: SPEC은 CONFIRMED→CANCELLED만 명시하나, ADMIN이 착오 홀드를 만료 전 즉시 해제할 운영 수단이 없어 HOLD 취소도 허용(재고 즉시 복귀, cancelReason 필수, ADMIN 전용). 거부 시 CONFIRMED만으로 축소
- Proposal에 홀드 시간 필드가 없음(schema) — "제안 생성 시 24h/48h 선택"은 T2.1에서 createHold의 `holdHours` override 인자로 전달하는 설계로 충족 (스키마 변경 회피, 규칙 #6)
