## 계약 필수 결정 (T3.1 인계 메모 3 — QA 판정 요청)

1. **서명 게이트화: 비게이트 유지** — T3.1 조건 C에서 무서명 CHECKED_IN이 이미 승인·운영 중. 강제 게이트는 현장 예외(고객 거부·기기 문제) 시 체크인 자체를 차단. 대신 ① 체크인 폼에 동의서·서명 단계 표준 노출(권장 흐름) ② **미서명 CHECKED_IN은 사후 서명 경로 제공**(아래) ③ 예약 상세·체크인 화면에 미서명 배지
2. **무서명 레코드 소급 처리: 사후 서명 경로로 해소** — `/bookings/[id]/checkin`이 CHECKED_IN + 미서명일 때 동의서·서명만 남긴 "사후 서명 모드" 렌더 → POST agreement
3. **서명 저장은 비공개 증빙 파이프라인 재사용** — 서명도 개인정보(필적). T3.1 조건 A의 `savePassportFile`+`/api/passports/[name]`(ADMIN 가드, private·no-store)를 그대로 사용, 파일명 `sig-` 접두로 구분. 신규 인프라 0
4. **T3.3 핫픽스 (인계 메모 1)**: `resolveDepositOutcome`에 현재 depositStatus 반영 — **NONE(미수취)이면 NONE 유지**(REFUNDED 둔갑 차단). NONE+파손은 deductionVnd를 청구 근거로 기록 허용하되 보증금 상태는 NONE
