# T-seo-media-library — 가이드 글용 자료 사진 라이브러리

## 배경

가이드 글(빌라 글이 아닌 주제 글)은 **의도적으로 본문 이미지가 없다**
(`app/api/cron/seo-draft/route.ts` §② 주석 — "빌라 사진은 본문 주제와 무관하다", 테오 지적 2026-07-22).
그 결과 가이드 글은 텍스트만 800자 이상인 글이 되고, 이미지 검색 유입·체류시간·공유 CTR을 모두 놓친다.

해결은 **빌라 사진이 아닌, 운영자가 직접 촬영한 자료 사진 저장소**를 따로 두는 것이다.
저작권은 자사 촬영본만 올려 해결한다(외부 이미지 업로드는 정책적으로 막지 않되 credit 필드로 출처를 남긴다).

## 설계 결정

1. **빌라 글은 건드리지 않는다.** 빌라 글은 이미 그 빌라의 실사진을 쓴다(주제-사진 일치). 이 계약은 가이드 글 경로만.
2. **주제 태그 = `ARTICLE_TOPICS`의 key 다중선택.** 빈 배열 = 범용(어느 주제에나 후보).
   주제 일치분을 먼저 쓰고, 모자라면 범용에서 채운다.
3. **덜 쓴 사진 우선**(`usedCount` asc). 같은 사진이 모든 글에 반복되면 그 자체가 저품질 신호다.
4. **alt 필수.** alt 없는 이미지는 파서(`parseArticleBody`)가 블록째 버리므로 애초에 저장을 막는다.
5. **라이브러리가 비어 있어도 글은 그대로 써진다.** 사진은 선택 재료 — 0장이면 현재와 동일하게 브랜드 커버 + 텍스트 본문.
6. **삭제하지 않는다.** `active=false`로 내린다(이미 발행된 글의 본문 URL이 죽으면 안 된다).
7. **업로드는 기존 `/api/uploads` 재사용.** 서버 액션 body 1MB 제한을 피하고, R2/디스크 자동 선택(ADR-0004)을 그대로 탄다.

## 완료 기준 (테스트 가능)

1. `/marketing/seo/media`에서 사진 업로드 → alt·주제 태그 지정 → 목록에 뜬다 (운영자 권한 필요, 비운영자 302)
2. alt 없이 저장 시도 → 400/폼 오류, 저장되지 않는다
3. `seo-draft` 실행 시 가이드 글에 라이브러리 사진이 커버 + 본문 이미지로 들어간다 (AuditLog `photos` 수 기록)
4. 라이브러리가 비면 기존과 동일하게 동작한다(브랜드 커버, 본문 이미지 0)
5. 같은 주제로 두 번 뽑으면 서로 다른 사진이 우선 선택된다(usedCount 반영)
6. 이미지 블록은 분량 계산에 포함되지 않는다 — 800자 하한 우회 불가 (기존 동작 유지 확인)
7. `npm run lint && npm run typecheck && npx vitest run tests/seo-media.test.ts` 통과
