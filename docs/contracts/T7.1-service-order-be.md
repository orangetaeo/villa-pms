# 계약: T7.1 부가서비스(ServiceOrder) 판매 — 백엔드 (Phase 2 착수)

## 배경
테오님 승인으로 Phase 2 착수. 스키마에 `ServiceOrder`·`ServiceType`(BBQ/TICKET/GUIDE/CAR_RENTAL/BREAKFAST)·
`ServiceOrderStatus` 완비, `Booking.serviceOrders` 역관계 존재 → **스키마 변경 불필요**.
신규 UI는 Stitch 디자인 선행 규칙에 묶이므로(메모리), 디자인 불필요한 **BE(lib+API)부터** 착수
(프로젝트 BE 선행 패턴 — T2.1 등). UI는 디자인 확보 후 후속(T7.2).

## 마진 비공개 (절대 규칙)
- 부가서비스는 원가(costVnd)·판매가(priceKrw)·마진을 다루므로 **ADMIN 전용**. SUPPLIER·공개 노출 절대 금지.
- 모든 라우트 첫 줄 role 검사(ADMIN 외 401/403). StayQuote 누수 규칙과 동일.

## 완료 기준 (테스트 가능)
1. POST 생성: type·costVnd(BigInt≥0)·priceKrw(Int≥0)·serviceDate(옵션)·vendor/note 검증, REQUESTED 기본, AuditLog
2. PATCH 전이: 전이표 위반 409, CANCELLED/DELIVERED 종결, AuditLog diff
3. GET: 해당 booking의 서비스 목록(ADMIN) — BigInt 직렬화(serializeBigInt)
4. 권한: 비ADMIN/비로그인 전부 401/403, 존재하지 않는 booking/service 404
5. 마진: computeServiceMarginKrw(costVnd, priceKrw, fx) = priceKrw − VND→KRW(costVnd), 환율 형식 검증
6. `npx vitest run lib/service-order.test.ts` 통과, `npm run typecheck` 내 파일 0 에러
7. 누수 0 — ADMIN 외 응답 차단
