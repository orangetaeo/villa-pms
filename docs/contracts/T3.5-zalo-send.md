# 계약: T3.5 Zalo OA 발송 + Notification 로그 + 재시도 cron

날짜: 2026-06-11 / 담당: INTEG / 평가: QA (독립)
근거: SPEC F5, schema Notification(type 9종·status PENDING/SENT/FAILED)·ZaloMessage(SYSTEM 미러 — ADR-0003)

## 범위

1. **`lib/zalo.ts`** — Zalo OA 메시지 발송 단일 소스:
   - `enqueueNotification(params)`: Notification(PENDING) 생성 — 비즈니스 로직이 호출하는 유일한 진입점 (발송 실패가 본 트랜잭션을 깨지 않도록 큐잉 분리)
   - `dispatchPendingNotifications(limit)`: PENDING(+재시도 대상 FAILED) 배치 발송 — zaloUserId 없는 사용자 SKIP 기록, 토큰 미설정 시 FAILED(error="ZALO_TOKEN_NOT_SET")로 기록만(크래시 금지)
   - 발송 성공 시 ZaloMessage(SYSTEM, OUTBOUND) 미러 기록 (ADR-0003 — ZaloConversation 있으면 연결)
   - 알림 본문 vi 템플릿: NotificationType별 메시지 빌더 (LOC 용어 사전 .claude/skills/loc/i18n-pattern.md 준수)
2. **`app/api/cron/notifications/route.ts`** — CRON_SECRET Bearer, dispatchPendingNotifications 호출. 재시도 정책: FAILED 최대 3회(payload에 attempt 기록), 그 이상은 영구 FAILED
3. 단위 테스트(vitest): 템플릿 빌더·재시도 판정 순수 로직

## 완료 기준

- [ ] enqueue → Notification PENDING 생성, cron 호출 → 토큰 미설정 환경에서 FAILED(ZALO_TOKEN_NOT_SET) 기록·크래시 없음
- [ ] zaloUserId null 사용자 → 발송 시도 없이 FAILED(NO_ZALO_LINK), 재시도 대상에서 제외
- [ ] 재시도 3회 초과 → 더 이상 시도 안 함
- [ ] cron 무토큰 401, vitest 통과, tsc·build 통과
- [ ] 실발송 검증은 ZALO_OA_ACCESS_TOKEN 입력 후 별도 수행 (잔여로 기록)
