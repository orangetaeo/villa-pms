# T6.7 — 관리자 반응형 마무리 계약 (Ralph Loop)

## 배경

핵심 인프라는 선행 세션에서 이미 구현됨:
- `<1024px` 햄버거 헤더 + 드로어: `components/admin/sidebar.tsx`
- `<768px` 카드 전환 공통 컴포넌트: `components/admin/responsive-table.tsx`
- 적용 완료: b5(`bookings/page.tsx`)·b12(`proposals/proposals-list.tsx`)·b13(`users/users-manager.tsx`)
- b9(`villas/page.tsx`)는 원래 반응형 카드 그리드(테이블 아님 — 적용 불요)

따라서 본 태스크의 잔여는 **전수 감사 + 360px 폭 잔여 결함 수정 + 회귀 없음 확정**이다.

## 범위 (이 계약이 건드릴 수 있는 파일 — 화이트리스트)

- `app/(admin)/**` 중 **목록·표 렌더 부분만** (반응형 클래스 조정)
- `components/admin/responsive-table.tsx` (필요 시 — 단 기존 시그니처 보존)
- `messages/ko.json`·`messages/vi.json` (키 **추가만**, 기존 키 수정 금지)
- `docs/contracts/T6.7-admin-responsive.md`, `PROGRESS.md`·`TASKS.md`(자기 행만)

## 수정 금지 구역 (다른 세션 점유 — 절대 건드리지 않음)

- `app/(admin)/villas/[id]/villa-actions.tsx` 및 빌라 반려(T1.2b) 관련 파일
- `app/(supplier)/my-villas/**` (T1.10·T6.4)
- `app/(admin)/messages/**`, `app/api/zalo/**`, `lib/zalo-chat.ts`, `lib/gemini.ts` (T6.6/T3.7)
- `prisma/schema.prisma`, `lib/hold.ts`, `lib/cleaning.ts`, `lib/proposal.ts` (타 세션 변경 중)
- `app/api/villas/**`

## 테스트 가능한 완료 기준

1. **드로어**: 모든 `(admin)` 라우트에서 `<1024px` 햄버거 헤더 + 드로어가 동작(`sidebar.tsx` 경로 변경 시 자동 닫힘 유지). 회귀 없음.
2. **목록 카드 전환**: b5·b9·b12·b13 목록 화면이 `<768px`에서 가로 스크롤 없이 카드/1열로 렌더(코드 검증: 목록 표는 `ResponsiveTable` 사용 또는 반응형 그리드).
3. **잔여 가로 넘침 점검**: `(admin)` 내 **목록성 표** 중 `<768px`에서 의도치 않게 잘리는 것이 없음. 상세/특수 표(`bookings/[id]` 결제 기록, `checkout-form` 미니바, `settlements-view`, `season-manager`, `rate-editor`)는 `overflow-x-auto`로 가로 스크롤 허용 — 의도된 동작이므로 변경 대상 아님(발견 시 계약에 명시만).
4. **i18n 혼입 0**: 새로 추가한 텍스트는 ko/vi 양쪽 키 등재. 하드코딩 한국어/영어 라벨 없음.
5. `npm run typecheck` 통과 (에러 0).
6. `npm run lint` 통과 (에러 0; 기존 경고 수준 유지).
7. `npm test` 전체 통과 (기존 스위트 회귀 0).

## 검증 방법 (Ralph Loop 자가 검증 — dev 서버 불요)

- 코드 정적 감사: `(admin)` 목록 화면의 반응형 클래스/`ResponsiveTable` 사용 여부 grep + 읽기
- `npm run typecheck && npm run lint && npm test` 3종 통과를 완료 신호로 사용
- 라이브 Playwright 360×800 검증은 **QA 후속**(dev 서버 단일 세션 규칙 — 본 루프는 정적+빌드 검증까지만)
