## 범위 (In)

1. **lib/pricing.ts** — 가격 계산 단일 소스
   - 순수 함수 층 (DB 무관, 단위 테스트 대상):
     - `resolveSeason(date, periods)` — 시즌 판정. 구간은 [startDate, endDate) half-open (프로젝트 날짜 규약 통일). 미등록 날짜 → LOW. 겹침 시 PEAK > HIGH > LOW 우선
     - `quoteStay({checkIn, checkOut, saleCurrency, rates, seasonPeriods})` — **박별 합산**: 각 숙박일의 시즌 요율을 1박씩 더함 (시즌 경계 걸친 예약은 박마다 다른 요율). 반환: nights, 박별 내역(날짜·시즌·판매가·원가), saleCurrency 해당 통화 총액(`totalSaleKrw: number` 또는 `totalSaleVnd: bigint` — 한쪽만), `totalSupplierCostVnd: bigint`. 해당 시즌 VillaRate 누락 시 throw
     - `computeSalePriceVnd(supplierCostVnd, marginType, marginValue)` — 마진 자동계산 (PERCENT: 원가×(100+%)÷100 BigInt 내림 / FIXED_VND: 원가+고정액). T1.2 요율 편집의 제안값 소스
     - `suggestSalePriceKrw(salePriceVnd, fxVndPerKrw)` — VND→KRW 환산 제안 (환율 문자열을 1e4 스케일 BigInt로 파싱, float 연산 금지, 반올림). ADMIN 라운딩 오버라이드 전 제안값
     - `assertSaleAmountColumns(saleCurrency, {krw, vnd})` — 듀얼 컬럼 검증 (SPEC: "saleCurrency에 해당하는 컬럼만 필수 — lib/pricing.ts에서 검증"). KRW⇒krw 필수·vnd 금지, VND⇒반대
   - DB 래퍼 층:
     - `quoteStayForVilla(db, villaId, range, saleCurrency)` — VillaRate·SeasonPeriod 로드 후 quoteStay 호출. DbClient(트랜잭션 클라이언트) 주입 가능 — T2.3 HOLD 스냅샷 재사용
     - `getFxVndPerKrw(db)` — AppSetting `FX_VND_PER_KRW` 조회 (미설정 시 null — 제안 생성 시 ADMIN에게 입력 요구)
2. **단위 테스트** (vitest, lib/pricing.test.ts) — 순수 함수 층 전수: 시즌 경계 걸친 박별 합산, LOW 폴백, 통화 분기(KRW number/VND bigint), 마진 2종, 환산 반올림, 듀얼 컬럼 검증 위반 케이스
