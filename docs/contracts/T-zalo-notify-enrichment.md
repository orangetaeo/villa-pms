# T-zalo-notify-enrichment — Zalo 알림 내용 보강 + 공유 문구 리치화

> 착수: 2026-07-09. 담당 세션: zalo-notify-enrichment worktree.
> 배경: 테오가 NOTIFICATIONS.md 전수 검토 후 "받는 사람이 다음 행동을 하는 데 필요한 정보" 기준 보강 승인.

## 범위

### ① 자동 알림 보강 (lib/zalo.ts buildNotificationText + 호출부 payload)
- A-01 직접예약: +공급자명, 예약자명, 예약번호
- A-02/03 빌라 승인대기·재제출: +단지(지역), 침실·욕실·정원, 사진 수
- A-05 명단 미입력: +판매 채널(파트너명·연락처)
- A-06 원가 변경: 베트남어→한국어 전환 + 빌라명·변경 전→후 원가·영향 제안 건수
- A-07~10 벤더 응답: +서비스 날짜·시간, 수량, 발주 금액(원가)
- B-01/02 홀드·확정: +예약번호, 인원 구성, 조식 여부, 게스트 요청사항(있을 때만)
- B-05 예약 변경: 변경 전→후 비교 표기
- C-01 청소 요청: +다음 체크인 날짜, 체크아웃 시각
- C-05 청소 반려: +문제 슬롯(공간)명
- D-01 발주: +이행 시간, 옵션 상세, 정산 예정 금액(벤더 자기 원가)
- E-01 파트너 확정: +예약번호, 금액, 잔금 납기 (lib/partner-notify.ts)

### ② 빌라 공유 기간 병기 (lib/zalo-share.ts + share route)
- 시즌 대표행 1개 → 전체 기간 나열: "기본: X / 성수기 (7.15~8.31): Y / 극성수기 · 2026 설 (2.5~2.11): Z"
- isBase 행은 "기본"으로, 기간 행은 label(있으면)+날짜 병기

### ③ 제안서 공유 보강 (lib/zalo-share.ts + share route)
- 빌라 수·기간·박수 + 빌라별 이름·침실·수영장·총액(판매가) + 링크 + 유효기간

### 문서
- docs/NOTIFICATIONS.md 동시 갱신 (문구 진실원천 = 코드)

## 완료 기준 (테스트 가능)
1. 기존 테스트 전부 통과 + 변경 빌더 단위테스트 갱신
2. 보안 불변식 유지: 공급자 문구에 판매가·KRW·마진·파트너명 없음 / 고객 문구에 원가 없음 / 벤더는 자기 원가만 — zalo-share의 타입 차단 구조 유지
3. NOTIFICATIONS.md ID·문구 코드와 일치
4. next build 통과
