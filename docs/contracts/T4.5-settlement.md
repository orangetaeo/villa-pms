# 계약: T4.5 F6 최소 정산 — ADMIN 월 집계·PAID + SUPPLIER 수익 조회

날짜: 2026-06-11 / 담당: BE(lib·API)·FE(b7)·UX-VN(a7) / 평가: QA (독립)
근거: SPEC F6, Stitch b7-settlements(다크 ko)·a7-my-earnings(라이트 vi). 점유 선언: TASKS.md 7e281f3

## 범위

1. **`lib/settlement.ts`** (BE):
   - `generateMonthlySettlements(yearMonth)`: 해당 월(체크아웃일 기준)의 CHECKED_OUT·NO_SHOW 예약을 공급자별 supplierCostVnd 합계로 Settlement(DRAFT)+SettlementItem 생성. 멱등 — 기존 DRAFT는 재집계(upsert), CONFIRMED/PAID는 불변(skip 보고). bookingId unique로 중복 방지
   - 상태 전이: DRAFT→CONFIRMED→PAID(paidAt). PAID 시 `enqueueNotification(SETTLEMENT_READY)` (T3.5 모듈)
2. **API** (ADMIN 전용 + AuditLog): `POST /api/settlements/generate {yearMonth}`, `GET /api/settlements?yearMonth=`, `PATCH /api/settlements/[id] {action: CONFIRM|MARK_PAID}` (전이표 가드 409)
3. **`/settlements`** (FE, b7 변환): 월 선택, 공급자별 정산 카드/행(합계 VND·예약 건수·상태), 집계 실행, 상세 내역(예약별 원가), CONFIRM·PAID 액션. 매출 요약은 통화별 분리(KRW/VND 합산 금지)
4. **`/earnings`** (UX-VN, a7 변환 — 기존 placeholder 교체, vi 라이트 모바일): 자기 빌라 월별 원가 합계 + 예약별 내역(빌라명·기간·원가·정산 상태). **판매가·마진·고객 연락처 미노출** (고객명은 SPEC "고객 상세 미노출" — a7 디자인 표기 따르되 식별 최소화)

## 완료 기준

- [ ] DRAFT→CONFIRMED→PAID 전이 + paidAt + AuditLog + Notification(SETTLEMENT_READY) enqueue. 역방향·건너뛰기 409
