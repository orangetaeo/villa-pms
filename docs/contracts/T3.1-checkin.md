## 범위 (In)

1. **lib/ai-utils.ts** — `extractJsonFromAIResponse` 5단계 추출기, `// [SHARED-MODULE] from _shared/templates/utility-modules.md v1.x` 표기 + 단위 테스트(전체 JSON·코드블록·balanced braces·실패 null)
2. **lib/gemini.ts (INTEG)** — Gemini REST(fetch, 의존성 0): `ocrPassport(imageBase64, mimeType)` → 구조화 프롬프트(JSON 스키마 지시) → extractJsonFromAIResponse → `{ surname, givenNames, passportNo, nationality, birthDate, expiryDate, sex }`(전부 string|null — 모델 출력 신뢰 금지, zod로 정형화). `GEMINI_API_KEY` 미설정 시 `GeminiNotConfiguredError`(UI 수동 입력 폴백). 타임아웃 30s, 응답 비JSON·필드 누락 시 null 필드 허용(부분 추출). 모델: gemini-2.0-flash (비전·저비용)
3. **POST /api/ocr/passport** — ADMIN 전용(401/403 분리). body `{ imageBase64, mimeType(jpeg|png|webp 화이트리스트) }`, 크기 상한(base64 8MB). Gemini 미설정 → 503 `{ error: "ocr_not_configured" }`(폴백 신호). **OCR 결과는 자동 저장하지 않음** — 화면에서 ADMIN이 확인·수정 후 체크인 완료에 포함(오인식 방어)
4. **lib/checkin.ts (BE)** — `completeCheckIn(prisma, input)` 단일 트랜잭션: ① Booking **CONFIRMED→CHECKED_IN** status 가드 updateMany(T2.3 패턴 — 동시성 안전) ② CheckInRecord 생성(passportPhotoUrls ≥1, passportOcrJson(ADMIN 확정본), notes, createdBy) ③ 보증금 기록: 입력 시 depositAmount(Int)+depositCurrency 필수 쌍 → depositStatus=HELD / 미수취 선택 시 NONE 유지 ④ writeAuditLog(status·deposit changes). 순수층 `assertCheckInInput`(여권 사진 ≥1, 보증금 쌍 검증, 통화 화이트리스트 KRW|VND|USD, **depositAmount 1~2,147,483,647 상한 — Int 오버플로 방어, QA 권고 3**)
5. **POST /api/bookings/[id]/checkin** — ADMIN 전용, zod, 404/409(전이 불가 — checked-in 아님·이미 체크인) 분리, serializeBigInt 응답
6. **여권 사진 전용 비공개 파이프라인 (QA 합의 조건 A — 공개 /api/uploads 미사용)**:
   - `lib/storage.ts`에 **추가만**(공유 파일 선언 — 커밋 확인됨·활성 세션 없음): `savePassportFile(buffer, mime, uploaderId)` — **R2 설정과 무관하게 항상 디스크(volume)의 `passports/` 하위 디렉터리** 저장(공개 CDN·공개 버킷 비대상 — R2 전환 시에도 여권은 비공개 유지), 반환은 파일명만. `getPassportDir()`
   - `POST /api/uploads/passport` — **ADMIN 전용**, mime 화이트리스트·5MB, 응답 `{ url: "/api/passports/<name>" }`
   - `GET /api/passports/[name]` — **첫 줄 ADMIN 검사**, `Cache-Control: private, no-store`(프록시·브라우저 캐시 잔존 차단 — 90일 삭제 정책 정합), nosniff, 경로 탈출 차단. CheckInRecord.passportPhotoUrls에는 이 경로만 저장
7. **b3 변환**: `app/(admin)/bookings/[id]/checkin/page.tsx`(RSC — booking 로드, CONFIRMED 아니면 상세로 redirect) + client 폼: 여권 사진 업로드(상기 비공개 파이프라인 + 클라 리사이즈 재사용, 인원수만큼 여러 장) → 장별 "OCR 인식" 버튼 → 결과 폼 채움(수정 가능) → 보증금(금액·통화, 수취 안 함 토글) → 체크인 완료 → 상세로 이동. OCR 503 시 수동 입력 안내 배너. **b3의 범위 밖 섹션 처리(QA 합의 조건 B)**: 동의서·서명 패드 섹션(T3.2)과 "공급자에게 전달" 버튼·"전달 완료" 배지(T3.6)는 **미렌더** — 가짜 완료 상태 표시 금지, 각 태스크가 자기 섹션을 추가
