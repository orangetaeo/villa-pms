# 계약 — T6.3-rates-api-test: ADMIN 요율(마진·판매가) PUT 엔드포인트 테스트

## 배경
운영자 마진·판매가 설정 경로 `PUT /api/villas/[id]/rates`는 사업 핵심 원칙 2(**마진 비공개 — ADMIN 전용**)의 핵심 표면이나, 라우트 레벨 자동 테스트가 부재(가격 *수식*은 lib/pricing.test.ts가 커버하나 API 계약은 미커버). 누수·권한·영속·증빙 회귀를 막기 위해 단위 테스트를 추가한다.

## 범위 (신규 파일 1개만)
- **신규**: `tests/villa-rates-api.test.ts` — `PUT /api/villas/[id]/rates` 단위 테스트 (auth·prisma·audit-log mock, villa-reject-api.test.ts 패턴 재사용)
- 제품 코드 변경 없음 (테스트 전용). 결함 발견 시에만 `app/api/villas/[id]/rates/route.ts` 수정 — 발견 시 계약에 명시.

## 완료 기준 (테스트 가능)
1. **권한 누수 가드**: 비로그인 401 / SUPPLIER 403 / CLEANER 403 (마진은 운영자 전용)
2. **입력 검증 400**: JSON 파싱 실패(INVALID_BODY), salePriceKrw 음수, marginValue 비숫자, 중복 시즌, 빈 배열, 4개 초과(VALIDATION_FAILED)
3. **미존재 빌라 404**
4. **성공 영속**: 존재 시즌은 `villaRate.update`로 marginType·marginValue·salePriceVnd·salePriceKrw 저장, `supplierCostVnd`는 data에서 제외(공급자 입력 영역 — 운영자 수정 차단 불변식)
5. **skipped**: VillaRate 레코드 없는 시즌은 생성하지 않고 응답 `skipped`에 표기
6. **AuditLog 증빙**: 갱신 시즌마다 `writeAuditLog(entity:"VillaRate", action:"UPDATE")` 호출 + changes에 old/new diff 포함
7. **응답 누수 0**: 응답 본문에 BigInt/원가/마진 값 미포함 — `{updated, skipped}` 시즌 키만
8. `npm run typecheck` 0 + `npx vitest run` 기존 회귀 0
