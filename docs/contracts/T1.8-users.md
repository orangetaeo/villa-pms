# 계약: T1.8 ADMIN 사용자 목록 — Zalo 연결·비활성화

날짜: 2026-06-11 / 담당: FE(화면)·BE(API) / 평가: QA (독립)
근거: SPEC F0 (자가 가입·Zalo 연결·비활성화), Stitch b13-users

## 범위

1. **`/users`** (Stitch b13-users 변환, ADMIN 다크):
   - 사용자 목록 — 역할 뱃지(ADMIN/SUPPLIER/CLEANER), Zalo 연결 상태 뱃지(연결/미연결), 활성 상태, 가입일, 빌라 수(SUPPLIER)
   - 비활성화/활성화 토글 (스팸·이탈 대응 — SPEC F0)
   - Zalo 수동 매칭: 미연결 사용자에 대해 userId 없는 ZaloConversation(미가입 팔로워) 선택 → User.zaloUserId 연결. 데이터 없으면(웹훅 T3.7 전) 빈 상태 안내
2. **API** (ADMIN 전용 + writeAuditLog):
   - `GET /api/users` — 목록 (passwordHash 등 민감 필드 select 제외, role 필터)
   - `PATCH /api/users/[id]` — `{ action: "DEACTIVATE" | "ACTIVATE" }` 또는 `{ action: "LINK_ZALO", zaloUserId }` / `{ action: "UNLINK_ZALO" }`
     - 자기 자신(ADMIN 본인) 비활성화 금지 (락아웃 방지) → 400
     - LINK_ZALO: zaloUserId 중복(타 사용자에 연결됨) → 409. ZaloConversation 있으면 userId 동기화
