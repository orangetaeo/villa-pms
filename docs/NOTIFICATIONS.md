# NOTIFICATIONS.md — 알림 발송 내역서 (전수)

> **용도**: 시스템이 보내는 모든 알림의 트리거·수신자·문구를 한 곳에 정리한 검토용 문서.
> 문구 수정을 원하면 **알림 번호(예: A-03)와 바꿀 문구**만 알려주면 된다.
> 최종 갱신: 2026-07-09 (T-zalo-notify-enrichment — 역할별 정보 보강·A-06 한국어 전환). 문구의 진실원천은 코드 — 수정 시 이 문서도 함께 갱신할 것.
> 문구 위치: Zalo 큐 알림=`lib/zalo.ts` buildNotificationText / 파트너=`lib/partner-notify.ts` / 벤더 인앱=`lib/inapp-notification.ts`

## 발송 구조 한눈에

- **Zalo 큐 알림**: 이벤트 발생 → Notification(PENDING) 적재 → cron이 배치 발송(실패 시 최대 3회 재시도, Zalo 미연결자는 영구 실패 처리). 발신자는 시스템 봇(테오 Zalo 계정).
- **인앱 알림**: 벤더(/vendor)·파트너(/partner) 포털의 벨 아이콘. Zalo와 무관하게 항상 적재.
- **직접 발송(큐 미경유)**: 비밀번호 재설정 코드·여권 사진·파트너 Zalo·운영자 요청 통지 — 이벤트 즉시 발송.
- **운영자 알림 그룹 발송 (ADR-0039, A 시리즈)**: 설정(`/settings` → "Zalo 알림 그룹")에서 그룹방을 지정하면, 운영자 대상 A 시리즈 Zalo 알림이 **운영자 개별 DM 대신 그룹방 1건**으로 발송된다(`lib/operator-notify.enqueueOperatorNotification`). **미설정 시 기존 개별 DM fan-out**(운영자별 1건). 대상 타입 화이트리스트: A-01(직접예약)·A-02/03(빌라 등록·재제출)·A-04(콘텐츠 수정)·A-05(명단 리마인더)·A-06(원가 변경)·A-07/08/09/완료(벤더 응답)·게스트 입금통보·부가서비스 요청. **제외**: 보안 경보(SECURITY_ALERT)·Zalo 리스너 끊김(ZALO_LISTENER_DOWN)은 개인 대응/폴백 채널이 달라 그룹 라우팅 대상 아님. 파트너 변경요청 통지(직접 발송)도 그룹 설정 시 그룹방 1건으로 전환.
- **수신 조건**: Zalo 알림은 수신자의 `User.zaloUserId`(또는 파트너는 `contactZaloUid`)가 연결돼 있어야 함.
  - 현재 운영자 수신 상태: **김태진·DK 연결 완료(수신됨)** / **테오는 발신 봇 본인이라 Zalo 수신 불가** — 대시보드 배너·화면으로 확인.

## 알림이 안 갈 때 (수신 조건)

1. **Zalo 미연결** — 공급자·청소직원은 테오 Zalo로 메시지를 1통 보내면 전화번호로 자동 연결. **운영자는 자동 연결이 안 되므로** 테오 Zalo와의 대화방 생성 후 수동 연결 필요(개발 요청). 파트너·벤더는 담당자 Zalo(contactZaloUid) 등록 필요.
2. **테오 본인** — 발신 봇이 테오 계정이라 Zalo 자체가 자기 수신을 거부. 두 번째 Zalo 계정을 연결하거나 대시보드에서 확인.
3. **봇 세션 끊김** — 배포 재시작 직후 약간의 공백, credential 만료 시 QR 재로그인 필요. 이 경우 알림은 실패로 쌓였다가 재로그인 후 자동 재발송.
