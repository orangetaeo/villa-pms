## 🔴 HIGH — 우선 수정

### H1. 라이트 포털 페이지네이션 i18n 깨짐 (파트너/공급자/벤더)
- **증상**: 파트너 `/partner`에서 페이지네이션 라벨이 raw 키로 노출 — 화면에 `pagination.summary`, `pagination.perPage` 그대로 표시. 콘솔 `MISSING_MESSAGE: pagination (vi)`.
- **범위**: **ko·vi 양쪽 모두** 깨짐(파트너에서 확인). 키 자체는 messages/ko.json·vi.json에 존재(`pagination.summary/perPage/prev/next`) → **라이트 포털 클라이언트 컴포넌트에 "pagination" 네임스페이스가 제공/화이트리스트되지 않은 것**([[admin-client-namespace-whitelist]]의 라이트 포털판). PR #85 페이지네이션은 배포됐으나 i18n 제공 누락.
- **수정 방향**: 공급자/파트너/벤더(라이트) 레이아웃의 클라 네임스페이스 목록에 `pagination` 추가 + 회귀 테스트(완성도 테스트 패턴 재사용).

### H2. 파트너↔운영자 미수 잔액 불일치
- **증상**: 같은 "모두투어"의 미수 잔액이 화면마다 다름. 파트너 자기화면(/partner/receivables) **183,876,468₫** vs 운영자(/partners·/partners/[id]) **173,516,468₫**. 차이 10,360,000₫.
- **위험**: 파트너와 운영자가 "얼마 받을지/줄지"를 다르게 봄 → 정산 신뢰 직결.
- **수정 방향**: 두 화면의 미수 집계 기준(상태 필터·대손·무효 포함 여부) 일원화. 한쪽이 특정 상태를 제외하는지 확인.

### H3. 청소 직원 네비 오류 + 무한 리다이렉트
- **증상**: CLEANER 로그인 시 **공급자용 전체 탭바(빌라·캘린더·예약·청소·수익·안내)가 그대로 노출**. 청소 직원은 청소만 가능한데, 다른 탭은 접근 차단→**`ERR_TOO_MANY_REDIRECTS` 무한 리다이렉트**(레이아웃 프리페치가 루프, 콘솔 6에러).
- **수정 방향**: CLEANER 전용 최소 네비(청소 + 안내 + 프로필)로 분기. 라우트 가드가 리다이렉트 루프 대신 정상 차단되도록.

## 🟠 MEDIUM

### M1. 게스트 부가옵션 가격 표시 불일치
- **증상**: `/g/[token]/options`에서 선택한 옵션 가격과 하단 합계가 불일치. 과일도시락: "두리안 1Kg 200,000₫" 선택인데 하단 "50,000₫/1개". 마사지: "풋마사지 30분 200,000₫" 선택인데 하단 "450,000₫/60분".
- **수정 방향**: 선택 옵션 단가가 즉시 합계/스테퍼 기준가에 반영되도록. (페이지 자체는 완전 구현·주문 가능·5개 언어·누수 없음 — 표시 로직만 결함.)

### M2. 빌라명 병기 불일치 (VN 사용자 못 읽음)
- `/my-villas`·공급자 캘린더·게스트 화면은 "쏘나씨 V25 (Sonasea V25)" 병기 → 정상.
- 그러나 공급자 `/earnings` 빌라 목록·**벤더 발주함**은 **한국어 전용**("쏘나씨 V21", "마리나 베이 C21") → VN 공급자·벤더가 못 읽음.
- **수정 방향**: 비운영자 전 경로에 `formatVillaName`(병기) 일관 적용([[villa-name-bilingual]] 확장).

### M3. 공급자 수익 차트 비어 있음
- `/earnings` 통계의 "수익 추이"·"가동률 추이" 차트가 축만 있고 데이터 막대/선 미렌더(빈 차트). 데이터 기간 매핑 또는 렌더 버그.

### M4. 모바일 상단 헤더 겹침
- 공급자 다화면(/my-villas 인사말, /calendar 빌라선택칩)에서 **좌상단 계정 아이콘이 페이지 최상단 콘텐츠를 가림**. "Xin chào" 인사말이 아이콘 뒤로 잘림.

### M5. 벤더 발주함 무페이징
- `/vendor` 신규 발주 79건을 페이지네이션 없이 16,500px 한 스크롤로 렌더. (PR #85가 라이트 포털 일부에 적용됐으나 벤더 발주 목록은 누락 또는 미배포 — H1과 함께 확인.)

### M6. 게스트 만료 링크 잘못된 문구
- 만료된 게스트 체크인 링크가 **"제안이 만료되었습니다"**(제안서 문구) 표시. 체크인 링크는 제안서가 아님 → 게스트 전용 만료 안내로 교체. (1차 감사 B1.)

## 🟡 LOW / 데이터 품질

### L1. 빌라명 쓰레기 데이터 + 검증 부재
- 공급자 빌라 목록에 이름이 `,ㅏㅏㅏ`(자모만), 파트너 채권에 빌라명 `DDDD`. 빌라명 입력 검증/정규화 없음 → 운영 화면에 깨진 이름 노출.

### L2. 데모 데이터 시점·혼입
- 운영자 예약(5월말~6월초) vs 파트너/일부(9~10월) 시점 불일치 → 오늘 기준 KPI와 목록 착시. 공급자 이름에 "(데모 공급자)" KO 혼입. 김태진에 데모 대화 28건 할당. (전부 데모 한정, 실거래 무관.)

### L3. STAFF 계정 부재
- 프로덕션 유일 STAFF 계정이 **소프트삭제+비활성** 상태였음 → STAFF 권한이 실사용 검증된 적 없음(이번에 복구해 검증). 출시 전 정식 STAFF 계정 운영 필요.
