# b11-availability-board — 빌라별 공실 보드 (운영자, 다크, ko)

운영자(ADMIN)가 공급자에게 "이 날짜 비어 있나요?" 확인한 결과로 날짜를 **잠그거나 푸는** 호텔 채널매니저 스타일 타임라인 보드. 예약/금액 정보는 다루지 않는다(단순 공실/잠금 토글).

- **생성 방식**: Stitch 클라우드 타임아웃 회피 — 기존 canonical export의 다크 쉘·토큰을 재사용한 **로컬 컴포지션**(b14·b1-mobile·a9 선례 따름). 쉘=b5-bookings(사이드바·탑바·#0F172A/#1E293B/#3B82F6·Public Sans+Noto Sans KR), 셀 패턴 idiom=a3-calendar(대각 빗금). 추후 Stitch 웹 재생성 시 docs/DESIGN.md에 B11 프롬프트 추가 권장.
- **사이드바**: 8메뉴 canonical(빌라 그룹 활성). 변환 시 공통 Sidebar 컴포넌트가 9메뉴로 일괄 처리(DESIGN.md 규칙) — 화면별 재수정 금지.

## 레이아웃 (확정 사양 구현)

- 세로(행)=빌라, 가로(열)=날짜(하루 1칸, `grid-cell` 34px 폭).
- **왼쪽 빌라명 열 sticky** (`.sticky-col`, 220px, `position:sticky; left:0`), 날짜 열만 가로 스크롤.
- 기간 **3개월**(2026.06~08). 디자인 데모는 셀 폭 가독성을 위해 월당 9일로 압축 — **변환 시 실제 월 일수(28~31)로 렌더**.
- 상단 기간 네비: `◀ 2026년 6월 ~ 8월 ▶` + **[오늘]** 버튼.
- 필터 바: 빌라명 검색 + **지역(area) 드롭다운** + **"확인 필요만" 토글**.
- 헤더 2행: 월 라벨 행(`#0F172A`) + 일/요일 행(`#1E293B`). 둘 다 `position:sticky; top` 적용 → 세로 스크롤 시 날짜 헤더 고정. 좌상단 코너 셀은 가로·세로 모두 고정(z-index 30).

## 마지막 확인일 뱃지 (이 화면의 핵심 차별점)

빌라명 옆에 배치. 운영자가 "어느 빌라를 다시 물어봐야 하는지" 한눈에 파악.
- **확인됨**: `check_circle` + green(`bg-green-500/10 text-green-400`), 예 "확인 6/20".
- **확인 필요**(오래 안 본 빌라): `warning` + amber(`bg-amber-500/15 text-amber-400 border-amber-500/30`) + `animate-pulse`로 시선 유도, 예 "확인필요 5/28".
- 상단 타이틀 우측에 요약 칩 "확인 필요 N개" + 필터 "확인 필요만" 토글로 연동.
- 임계 규칙(변환 시): `lastCheckedAt` 기준 N일 경과(예: 14일) 시 확인필요로 전환. 색만이 아니라 아이콘(check_circle vs warning)으로도 구분 → 색약 대응.

## 셀 상태 토큰 (3종, 색 + 패턴 병행 — 색약 대응)

| 상태 | 색 | 패턴 | 아이콘 | 동작 |
|---|---|---|---|---|
| 공실 AVAILABLE | 투명(빈 셀), hover 시 `rgba(59,130,246,.10)` | 없음 | 없음 | 탭 → 잠금 팝오버 |
| 잠금 MANUAL block | 슬레이트 채움 `#475569` | **정방향 45° 빗금**(`repeating-linear-gradient(45deg, …)`) | 없음 | 탭 → 잠금/해제 팝오버 |
| 외부 채널 ICAL block | 호박(앰버) `rgba(245,158,11,.22)` | **역방향 -45° 빗금** | `sync`(앰버) | 탭 → **읽기전용 안내만** |

- 공급자 차단/iCal 차단은 **색(슬레이트 vs 앰버) + 빗금 방향(45° vs -45°) + 아이콘 유무** 3중 구분 → 흑백/색약에서도 식별 가능.
- 판매예약(HOLD/CONFIRMED)은 **이 보드에 표시하지 않음**(요구사항). 셀 상태는 공실/MANUAL/ICAL 3종뿐.
- 주말 열은 `col-weekend`로 살짝 어둡게, 오늘 열은 `col-today`(좌우 블루 inset), 월 경계는 `month-edge`(좌측 슬레이트 라인).

## 셀 탭 인터랙션

- **공실/MANUAL 셀** → `#popover`: 빌라명 + 날짜(YYYY.MM.DD (요일)) + 현재 상태 + **[잠금](slate)** / **[해제](outline)** 버튼 + "변경 즉시 저장됩니다" 안내. 다크용 팝오버(요구의 바텀시트 대안 — PC 다크에 적합).
- **ICAL 셀** → `#popover-ical`: 앰버 보더, "외부 채널 잠금", 출처(예: Airbnb 캘린더), 회색 안내 "이 화면에서 수정할 수 없습니다. 채널 원본에서 변경하세요." → **읽기전용**. `cursor:not-allowed`.
- 팝오버는 클릭 좌표에 띄우고 화면 밖 클릭 시 닫힘.

## 가로 스크롤 동작

- 그리드 컨테이너 `overflow:auto`, 테이블 `table-layout:fixed`. 빌라 열 sticky로 가로 스크롤 중 항상 노출.
- 모바일에서도 빌라 열 고정 + 가로 스크롤로 동일 동작(요구사항). 사이드바는 변환 시 모바일 드로어 처리(b1-mobile 패턴) — 본 디자인은 PC 1440 우선.

## 변환 메모 (FE)

- 더미 데이터(`VILLAS`, `COLS`)는 전부 Prisma 쿼리로 대체. 셀 상태는 `availability`(MANUAL) + iCal import 결과(ICAL) 합성, 예약 상태는 **제외**.
- 잠금/해제는 낙관적 업데이트 + writeAuditLog (날짜·빌라·이전/이후 상태 기록 — 글로벌 감사로그 규칙).
- 날짜 헤더 sticky `top` 값은 행 높이에 맞춰 계산(현재 34px 하드코딩 → 변환 시 변수화).
- keep-all/nowrap 전역 CSS는 공통 globals.css로 강제(DESIGN.md 규칙).

## 디자인 평가 4기준 1차 자가검토

| 기준 | 점수 | 평 |
|---|---|---|
| **디자인 품질** | 상 | b 계열 다크 토큰(#0F172A/#1E293B/#3B82F6)과 완전 응집. 타임라인 격자가 부품 나열이 아니라 하나의 "운영 도구"로 읽힘. 월/일 2단 헤더·오늘 열 강조·주말 음영으로 밀도 높지만 정돈됨. |
| **독창성** | 상 | AI 기본값(흰 카드+그라디언트) 아님. **마지막 확인일 뱃지**라는 도메인 고유 장치(공급자가 재고를 안 주는 사업 특성)를 전면에 배치 — 일반 채널매니저에 없는 의도적 선택. 빗금 방향으로 MANUAL/iCal을 구분한 것도 색약 대응이자 독창적. |
| **완성도** | 중상 | 타이포 위계(타이틀/뱃지/셀)·간격 일관. 대비 양호. **개선여지**: 34px 셀의 iCal `sync` 글리프가 다소 작아 밀집 시 빗금과 겹쳐 보임 → 변환 시 셀 폭 36~40px 또는 글리프를 셀 우상단 코너로. 압축된 9일/월은 실제 30일 렌더 시 가독성 재검증 필요. |
| **기능성** | 상 | 운영자가 추측 없이 (1) 어느 빌라를 다시 물을지(뱃지) (2) 어느 날이 비었는지(셀) (3) 잠금/해제(팝오버) (4) iCal은 못 고친다(읽기전용 안내)를 모두 파악·수행 가능. 범례가 색+패턴+아이콘을 모두 설명. |

**전략 판단(1회 수행)**: "현재 방향 다듬기 vs 미학 전환" → **다듬기 채택**. 다크 채널매니저 격자는 운영 밀도 요구에 정확히 부합하므로 미학 전환 불필요. 다듬기 항목 = ① iCal 글리프/셀 폭 가독성 ② 30일 풀 렌더 시 월 헤더 정렬 ③ 확인필요 pulse가 과하면 정적 배지로 완화. 이들은 변환 단계에서 실데이터로 재검증.
