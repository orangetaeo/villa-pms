# a8-my-statistics — Thống kê (공급자 매출·가동율 통계)

공급자(SUPPLIER) 전용 모바일 화면. "내 빌라 매출·가동율 통계".
역할: 공급자가 자기 빌라의 **자기 수익(VND)** 과 **가동율(%)** 만 한눈에 본다.

## 생성 경위 (중요)
- **Stitch MCP 생성 실패.** `generate_screen_from_text`를 2회 호출했고 둘 다
  `The operation timed out` 반환. 가이드대로 +45s / +105s / +75s 대기하며
  `list_screens`를 3회 폴링했으나 화면 수가 87개로 **변동 없음**(byte 단위 동일,
  새 "Thống kê" 화면이 프로젝트에 persist 되지 않음). → 생성이 실제로 완료되지 않음.
- 따라서 DESIGN.md 폴백 규칙에 따라 **a7-my-earnings 디자인 언어를 그대로 계승한
  정적 HTML 목업을 직접 작성**해 이 폴더에 저장. Playwright 390px 렌더 육안검증 완료
  (screenshot.png = 실제 브라우저 풀페이지 캡처).
- 후속: Stitch MCP가 복구되면 동일 프롬프트(아래)로 정식 생성·교체 권장.

## 디자인 의도
- a7(Thu nhập)과 **완전히 같은 디자인 언어**: bg-slate-50, Be Vietnam Pro, teal #0D9488,
  white rounded-xl/2xl 카드 + soft shadow, 상단 sticky 화이트 앱바, 하단 탭바.
- 정보 위계: 세그먼트 → 기간칩 → KPI(요약) → 추이 차트 2종 → 빌라별 상세. 위에서 아래로
  "전체 → 추이 → 개별" 줌인 흐름.
- 차트는 **정적 목업**: 막대는 div height, 추이선은 inline SVG(area gradient + gridline + dots,
  현재월 T7 강조). 변환 시 recharts 등으로 실데이터 연결.

## 구성 요소
- **세그먼트 컨트롤(2)**: `Thống kê`(활성, 화이트 알약) | `Chi tiết`(정산내역, 비활성).
- **기간칩(단일선택)**: `Tháng này`(활성 teal) / `Tháng trước` / `Năm nay`.
- **KPI 2x2**: Tổng doanh thu(45.000.000₫, teal) · Tỷ lệ lấp đầy(78%) · Số lượt đặt(12) ·
  Số đêm trung bình(2,8). 각 카드 = 색 아이콘원 + uppercase 라벨 + 큰 값.
- **Doanh thu theo tháng**: teal 막대 6개(T2~T7), 최고치 T7 darker teal + "45tr" 라벨.
- **Tỷ lệ lấp đầy**: 0–100% area/line 차트(teal gradient fill), T7 끝점 강조.
- **Theo từng villa**: 빌라 3장(Sonasea V11 82% / Sonasea V12 71% / Sunset Sanato A3 64%),
  각 카드 = 빌라명 + 수익(VND teal) + 가동율 progress bar + % 라벨.
- **하단 탭바(5)**: Villa / Lịch / Dọn dẹp / **Thu nhập(활성, teal 알약, 채운 아이콘)** / Hướng dẫn.

## 표기 규칙 준수
- VND **점(dot) 천단위** — 공급자 화면 규칙(45.000.000₫). 십진은 베트남식 콤마(2,8).
- 라벨 전부 베트남어. "villa" 사용("biệt thự" 금지).

## 마진 비공개 (절대원칙 — 위반 0 확인)
- 판매가(KRW) ❌ · 마진 ❌ · 환율/환산 ❌ · 고객명 ❌ — 한 픽셀도 없음.
- 공급자는 **자기 수익(VND) + 가동율(%)** 만 본다. KPI·차트·빌라카드 전부 이 두 축만 사용.

## 상태/색 토큰 (a7 계승)
- primary teal `#0D9488` / 강조막대 `teal-600` / KPI 아이콘 틴트: teal·emerald·blue·amber-50
- 진행바: 트랙 `slate-100`, 채움 `teal-500`
- 텍스트: 본문 `slate-900`, 보조 `slate-400`, 라벨 uppercase `slate-400`

## 자가검토 — 디자인 평가 4기준 (1차)
| 기준 | 채점 | 메모 |
|---|---|---|
| 정보 위계 | 양호 | 요약(KPI)→추이→개별 줌인. 큰 숫자 = 가독성 1순위. 차트 제목+단위칩으로 맥락 명확. |
| 터치 타겟 | 양호 | 세그먼트 h-11(44px), 기간칩 h-10(40px), 탭바 h-20. 칩은 살짝 작음 → 변환 시 44px 권장. |
| 베트남어 단순성 | 양호 | 1화면=조회 1과업, 입력 0, 버튼(조작) 최소. 추측 없이 "내 수익·가동율" 파악 가능. |
| 브랜드 일관성 | 우수 | a7 토큰·앱바·탭바·카드 스타일 1:1 계승. 부품 나열이 아닌 하나의 정체성. |

**"다듬기 vs 미학 전환" 판단**: a7과의 강한 가족유사성이 공급자 앱 전체 일관성에
필수이므로 **현재 방향 다듬기**를 선택(미학 전환 불필요). 적용한 다듬기:
KPI에 아이콘원 추가로 스캔성↑, 차트에 단위칩(₫/triệu, 0–100%)으로 모호함 제거,
현재월(T7) 강조로 "지금"이 어디인지 즉시 인지.

**잔여 개선(변환 단계 권고)**:
1. 기간칩 터치 높이 44px로 통일.
2. KPI/차트/빌라 데이터를 Prisma 집계로 대체(가동율 = 예약일수/가용일수), VND 점표기 헬퍼 일원화.
3. "Chi tiết" 세그먼트 클릭 시 a7-my-earnings(정산내역)로 라우팅 — 두 화면이 한 메뉴의
   두 탭임을 변환 시 명시.
4. 빈 상태(데이터 없는 신규 공급자) 화면 추가 필요.

## 인계
- UX-VN에 변환 인계: "베트남 중계인이 설명 없이 자기 수익·가동율을 읽을 수 있는가" 기준 동반 점검.
- 최종 채점은 QA(작성자 자가채점은 무효).
