# a-supplier-checkout — 공급자 직접 체크아웃 검수 (F10 Phase A D5, ADR-0021 §6)

- 영역: 공급자(SUPPLIER) / vi / 모바일 390px
- 디자인 시스템: Villa PMS Light (`assets/16166333700622029900`)
- 운영자 b4-checkout(ko, 다크)을 **vi 라이트 모바일로 재해석**

## 기획 의도 (ADR-0021 D5)
1) **공간별 기준 사진 ↔ 현재 사진 비교 그리드**(Ảnh gốc / Ảnh hiện tại, 방별 카드, Đã chụp/Chưa chụp 상태) 2) **파손 토글(Hư hỏng)** — ON 시 파손 사진 + 차감액(Số tiền trừ, VND, 빨강) 노출 3) **보증금 정산 요약**(cọc − 차감 = hoàn lại, teal 카드) 4) "Hoàn tất trả phòng" → CleaningTask 생성 + isSellable=false

## b4 대비 핵심 차이
- 다크/블루 → 라이트 teal, ko → vi, PC 2열 → 모바일 세로 스택 카드(방별 2-up 비교)
- ~~b4의 "미니바 확인(읽기전용)" 블록 제외~~ → **ADR-0021 D6으로 미니바 소모 정산 섹션 추가**(미니바=운영자 재고·매출, 공급자 직접판매 게스트 소비분도 체크아웃에서 정산). 보증금/파손 ↔ CTA 사이 배치. 품목별 소비수량 스테퍼 × **판매가(VND)만** = 항목금액 → Tổng minibar 합계. 원가(costVnd)·마진 절대 비노출. 실재고(ADR-0019)는 Phase 2 미구축이라 현재는 수기 입력 UI(연동 시 남은수량 자동계산)
- 통화 VND dots만. 차감액·환불은 빨강/teal로 의미색 일관

## 마진·재고 비공개 (확인 PASS)
- 판매가·마진·KRW 없음. 보증금/차감/환불 전부 VND. 다른 예약·전체 재고 노출 없음

## UX-VN 점검
- 파손 토글 캡션 "Bật khi có hư hỏng…"로 토글 의미 명시(비기술 사용자)
- 정산 요약은 산식이 한눈에(받은 cọc → 차감 → 환불) 떨어지게 teal 카드 1장

## 변환 메모
- 방 목록·기준 사진은 빌라 등록 사진 파이프라인에서, 현재 사진은 신규 업로드
- 파손 OFF 기본값, ON 시에만 차감 필드 표시·합산
- fixed 하단 CTA는 fullPage 캡처 겹침 = 캡처 아티팩트(실기기 정상)
