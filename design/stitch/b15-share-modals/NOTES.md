# b15-share-modals — 디자인 노트 (ADR-0009)

## 화면 의도
b14 채팅에서 호출되는 공유 선택 모달 3종 + 상대 분류 컨트롤 + 별명 편집 + 첨부 메뉴 3변형을 한 파일에 디자인 참고로 모음. 다크 운영자 톤(b14 토큰 동일). 변환 시 각각 별도 컴포넌트.

## 구성 (6 블록)
1. **빌라 선택 모달** — 검색 + 빌라 리스트(체크 선택) + 시즌/사진 옵션. 공급자 대화면 그 공급자 빌라 우선 + 원가 표기.
2. **제안 선택 모달** — 활성 제안 리스트(만료 카운트다운) + /p/[token] 링크 안내. 고객 대화 전용.
3. **정산 선택 모달** — 본인(supplierId=userId) 월 정산 리스트(VND) + "본인 정산만" 가드 안내. 공급자 대화 전용.
4. **상대 분류 컨트롤** — 미분류 대화 분류 배너(공급자/고객) + 분류 후 헤더 드롭다운 재변경.
5. **별명 편집 모달** — 인풋 + 해제 안내(빈값) + Zalo 원본 보존 표기 (D9).
6. **첨부 메뉴 3변형** — 공급자/고객/미분류별 가시성 (D2 누수 매트릭스 시각화).

## 누수 차단 (정확한 필드)
- **빌라 선택 모달(공급자)**: 이름·사진·단지·침실/욕실·**원가만**. 마진·판매가·KRW 없음.
- (고객 변형은 동일 컴포넌트가 counterpartyType=CUSTOMER일 때 **판매가만** 표기 — 원가·마진 없음. 변환 시 select 화이트리스트 분기, D4.1)
- **제안 모달**: 고객 전용. 표기 금액은 제안 총 판매가(채널 통화: VND 쉼표 / KRW 원). 원가·마진 없음. 하단에 "판매가만 표시(원가·마진 비노출)" 명시.
- **정산 모달**: 공급자 전용. 본인 정산 총 지급액(VND)·건수·상태. 판매가·마진·타 공급자 없음. "본인 정산만 공유 가능" 가드.
- **마진**: 전 모달·메뉴에 없음.
- VND 표기: 원가/정산(공급자 맥락)은 점(2.500.000₫), 제안 판매가(고객 맥락 여행사)는 쉼표(9,800,000₫), 직접 채널은 원(1,280,000원) — DESIGN.md 통화 규칙.

## 상태색
빌라/제안 선택=blue 강조, 정산=amber 강조, 공급자=teal, 고객=indigo, 만료 임박=red, 만료 여유=amber 카운트다운.

## 자가 채점 (4기준)
- 디자인 품질 (상): b14 토큰 완전 일치, 6블록이 하나의 시스템.
- 독창성 (중상): 모달마다 강조색을 공유 타입에 매핑(빌라 blue·정산 amber)해 누수 경계를 색으로 학습 가능하게 함.
- 완성도 (상): 체크 선택 패턴·하단 가드 배너·버튼 쌍 일관. keep-all/nowrap 적용.
- 기능성 (상): 선택→옵션→공유 3스텝 명확. 잘못된 상대 타입엔 해당 모달이 애초에 열리지 않음(메뉴 가시성).
- **다듬기 vs 전환**: b14와 동일 미학 유지(다듬기). 모달은 운영자 데이터 밀도에 맞는 검증된 카드 패턴 재사용.

## 변환 인계 (FE)
- 3개 모달은 별도 컴포넌트. 빌라 모달은 counterpartyType으로 원가/판매가 select 분기(서버 buildVillaShareText, D4.1).
- 분류 컨트롤 → PATCH(분류 액션), 분류 전 빌라/제안/정산 메뉴 비활성.
- 별명 모달 → PATCH SET_NICKNAME(빈값→null 정규화, 1~40자).
