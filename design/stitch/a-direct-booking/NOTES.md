# a-direct-booking — 직접 예약 기록 바텀시트 (F10 Phase A, ADR-0021 §6)

- 영역: 공급자(SUPPLIER) / vi / 모바일 390px
- 디자인 시스템: Villa PMS Light — Supplier/Public (`assets/16166333700622029900`), a3·a14 계승
- 진입: `/calendar`에서 빈 날짜 탭 → 바텀시트

## 기획 의도 (ADR-0021 §6 1·2)
빈 날짜 탭 시 **기존 "차단(Khóa ngày)"과 "직접 예약 기록(Ghi đặt phòng trực tiếp)"이 공존**하는 2-옵션 칩. 직접예약 옵션 active 상태로 폼 표시.
- 입력(텍스트 최소): 체크인/아웃(선택값 read), **고객명(유일 필수 텍스트, Tên khách \*)**, 인원(스테퍼 ±), (선택)받은 금액 VND(Số tiền nhận · Không bắt buộc), (선택)연락처(Liên hệ · Không bắt buộc)
- 저장: "Lưu đặt phòng" — 제출 시 `POST /api/supplier/bookings`(가용성 게이트 통과) 연결

## 마진·재고 비공개 (확인 PASS)
- KRW 없음 / 마진 없음 / 판매가 없음. 금액은 **공급자가 자기 고객에게 받은 금액(supplierSalePriceVnd)**, VND dots(4.500.000₫)만. 선착순 패배 시 "이미 예약됨"은 별도 상태(여기 미표현 — 토스트/배너로 변환).

## UX-VN 원칙 점검 ("설명 없이 쓸 수 있는가")
- 1화면 1작업, 버튼 위계 명확(저장 1개), 스테퍼·숫자패드 중심, 텍스트 입력은 고객명 1개만 필수
- "Không bắt buộc" 칩으로 선택 항목 명시 → 비기술 중계인이 부담 없이 건너뜀

## 변환 메모
- 날짜 행은 선택 범위에서 자동 채움(read), 탭 시 캘린더로 복귀 편집
- 2-옵션 칩: "Khóa ngày" 선택 시 기존 a3 차단 시트로 분기, "Ghi…" 선택 시 본 폼
- 금액 입력은 dot 천단위 자동 포맷(vi 규칙)
